自行實作 Functor

2020-11-17

Functor 是 Monadic Programming 中最基本 Container，透過自行實作 Functor 可更清楚了解 Functor 本質。

Version

ECMAScript 2015

Class

import { map, inc } from 'ramda'

class Functor {
  constructor(x) {
    this._x = x
  }

  map(f) {
    return new Functor(f(this._x))
  }
}

let functor = new Functor(2)
map(inc, functor) // ?

使用 class 實現 container。

第 4 行

constructor(x) {
  this._x = x
}

在 constructor() 內將 x 值存進 _x 內。

第 8 行

map(f) {
  return new Functor(f(this._x))
}

Functor 俗稱 Mappable，白話就是具有 map() 的 container，定義 map()。

將 container 內部 _x 透過傳入 f() 運算後，重新使用 Functor.of() 包成 Functor 回傳。

13 行

let functor = new Functor(2)
map(inc, functor) // ?

由 new Functor 建立 functor，因為符合 Functor 規定，因此可使用 Ramda 的 map()。

map()
Functor f => (a → b) → f a → f b

Ramda 的 map() 規定 f a container 的 f 為 Functor。

可發現只要符合 Functor 定義的 container，就可使用 Ramda 的 map()

Prototype

import { map, inc } from 'ramda'

let Functor = function(x) {
  this._x = x
}

Functor.prototype.map = function(f) {
  return new Functor(f(this._x))
}

let functor = new Functor(2)
map(inc, functor) // ?

也可使用 prototype 實現 container。

第 3 行

let Functor = function(x) {
  this._x = x
}

以 constructor function 實現 constructor，並將 x 存進 _x 內。

第 7 行

Functor.prototype.map = function(f) {
  return new Functor(f(this._x))
}

直接在 prototype 建立 map() instance method。

13 行

let functor = Functor.of(2)
map(inc, functor) // ?

用法完全不變。

Array is Functor

import { map, inc } from 'ramda'

let data = [1, 2, 3]
map(inc, data) // ?

由上可知無論使用 prototype 或 class 寫法，重點是該 container 必須支援 map() 才可稱為 Functor。

[1, 2, 3] 為 Array，由於 ECMAScript 天生已具由 Array.prototype.map()，因此 Array 就是 Functor。

只是 Array 為一般性觀念，而 Functor 為 Monadic Programming 觀念，只是看法角度不同。

Conclusion

  • Functor 為 Monadic Programming 基礎，是最基本 container，其定義是必須支援 map()
  • Functor 可使用 prototype 或 class 實踐，可發現 OOP 將 class 視為 type，而 FP 將 class 視為 container
  • 因為 Array 天生就支援 map()，因此 Array 就是 Functor
