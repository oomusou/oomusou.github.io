Functor、Apply 與 Chain 簡單比較

2021-07-07

Functor、Apply 與 Chain 本質都是 Typeclass，只是各提供不同方式處理 Function 而已。

Version

Sanctuary 3.1.0

Functor

import { Just, map, add } from 'sanctuary'

let data = Just (1)

let f = map (add (1))

f (data) // ?

data 為 Just 1，我們希望提供 function 使其成為 Just 2。

Functor 提供 map 讓我們直接傳入 function 以改變內部值，因此可直接傳入 add (1) 進 map 即可。

map
Functor f => (a -> b) -> f a -> f b
將 a -> b 傳入 Functor 以改變 Functor 內部值

a -> b：傳入 unary function

f a：data 為 Functor

f b：回傳新 Functor

Apply

import { Just, ap, add } from 'sanctuary'

let data = Just (1)

let f = ap (Just (add (1)))

f (data) // ?

data 仍為 Just 1，我們希望提供 function 使其成為 Just 2。

Apply 提供 ap 讓我們將 function 包進 Apply 後再傳入以改變內部值，因此可將 add (1) 包進 Just 後再傳入 ap 即可。

ap
Apply f => f (a -> b) -> f a -> f b
將 f (a -> b) 傳入 Apply 以改變 Apply 內部值

f (a -> b)：傳入包在 Apply 的 unary function

f a：data 為 Apply

f b：回傳新 Apply

Chain

import { compose, Just, chain, add } from 'sanctuary'

let data = Just (1)

let add_ = n => compose (Just) (add (n))

let f = chain (add_ (1))

f (data) // ?

data 仍為 Just 1，我們希望提供 function 使其成為 Just 2。

Chain 提供 chain 讓我們將回傳 Chain 的 function 傳入以改變內部值，因此先將 add 改成 add_ 回傳 Chain，再將 add_ 傳進 chain 即可。

chain
Chain m => (a -> m b) -> m a -> m b
將 a -> m b 傳入 Chain 以改變 Chain 內部值

a -> m b：傳入回傳 Chain 的 function

m a：data 為 Chain

m b：回傳新 Chain

Conclusion

  • 為什麼本文都以 Maybe 討論呢 ? 因為 Maybe 既是 Functor、Apply 也是 Chain，因此 map、ap 與 chain 都可使用，只是傳入 function 形式不同而已
