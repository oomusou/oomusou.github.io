自行實作 Monad

2020-11-16

Monad 一直是很玄的概念，其實只要自行實作一次，也就沒這麼遙不可及，事實上 Promise、Maybe 與 Either 概念都來自於 Monad。

Version

ECMAScript 2015

Class

class Monad {
  constructor(x) {
    this._x = x
  }

  static of(x) {
    return new Monad(x)
  }

  map(f) {
    return Monad.of(f(this._x))
  }

  chain(f) {
    return f(this._x)
  }

  get() {
    return this._x
  }
}

let inc = x => x + 1
let inc_ = x => Monad.of(x + 1)

Monad.of(1).map(inc).get() // ?
Monad.of(1).chain(inc_).get() // ?

實現 Monad 需滿足 4 個條件：

  • Type Constructor：建立 Monad container
  • Unit Function：將普通 value 包進 Monad，並回傳 Monad
  • Bind Function：用來改變 Moand 內部值，來自於 Functor 故仍是 map()
  • Chain Function：當提供 function 回傳 monad 時，可避免兩層 monad

第 2 行

constructor(x) {
  this._x = x
}

實現 type constructor，除了建立 Monad 外，並將 x 包進 _x。

第 6 行

static of(x) {
  return new Monad(x)
}

實現 unit function，使用 Monad constructor 建立 Monad 並回傳。

10 行

map(f) {
  return Monad.of(f(this._x))
}

實現 bind function，將 Monad 內的值套用在傳入 function，因為 f() 只會回傳普通 type，因此還要使用 unit function 包一遍才會成為 Monad。

14 行

chain(f) {
  return f(this._x)
}

實現 chain function，將 Monad 內的值套用在傳入 function，因為傳入 f() 已經回傳 Monad，因此不必再使用 unit function，否則會變成兩層 Monad。

18 行

get() {
  return this._x
}

取出 Monad 內部值。

23 行

let inc = x => x + 1
let inc_ = x => Monad.of(x + 1)

inc() 與 inc_() 都會 +1，唯 inc() 傳回正常值，而 inc_() 回傳 Monad。

26 行

Monad.of(1).map(inc).get() // ?
Monad.of(1).chain(inc_).get() // ?

皆可使用 Method Chaining：

  • map()：串接一般 function
  • chain()：串接回傳 Monad 的 function

Prototype

let Monad = function(x) {
  this._x = x
}

Monad.of = x => new Monad(x)

Monad.prototype.map = function(f) {
  return Monad.of(f(this._x))
}

Monad.prototype.chain = function(f) {
  return f(this._x)
}

Monad.prototype.get = function() {
  return this._x
}

let inc = x => x + 1
let inc_ = x => Monad.of(x + 1)

Monad.of(1).map(inc).get() // ?
Monad.of(1).chain(inc_).get() // ?

Monad 亦可使用 prototype 實現。

第 1 行

let Monad = function(x) {
  this._x = x
}

以 constructor function 實現 constructor，並將 x 存進 _x 內。

第 5 行

Monad.of = x => new Monad(x)

直接對 Maybe container 建立 of() static method，因為沒使用 this，可直接使用 arrow function。

第 7 行

Monad.prototype.map = function(f) {
  return Monad.of(f(this._x))
}

直接在 prototype 建立 map() instance method。

11 行

Monad.prototype.chain = function(f) {
  return f(this._x)
}

直接在 prototype 建立 chain() instance method。

15 行

Monad.prototype.get = function() {
  return this._x
}

直接在 prototype 建立 get() instance method。

19 行

let inc = x => x + 1
let inc_ = x => Monad.of(x + 1)

Monad.of(1).map(inc).get() // ?
Monad.of(1).chain(inc_).get() // ?

這部分完全相同。

Conclusion

  • Monad 擁有 Functor 所有特性，所以要實作 map()
  • Monad 與 Functor 最大不同在於 chain()，這使得 Moand 不會有多層 Functor 老問題

Reference

Lius Atencio, Functional Programming in JavaScript
