深入淺出 Monad Laws

2021-07-14

Monad 遵循 3 個 Monad Laws，專門描述 chain 的 3 個特性。

Version

Sanctuary 3.1.0

Left Identity

import { inc } from 'ramda'
import { compose, Just, chain } from 'sanctuary'

let data = 1

let f = compose (Just) (inc)

chain (f) (Just (data)) // ?
f (data) // ?

當存在回傳 Monad 的 function 時：

  • 使用 chain 將 Monad 與回傳 Monad 的 function 綁定
  • 直接將 value 傳入回傳 Monad 的 function

兩者結果相同。

也就是 chain 能將 Moand 與回傳 Monad 的 function 綁定，然後改變 Monad 內部 value。

Left Identity
return a >>= f === f a

  • return a >>= f：使用 chain 將 Monad 與 function 綁定
  • f a：value 直接傳入 function
  • ===：兩者結果相同

Right Identity

import { Just, chain } from 'sanctuary'

let data = 1

let f = Just

chain (f) (Just (data)) // ?
Just (data) // ?

當存在 return 時：

  • 使用 chain 將 Monad 與 return 綁定
  • 將 value 直接包進 Monad

兩者結果相同。

也就是將 Monad 與 return 綁定時不會改變 Monad。

Right Identity
m >>= return === m

  • m >>= return：使用 chain 將 Monad 與 return 綁定
  • m：Monad
  • ===：兩者結果相同

Associativity

import { compose, Just, add, mult, chain } from 'sanctuary'

let data = 1

let f = compose (Just) (add (1))
let g = compose (Just) (mult (2))

chain (g) (chain (f) (Just (data))) // ?
chain (Just) (chain (g) (f (data))) // ?
  • 將 value 先包進 Monad，再透過 f 與 g 運算
  • 將 value 先經過 f 與 g 運算，最後再包進 Monad

兩者結果相同。

也就是先將 value 包進 Monad 再經過 function 運算，與 value 先將過 function 運算最後包進 Monad 結果相同。

Associativity
(m >>= f) >>= g === m >>= (\x -> f x >>= g)

  • (m >>= f) >>= g：使用 chain 將 Monad 先與 f 綁定，結果再使用 chain 與 g 綁定
  • m >>= (\x -> f x >>= g)：value 先經過 f 運算，再使用 chain 將 Monad 與 g 綁定，最後再使用 chain 將 Monad 與 return 綁定
  • ===：兩者結果相同

Conclusion

  • 可發現 Haskell 的 >>= infix 表示法特別適合描述 Monad Laws
  • return 在 Haskell 用來將 value 建立 Monad；在 Fandasy Land 則稱為 of，為 Applicative 必備 function，如 of (Maybe) 或 Just
  • >>= 在 Haskell 稱為 bind，在 Fantasy Land 稱為 chain，為 Chain 必備 function
