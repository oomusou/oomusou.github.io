ECMAScript 之 Optional Chaining Operator

2020-01-03

ECMAScript 2020 支援了 ?: Optional Chaining Operator，至此 ECMAScript 終於有專屬的 Operator 判斷 Nested Object，不必再借用 && 與 Falsy Value。

Version

macOS Catalina 10.15.2
WebStorm 2019.3.1
Babel 7.6.4
ECMAScript 2020

Object

let book = {
  title: 'FP in JavaScript',
  price: {
    amazon: 100,
    google: 200
  }
}

book.price && book.price.amazon |> console.log
book.fee && book.fee.amozon |> console.log

price 為 object，若不確定 price 是否存在，傳統會先使用 book.price 判斷，再搭配 && 若存在才讀取 object 的 property。

如 fee object 根本不存在，則 && 右側不會執行，直接 short-circuiting 回傳 undefined。

但 && 其實是判斷 truthy value，而 1、non-empty string 與 true 皆是 truthy value，因此可能誤判。

let book = {
  title: 'FP in JavaScript',
  price: {
    amazon: 100,
    google: 200
  }
}

book.price?.amazon |> console.log
book.fee?.amozon |> console.log

ES2020 支援了 ?.，只判斷 null 與 undefined，因此不用擔心誤判。

|> 為 pipeline operator，目前還在 stage 1，可安裝 Babel plugin 使用

Function

let book = {
  title: 'FP in JavaScript',
  price: 100,
  showDescription: function() {
    return `${this.title} : ${this.price}`
  }
}

book.showDescription?.() |> console.log
book.show?.() |> console.log

若呼叫不存在的 method，會直接 runtime error，傳統也會使用 && 檢查 method 是否存在才呼叫，ES2020 可在 method 名稱之後加上 ?.()，若 method 不存在則回傳 undefined，若存在則執行該 method 回傳結果。

let book = {
  title: 'FP in JavaScript',
  price: 100,
  showDescription: function(onError) {
    try {
      throw new Error('Exception occurred')
      return `${this.title} : ${this.price}`
    }
    catch(err) {
      onError?.(err.message);
    }
  }
}

book.showDescription(console.log) |> console.log

實務上若 function 的傳入為 optional，則可使用 ?. 判斷，若有傳入則執行，否則不執行。

Array

let foo = [1, 2, 3]
let too = undefined

foo?.[1] |> console.log
too?.[1] |> console.log

?. 也能用在 array，會先判斷 array 是否為 undefined，若不是才取其 index。

Combine with ??

let book = {
  title: 'FP in JavaScript',
  price: {
    amazon: 100,
    google: 200
  }
}

book?.name ?? "Unknown name" |> console.log

?. 也常與 ?? 搭配使用，先使用 ?. 判斷 object 是否存在，再使用 ?? 判斷 property 是否存在。

Conclusion

  • ?. 已經是 ES2020 標準，V8 8.0 已經支援，Node 也應該會很快支援，現階段可先搭配 Babel 使用
  • ?. 也如同 ?? 支援 short-circuiting，如 ?. 左側是 null 或 undefined，則 ?. 將不會執行
