ECMAScript 之 Nullish Coalescing Operator

2020-01-03

ECMAScript 2020 支援了 ?? Nullish Coalescing Operator，至此 ECMAScript 終於有專屬的 Operator 判斷 null 與 undefined，不必再借用 || 與 Falsy Value。

Version

macOS Catalina 10.15.2
WebStorm 2019.3.1
Babel 7.6.4
ECMAScript 2020

Default Value

let fn = str => {
  let name = str || 'World'
  return `Hello ${name}`
}

fn() |> console.log
fn('Sam') |> console.log

第 2 行

let name = str || 'World'

傳統我們會借用 || 判斷 null 或 undefined，但 || 本意是判斷 falsy value，因此 0，empty string、false 皆是 falsy value，因此可能誤判。

let fn = str => {
  let name = str ?? 'World'
  return `Hello ${name}`
}

fn() |> console.log
fn('Sam') |> console.log

ES2020 支援了 ??，只判斷 null 與 undefined，因此不用擔心誤判。

|> 為 pipeline operator，目前還在 stage 1，可安裝 Babel plugin 使用

Short-circuiting

let foo = () => undefined
let too = () => false
let hello = () => 'Hello World'

foo() ?? hello() |> console.log
too() ?? hello() |> console.log

?? 如同 && 與 ||，只有當 ?? 左側為 null 或 undefined 時才會執行右側的 expression，否則會 short-circuiting 不執行。

No Chainging with AND or OR Operator

let fn = () => null || undefined ?? "foo"

fn() |> console.log

若與 && 或 || 搭配時，並沒有所謂 由左至右 規則，會直接 syntax error。

let fn = () => (null || undefined) ?? "foo"

fn() |> console.log

必須手動加上 ()。

Conclusion

?? 已經是 ES2020 標準，V8 8.0 已經支援，Node 也應該會很快支援，現階段可先搭配 Babel 使用
