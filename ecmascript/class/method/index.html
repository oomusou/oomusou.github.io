Class 的 Prototype Method 與 Static Method

2019-08-19

在使用 Array 時，我們會發現有兩類 Method：一類如 Array.from() ，另一類如 Array.prototype.map()，使用方式也有差別 ，這在 ES5 與 ES6 該如何實現呢 ?

Version

macOS Mojave 10.14.5
VS Code 1.37.1
Quokka 1.0.240
ECMAScript 5
ECMAScript 2015

ECMAScript 2015

Constructor & Prototype Method

class Rectangle {
  constructor(width, height) {
    this.width = width;
    this.height = height;
  }

  calcArea() {
    return this.width * this.height;
  }
}

let rectangle = new Rectangle(10, 20);
rectangle.calcArea(); // ?

先介紹較簡單的 ES6。

第 1 行

class Rectangle {
  constructor(width, height) {
    this.height = height;
    this.width = width;
  }

  calcArea() {
    return this.width * this.height;
  }
}

ES6 迎來了 class 語法：

  • constructor()：class 內只能有一個 constructor，負責初始化 property
  • calcArea()：prototype method，可自行定義 object 的 method

為什麼稱 prototype method，而非一般 OOP 稱 instance method ? 稍後會解釋

12 行

let rectangle = new Rectangle(10, 20);
rectangle.calcArea(); // ?

由於 calcArea() 是 object 的 method，因此必須 new 成 object 後才能使用。

這在 OOP 是很標準寫法，有 class 與 new，非常直覺易懂。

這就類似 Array.prototype.map()，須建立 array object，才能由 Array object 呼叫 map()

Static Method

class Rectangle {
  constructor(width, height) {
    this.width = width;
    this.height = height;
  }

  calcArea() {
    return this.width * this.height;
  }

  static create(width) {
    return new Rectangle(width, width);
  }
}

let square = Rectangle.create(10);
square.calcArea(); // ?

我們知道 正方形 為 長方形 的特例，且 width = height，因此希望 constructor 只提供一個 argument 即可，但目前 constructor 有兩個 argument，且 ES6 限定一個 class 只能有一個 constructor，因此無法以 constructor overload 提供只有一個 argument 的 constructor。

11 行

static create(width) {
  return new Rectangle(width, width);
}

雖然一個 class 只能有一個 constructor，但卻能有無限 static method，因此可自行建立只有一個 argument 的 create()，再由此 method 由 new 呼叫 constructor 建立 object。

此 pattern 也稱為 named constructor，適用於只能有單一 constructor 的語言，如 PHP、ECMAScript

16 行

let square = Rectangle.create(10);
square.calcArea(); // ?

有別於 prototype method 屬於 object，static method 是屬於 class，因此可由 Rectangle class 直接使用 create()，不需要 new 成 object。

這也是 OOP 很標準寫法，由 class 呼叫 static method，非常直覺易懂。

這就就類似 Array.from()，無須建立 array，可直接由 Array class 呼叫 from()

ECMAScript 5

Constructor Function & Prototype Function

var Rectangle = (function () {
  function Rectangle(width, height) {
    this.width = width;
    this.height = height;
  }

  Rectangle.prototype.calcArea = function() {
    return this.width * this.height;
  }

  return Rectangle;
})();

var rectangle = new Rectangle(100, 200);
rectangle.calcArea(); // ?

回到 ES5，在沒有 class 世界，該如何實現 prototype method 與 static method 呢 ?

第 1 行

var Rectangle = (function () {
  ...
})();

宣告 Rectangle()，使用 IIFE 定義出 scope，相當於 ES6 的：

 class Rectangle {
   ...
 }

也因為 Rectangle 是 function，證明 class 是由 function 模擬出來的

第 2 行

function Rectangle(width, height) {
  this.width = width;
  this.height = height;
}

Rectangle() 為 constructor function，將來會搭配 new 建立 object，且 this 的 execution context 會指向 object，因此可透過 this 存取 property。

一般 function 是使用 camelCase，但 constructor function 會使用 CamelCase

相當於 ES6 的：

constructor(width, height) {
  this.width = width;
  this.height = height;
}

第 7 行

Rectangle.prototype.calcArea = function() {
  return this.width * this.height;
}

在 Rectangle.prototype 下新增 calcArea()，因此 object 都有 calcArea()。

類似在 Array.prototype 下定義 map()，因此 array object 都有 map()

相當於 ES6 的：

calcArea() {
  return this.width * this.height;
}

在此更可理解為什麼 ECMAScript 稱此為 prototype method，而非 instance method，因為該 method 實際上是定義在 prototype 下。

11 行

return Rectangle;

之前只定義了 Rectangle() 與 Rectangle.prototype.calcArea()，且包在 IIFE 內，因此必須 return 之後，外界才用得到，類似 ES6 的 export 概念。

14 行

var rectangle = new Rectangle(100, 200);
rectangle.calcArea(); // ?

使用 new 建立 object，並呼叫 calcArea()，這與 ES6 與一般 OOP 相同。

Static Function

var Rectangle = (function () {
  function Rectangle(width, height) {
    this.width = width;
    this.height = height;
  }

  Rectangle.prototype.calcArea = function() {
    return this.width * this.height;
  }

  Rectangle.create = function(width) {
    return new Rectangle(width, width);
  };

  return Rectangle;
})();

var square = Rectangle.create(100);
square.calcArea(); // ?

Prototype method 要定義在 prototype 下，那 static method 要定義在哪裡呢 ?

11 行

Rectangle.create = function(width) {
  return new Rectangle(width, width);
};

在 constructor function 下新增 create() ，也由於 ES6 class 相當於 ES5 的 constructor function，因此在 constructor function 新增 function 就相當於 class 的 static method。

相當於 ES6 的：

static create(width) {
  return new Rectangle(width, width);
}

18 行

var square = Rectangle.create(100);
square.calcArea(); // ?

由 constructor function 呼叫 create()，這與 ES6 與一般 OOP 由 class 呼叫 static method 相同。

Conclusion

  • 由 ES5 可發現 instance method 是掛在 prototype 下，因此稱為 prototype method，且 class 就是 constructor function 的 syntactic sugar 而已
  • Prototype method 要定義在 constructor function 的 prototype 下，而 static method 直接定義在 constructor function 下即可
  • 其實認真思考 ES5 的 syntax，也是有它的邏輯，只是沒 class 寫法可讀性高而已，也比較冗長
