使用 splice() 直接修改原本 Array

2020-01-21

splice() 是功能很強的 Method，可處理 Insert、Update 與 Remove，唯它是少數幾個會修改原本 Array 的 Method，須小心使用。

Version

macOS Catalina 10.15.2
VS Code 1.41.1
Quokka 1.0.274
ECMAScript 2015
Ramda 0.26.1

insert()

let data = [0, 1, 3, 4]

data.splice(2, 0, 2)

data // ?

欲從指定 index 之前新增單一值至 array，splice() 會直接修改 array，而非回傳新的 array。

import { insert } from 'ramda'

let data = [0, 1, 3, 4]

insert(2)(2)(data) // ?

data // ?

比較好的方式是改用 Ramda 的 insert()，它不會修改原本 array。

insertAll()

let data = [0, 1, 4]

data.splice(2, 0, 2, 3)

data // ?

欲從指定 index 之前新增多值至 array，splice() 會直接修改 array，而非回傳新 array。

import { insertAll } from 'ramda'

let data = [0, 1, 4]

insertAll(2)([2, 3])(data) // ?

比較好的方式是改用 Ramda 的 insertAll()，它不會修改原本 array，而是回傳新 array。

update()

let data = [0, 1, 2, 3]

data.splice(2, 1, 4)

data // ?

欲從指定 index 修改 array 單一值，splice() 會直接修改 array，而非回傳新 array。

import { update } from 'ramda'

let data = [0, 1, 2, 3]

update(2)(4)(data) // ?

data // ?

比較好的方式是改用 Ramda 的 update()，它不會修改原本 array，而是回傳新 array。

updateAll()

let data = [0, 1, 3, 4]

data.splice(2, 2, 2, 3)

data // ?

欲從指定 index 修改多值至 array，splice() 會直接修改 array，而非回傳新 array。

import { remove, insertAll, pipe } from 'ramda'

let data = [0, 1, 3, 4]

let updateAll = idx => arr => pipe(
  remove(idx)(arr.length),
  insertAll(idx)(arr)
)

updateAll(2)([2, 3])(data) // ?

data // ?

Ramda 並沒有提供 updateAll()，但我們可組合 remove() 與 insertAll()：

  • 先使用 remove() 從指定 index 移除等數量 element
  • 再使用 insertAll() 從指定 index 新增 array

updateAll() 不會修改原本 array，而是回傳新 array。

remove()

let data = [0, 1, 2]

data.splice(1, 1)

data // ?

欲從 array 刪除指定 index 的 element，splice() 會直接修改 array，而非回傳新的 array。

import { remove } from 'ramda'

let data = [0, 1, 2]

remove(1)(1)(data) // ?

data // ?

比較好的方式是改用 Ramda 的 remove()，它不會修改原本 array，而是回傳新 array。

let data = [0, 1, 2]

data.splice(1)

data // ?

欲從 array 刪除指定 index 後所有 element，splice() 會直接修改 array，而非回傳新的 array。

import { remove } from 'ramda'

let data = [0, 1, 2]

remove(1)(Infinity)(data) // ?

data // ?

比較好的方式是改用 Ramda 的 remove()，它不會修改原本 array，而是回傳新 array。

但 remove() 須明確提供第二個 argument，因此傳入 Infinity 。

Conclusion

  • splice() 功能強大，新增、修改、刪除皆可包辦，但 side effect 亦強大，應小心使用，實務上建議改用 Ramda 所提供的 pure function
