使用 entries() 獲得 Array 的 Index 與 Element

2020-07-14

ECMAScript 2015 迎來了 Array.prototype.entries()，可同時回傳 Array 的 Index 與 Element。

Version

macOS Catalina 10.15.5
VS Code 1.47.0
Quokka 1.0.307
ECMAScript 2015

Definition

Array.prototype.entries()
回傳 Array Iterator，其中包含 index 與 element

entries() 最大的特色就是除了回傳 element 外，還包含了 index，但其型別為 Array Iterator。

For Of Loop

let data = ['Sam', 'Kevin', 'Jimmy']

let f = a => {
  let result = []
  let elements = a.entries()

  for(let [i, x] of elements)
    result.push(`${i}: ${x}`)

  return result
}

f(data) // ?

對 Array Iterator 使用 for of loop，由於 entries 為 Array，可使用 array destructuring 分解出 i 與 x。

若以結果論，我們完全可以自行建立 entries()，回傳 Array of Array，用法與 Array Iterator 完全相同。

let data = ['Sam', 'Kevin', 'Jimmy']

let entries = a => a.map((x, i) => [i, x])

let f = a => {
  let result = []
  let elements = entries(a)

  for(let [i, x] of elements)
    result.push(`${i}: ${x}`)

  return result
}

f(data) // ?

對 Array of Array 使用 for of loop，由於 elements 為 Array，可使用 array destructuring 分解出 i 與 x。

目前為止 Array Iterator 與 Array of Array 功能看起來完全一樣。

For Loop

let data = ['Sam', 'Kevin', 'Jimmy']

let entries = a => a.map((x, i) => [i, x])

let f = a => {
  let result = []
  let elements = entries(a)

  for(let index = 0; index < elements.length; index++) {
    let [i, x] = elements[index]
    result.push(`${i}: ${x}`)
  }

  return result
};

f(data) // ?

若使用自行建立的 entries()，可使用 for loop 沒問題，因為 entries() 回傳為 Array。

let data = ['Sam', 'Kevin', 'Jimmy']

let f = a => {
  let result = []
  let elements = a.entries()

  for(let index = 0; index < elements.length; index++) {
    let [i, x] = elements[index]
    result.push (`${i}: ${x}`)
  }

  return result
}

f(data) // ?

若將 Array Iterator 使用 for loop，會發現無法執行。

因為 Array Iterator 並非 Array，也不像 Array-like Object 有 length property，因此無法使用 for loop。

let data = ['Sam', 'Kevin', 'Jimmy']

let f = a => {
  let result = []
  let elements = Array.from(a.entries())

  for(let index = 0; index < elements.length; index++) {
    let [i, x] = elements[index]
    result.push(`${i}: ${x}`)
  }

  return result
}

f(data) // ?

可使用 ES6 的 Array.from() 將 Array Iterator 轉成真正 Array，就可以使用 for loop 了。

map()

let data = ['Sam', 'Kevin', 'Jimmy']

let entries = a => a.map((x, i) => [i, x])

let f = a => entries(a).map(([i, x]) => `${i}: ${x}`)

f(data) // ?

若使用自行建立的 entries()，可使用 Array.prototype.map() 沒問題，因為 entries() 回傳為 Array。

let data = ['Sam', 'Kevin', 'Jimmy']

let f = a => a.entries().map(([i, x]) => `${i}: ${x}`)

f(data) // ?

若將 Array Iterator 使用 map()，會發現無法執行。

因為 Array Iterator 並非 Array，因此無法使用 Array.prototype 下的 method。

let data = ['Sam', 'Kevin', 'Jimmy']

let f = a => Array.from(a.entries()).map(([i, x]) => `${i}: ${x}`)

f(data) // ?

可使用 ES6 的 Array.from() 將 Array Iterator 轉成真正 Array，就可以使用 Array.prototype 下的 method 了。

Conclusion

  • 若以使用角度，Array.prototype.entries() 主要功能是回傳包含 index 與 element 的 Array Iterator，觀念上可視為 Array of Array 使用，但由於 Array Iterator 不是 Array，因此只能使用 for of loop，無法使用 for loop 與 Array.prototype 下的 method
  • 若要使用 for loop 與 Array.prototype 下的 method，必須使用 Array.from() 轉成真正 Array 才能使用
  • 除了 Array.entries() 外，Array.keys() 與 Array.values() 亦回傳 Array Iterator
  • entries() 回傳的 index 與 element 順序剛好跟 map() callback 相反，可以想成 entries() 回傳為 key / value 形式，故 index 在前面；但 map() callback 主要回傳 element，故 index 在後面
