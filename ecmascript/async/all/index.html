使用 Promise.all() 將多個 Promise 合併

2019-09-27

Promise 由於是 Asynchronous，產生速度有快有慢，實務上若必須等全部 Promise 都 Resolve 時一起顯示結果，可使用 Promise.all()，它會自動等最慢的 Promise ，最後合併成單一 Promise。

Version

macOS Mojave 10.14.6
VS Code 1.38.1
Quokka 1.0.253
ECMAScript 2015

All Fulfilled Promise

import { resolveAfter } from 'wink-fp';

let p0 = resolveAfter(100)('Sam');
let p1 = resolveAfter(200)('Jessie');
let p2 = resolveAfter(300)('Kevin');

Promise.all([p0, p1, p2])
  .then(console.log)
  .catch(console.error);

p0、p1、p2 三個 promise 分別在 100ms、200ms 與 300ms 後產生。

但 Promise.all() 會將等 3 個 promise 都 fulfilled 後才合併成單一 promise。

With Rejected Promise

import { resolveAfter, rejectAfter } from 'wink-fp';

let p0 = resolveAfter(100)('Sam');
let p1 = rejectAfter(200)('Jessie');
let p2 = resolveAfter(300)('Kevin');

Promise.all([p0, p1, p2])
  .then(console.log)
  .catch(console.error);

p1 為 rejected promise，Promise.all() 中只要有一個為 rejected promise，則最後結果為 rejected promise。

Conclusion

  • 實務上若要綜合數個 API 結果加以顯示，且每個 API 之間又沒有依賴關係，則可各自呼叫 API 回傳 promise，最後再使用 Promise.all() 整合在一起

Reference
