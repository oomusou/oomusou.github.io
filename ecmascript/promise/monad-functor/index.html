Promise 之 Functor 與 Monad 二元性

2020-03-08

Promise 是 ECMAScript 2015 所提出的新型態，專門處理 Asynchronous，雖然也可 await 之後繼續以 Imperative 處理，但 Promise 本身具有 Functor 與 Monad 的一些特性，可搭配 Pure Function 繼續處理。

Version

macOS Catalina 10.15.3
VS Code 1.42.1
Quokka 1.0.282
Ramda 0.27.0

Functor

let data = Promise.resolve(1)

let f = p => p
  .then(x => x * 2)
  .then(x => x + 1)

f(data) // ?

Promise 具有 Functor 特性，因此可將 then() 當 map() 用。

Monad

let data = Promise.resolve(1)

let f = p => p
  .then(x => Promise.resolve(x * 2))
  .then(x => Promise.resolve(x + 1))

f(data) // ?

Promise 也具有 monad 特性，也可將 then() 當 bind() 用，儘管回傳 promise，但最後也會 flatten() 成一層 promise。

Associative Law

import { pipe } from 'ramda'

let data = Promise.resolve(1)

let t = pipe(
  x => x * 2,
  x => x + 1
)

let f = p => p.then(t)

f(data) // ?

除了多次 then() 以外，promise 也支援 associative law，可將 pure function 先組合起來，一次傳給 then() 執行。

Conclusion

  • Promise 的 then() 兼具 map() 與 bind() 特性，使得 promise 看起來很像 functor 又很像 monad，但卻又不是
  • Promise 支援 associate law，因此可將 then() 的 pure function 先組合起來，再一次傳給 then()

Reference

Joel Thoms, Functional JavaScript - Functors, Monads, and Promises
