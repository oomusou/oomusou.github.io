Monad-tw 之 Infinity Page 解析

2020-10-07

Infinity Page 為 Monad-tw 另外一個首頁，負責顯示 Featured Post 與 Latest Post，與 Index Page 不同的是它會如同 Facebook 一樣沒有分頁，而是在捲動到底才載入下一頁，因此需要 CSS 實現 Fade-in。

Version

Monad-tw 0.0.1

Vue-infinite-loading

package.json

Gridsome 並沒有內建 infinite loading，必須另外安裝 vue-inifinite-loading。

Infinity Page

pages/Infinity.vue

<template>
  <navigation-link>
    <content-header :title="$static.metadata.siteName" :sub="$static.metadata.siteDescription" image="phoenix-han-Nqdh0G8rdCc-unsplash.jpg"></content-header>
    <div class="container mx-auto">
      <transition-group name="fade" class="flex flex-wrap my-4" tag="div">
        <FeaturedCard key="featured_post" v-if="$page.featured.totalCount>0" :records="$page.featured.edges"/>
        <CardItem v-for="{ node } of loadedPosts" :key="node.id" :record="node" />
      </transition-group>
      <ClientOnly>
        <infinite-loading @infinite="onInfinite" spinner="bubbles">
          <div slot="no-more"></div>
          <div slot="no-results"></div>
        </infinite-loading>
      </ClientOnly>
    </div>
  </navigation-link>
</template>

<script>
import { pipe, andThen as then, path } from 'ramda'
import { fetch } from 'gridsome'
import InfiniteLoading from 'vue-infinite-loading'
import NavigationLink from '@/layouts/NavigationLink'
import CardItem from '@/components/Content/CardItem'
import FeaturedCard from '@/components/Content/FeaturedCard'
import ContentHeader from '@/components/Partials/ContentHeader'

let onInfinite = function($state) {
  if (this.currentPage + 1 > this.$page.entries.pageInfo.totalPages) {
    $state.complete()
    return
  }

  let sideEffect = x => {
    if (x.edges.length === 0) {
      $state.complete()
      return
    }

    this.currentPage = x.pageInfo.currentPage
    this.loadedPosts.push(...x.edges)
    $state.loaded()
  }

  pipe(
    fetch,
    then(path(['data', 'entries'])),
    then(sideEffect)
  )(`/infinity/${this.currentPage + 1}`)
}

let created = function() {
  this.loadedPosts = this.$page.entries.edges
}

export default {
  metaInfo: {
    title: 'Hello World!'
  },
  components: {
    InfiniteLoading,
    NavigationLink,
    CardItem,
    FeaturedCard,
    ContentHeader
  },
  data: () => ({
    loadedPosts: [],
    currentPage: 1
  }),
  methods: {
    onInfinite
  },
  created,
}
</script>

<page-query>
query($page: Int) {
  featured: allBlog(limit: 4, filter: { featured: { eq: true } }, sortBy:"created") {
    totalCount
    edges {

  }

    totalCount
    pageInfo {
      totalPages

      node {
        id
        title
        image(width: 800)
        path

        featured
        humanTime: created(format: "DD MMM YYYY")
        datetime: created
        category {
          id
          title
          path
        }

          id

        }
      }
    }
  }
}
</page-query>

第 5 行

<transition-group name="fade" class="flex flex-wrap my-4" tag="div">

由於 Infinity page 主要在於捲動到底才載入下一個 page，因此會有 fade-in 需求，特別將 FeaturedCard 與 CardItem component 包在 Vue 內建的 TransitionGroup component 內，表示 FeaturedCard 與 CardItem 將一起 render，並設定 name 為 fade 方便 CSS 控制。

148 行

.fade-enter-active, .fade-leave-active {
  transition: ease opacity 0.3s;
}

.fade-enter, .fade-leave-to {
  opacity: 0;
}

以 fade 控制 fade-in。

22 行

import InfiniteLoading from 'vue-infinite-loading'

從 vue-infinite-loading 引用 InfiniteLoading component。

60 行

components: {
  InfiniteLoading,
  NavigationLink,
  CardItem,
  FeaturedCard,
  ContentHeader
},

使用 InfiniteLoading component。

67 行

data: () => ({
  loadedPosts: [],
  currentPage: 1
}),
  • loadedPosts：CardItem component 所依賴 v-for 的資料，將在捲動到底而陸續載入，不像 Index.vue 中的 CardItem component 是直接使用 $page.entries.edges 每一 page 的所有資料
  • currentPage：inifinite loading 看似沒有 page，骨子仍然有 page 概念，currentPage 則記載目前邏輯上的 page，將隨捲動到底而遞增

52 行

let created = function() {
  this.loadedPosts = this.$page.entries.edges
}

loadedPosts 一開始將由 $page.entries.edges 提供。

10 行

<infinite-loading @infinite="onInfinite" spinner="bubbles">
  • infinite event：當捲動到底時，InfiniteLoading component 會發出 infinite event，需提供 onInfinite() 載入下一批資料
  • spinner prop：指定載入資料時的動畫，可以設定 default、spiral、circles、bubbles 與 waveDots

28 行

let onInfinite = function($state) {
  if (this.currentPage + 1 > this.$page.entries.pageInfo.totalPages) {
    $state.complete()
    return
  }

  let sideEffect = x => {
    if (x.edges.length === 0) {
      $state.complete()
      return
    }

    this.currentPage = x.pageInfo.currentPage
    this.loadedPosts.push(...x.edges)
    $state.loaded()
  }

  pipe(
    fetch,
    then(path(['data', 'entries'])),
    then(sideEffect)
  )(`/infinity/${this.currentPage + 1}`)
}

onInfinite() 回應 infinite event，且提供 $state argument 控制 InfiniteLoading component。

if (this.currentPage + 1 > this.$page.entries.pageInfo.totalPages) {
  $state.complete()
  return
}

當 currentPage 超過 totalPage 時，對 InifiniteLoading 設定為 $state.complete()，表示已經全部載入，即使之後捲動到底，也不再發起 infinite event。

45 行

pipe(
  fetch,
  then(path(['data', 'entries'])),
  then(sideEffect)
)(`/infinity/${this.currentPage + 1}`)

若 currentPage 未超過 totalPage，表示需要載入下一批資料。

  • 使用 Gridsome 所提供的 fetch() 對 infinity page 取得指定 page 資料
  • fetch() 回傳為 Promise，因此可使用 Ramda 的 then() 加以 Function Pipeline
  • 使用 path() 以 Point-free 取得 data.entries
  • 最後以 sideEffect() 執行 side effect 部分

21 行

import { fetch } from 'gridsome'

使用 Gridsome 所提供的 fetch()，而非 ECMAScript 內建的 fetch()。

34 行

let sideEffect = x => {
  if (x.edges.length === 0) {
    $state.complete()
    return
  }

  this.currentPage = x.pageInfo.currentPage
  this.loadedPosts.push(...x.edges)
  $state.loaded()
}
  • x.edges 將只回傳指定 page 資料
  • 若 edges 沒有任何資料，則 $state.complete() 完成 loading，之後捲動到底也不會發起 infinite event
  • 使用 pageInfo.currentPage 更新 this.currentPage
  • 將下一批以 ...x.edges 展開以 push() 新增至 loadedPosts Array
  • 設定 $state.loaded()，表示下次捲動到底時，會發起 infinite event

105 行

entries: allBlog(perPage: 6, page: $page, sortBy:"created")

使 Index.vue 不同的是 perPage 由 24 改成 6，表示每次捲動到底將只載入 6 筆新 post。

第 9 行

<ClientOnly>
  <infinite-loading @infinite="onInfinite" spinner="bubbles">
    <div slot="no-more"></div>
    <div slot="no-results"></div>
  </infinite-loading>
</ClientOnly>

由於 InfiniteLoading 並不是特別為 JAMStack 設計，Gridsome 建議將此類 component 包在 Gridsome 提供的 <ClientOnly> 內確保正確執行。

Conclusion

  • Gridsome 雖然沒提供 infinite loading，但藉由 Vue-infinite-loading component，我們只要處理 Vue-infinite-loading 的 infinite event 就可達成 infinite loading
  • Vue-infinite-loading 並非為 JAMStack 設計，因此要包在 <ClientOnly> 內
  • 使用 Vue 的 TransitionGroup component 可使 fade-in 效果一起呈現
  • 對 TransitionGroup 設定 name property 可繼續使用 CSS 控制 fade-in
