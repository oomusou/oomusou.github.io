Monad-tw 之 Navbar Component 解析

2020-10-10

Navbar Component 為 NavagationLink Layout 中最主要 Component，其中包含 Desktop 與 Mobile 的 Navbar 與 Modal 顯示。

Version

Monad-tw 0.0.1

Navbar Component

Navbar 為 NavigationLink layout 內最主要 component。

components/Navbar/Navbar.vue

<template>
  <div class="h-16">
    <headroom :classes="{'initial': 'headroom bg-white dark:bg-black border-b dark:border-gray-900'}" @pin="onPin" @unpin="onUnPin">
      <navbar-desktop :theme="theme" :isShowSubNav="isNavbarPinned" @openSearchModal="onOpenSearchModal" v-on="$listeners"/>
      <navbar-mobile :theme="theme" @openSearchModal="onOpenSearchModal" @openNavbarModal="onOpenNavbarModal" v-on="$listeners"/>
    </headroom>
    <!-- search modal -->
    <modal :isShowModal="isShowSearchModal" @close="onCloseSearchModal">
      <search-modal/>
    </modal>

    <!-- hamburger modal -->
    <modal :isShowModal="isShowNavbarModal" @close="onCloseNavbarModal">
      <navbar-modal/>
    </modal>
  </div>
</template>

<script>
import { headroom } from 'vue-headroom'
import NavbarDesktop from '@/components/Navbar/NavbarDesktop'
import NavbarMobile from '@/components/Navbar/NavbarMobile'
import Modal from '@/components/Modal/Modal'
import NavbarModal from '@/components/Modal/NavbarMobileModal'
import SearchModal from '@/components/Modal/SearchModal'

let onUnPin = function() {
  this.isNavbarPinned = false
}

let onPin = function() {
  this.isNavbarPinned = true
}

let onOpenSearchModal = function() {
  this.isShowSearchModal = true
}

let onCloseSearchModal = function() {
  this.isShowSearchModal = false
}

let onOpenNavbarModal = function() {
  this.isShowNavbarModal = true
}

let onCloseNavbarModal = function() {
  this.isShowNavbarModal = false
}

let route_ = function() {
  this.isShowNavbarModal = false
  this.isShowSearchModal = false
}

export default {
  components: {
    NavbarDesktop,
    NavbarMobile,
    Modal,
    SearchModal,
    NavbarModal,
    headroom
  },
  props: {
    theme: { default: '' }
  },
  data: () => ({
    headerHeight: 100,
    isShowSearchModal: false,
    isShowNavbarModal: false,
    isNavbarPinned: true
  }),
  methods: {
    onUnPin,
    onPin,
    onOpenSearchModal,
    onCloseSearchModal,
    onOpenNavbarModal,
    onCloseNavbarModal
  },
  watch: {
    $route: route_
  } 
}
</script>

第 2 行

<div>
  <headroom>
    <navbar-desktop/>
    <navbar-mobile/>
  </headroom>
  <modal>
    <search-modal/>
  </modal>
  <modal>
    <navbar-modal/>
  </modal>
</div>

其內主要包含 4 個 component：

  • NavbarDesktop component：desktop 所使用 navbar
  • NavbarMobile component：mobile 所使用 navbar
  • SearchModal component：search 所使用 modal
  • NavbarModal component：mobile 所使用 modal

第 2 行

<div class="h-16">
  • 使用 h-16 設定 height

第 3 行

<headroom :classes="{'initial': 'headroom bg-white dark:bg-black border-b dark:border-gray-900'}" @pin="onPin" @unpin="onUnPin">
  <navbar-desktop/>
  <navbar-mobile/>
</headroom>

Headroom component 來自於 vue-headroom package，它使其內 component 只要向下捲動就會 unpin 隱藏，向上捲動就會 pin 顯示，因此特別適合用於 navbar：

  • classes prop：設定 Headroom 所使用 CSS class，其中 initial 為初始化使用
  • unpin event：當向下捲動 unpin 隱藏時觸發
  • pin event：當向上捲動 pin 顯示時觸發

在 WebStorm 會 headroom 引用錯誤，此為 WebStorm 誤判可忽略。

第 4 行

<navbar-desktop :theme="theme" :isShowSubNav="isNavbarPinned" @openSearchModal="onOpenSearchModal" v-on="$listeners"/>

NavbarDesktop component 為 desktop 所使用 navbar：

  • theme prop：傳入 CSS mode 是 dark 或 light mode
  • isShowSubNav prop：根據 navbar 是否 pin 顯示 subnav
  • openSearchModel event：當 user 按下 search button 時觸發

第 5 行

<navbar-mobile :theme="theme" @openSearchModal="onOpenSearchModal" @openNavbarModal="onOpenNavbarModal" v-on="$listeners"/>

NavbarDesktop component 為 mobile 所使用 navbar：

  • theme prop：傳入 CSS mode 是 dark 或 light mode
  • openSearchModel event：當 user 按下 search button 時觸發
  • openNavbarModal event：當 user 按下 hamberger button 時觸發，只有 mobile 時才顯示

第 8 行

<modal :isShowModal="isShowSearchModal" @close="onCloseSearchModal">
  <search-modal/>
</modal>

SearchModal component 為 search 部分，因為它以 modal 顯示，所以採用 slot 傳進 modal component。

  • isShowModal prop：控制 modal 是否顯示
  • close event：當按下 x button 時，會觸發 close event

13 行

<modal :isShowModal="isShowNavbarModal" @close="onCloseNavbarModal">
  <navbar-modal/>
</modal>

NavbarModal component 為 mobile 時，按下 hamberger button 後顯示部分，因為它以 modal 顯示，所以採用 slot 傳進 modal component。

  • isShowModal prop：控制 modal 是否顯示
  • close event：當按下 x button 時，會觸發 close event

57 行

data: () => ({
  headerHeight: 100,
  isShowSearchModal: false,
  isShowNavbarModal: false,
  isNavbarPinned: true
}),
  • isShowSearchModal state：是否顯示 SearchModal component，由 event 控制，以 prop 傳進 modal component
  • isShowNavbarModal state：控制是否顯示 NavbarModal component，由 event 控制，以 prop 傳進 modal component
  • isNavbarPinned state：是否顯示 ... 所展開的 SubNavigation component，由 pin 與 unpin event 控制，以 prop 傳進 NavbarDesktop component

27 行

let onUnPin = function() {
  this.isNavbarPinned = false
}

let onPin = function() {
  this.isNavbarPinned = true
}

當 向下捲動 時觸發 unpin event，此時 NavbarDesktop component 內的 SubNavigation component 將隱藏。

當 向上捲動 時觸發 pin event，此時將 NavbarDesktop component 內的 SubNavigation component 將顯示。

35 行

let onOpenSearchModal = function() {
  this.isShowSearchModal = true
}

let onCloseSearchModal = function() {
  this.isShowSearchModal = false
}
  • 當按下 Search button 時觸發 openSearchModal event，此時將顯示 SearchModal component
  • 當按下 x button 觸發時觸發 closeSearchModal event，此時將隱藏 SearchModal component

43 行

let onOpenNavbarModal = function() {
  this.isShowNavbarModal = true
}

let onCloseNavbarModal = function() {
  this.isShowNavbarModal = false
}

只有在 mobile 時使用：

  • 當按下 hamberger button 時觸發 openNavbarModal event，此時將顯示 NavbarModal component
  • 當按下 x button 觸發時觸發 closeNavbarModal event，此時將顯示 NavbarModal component

82 行

watch: {
  $route: route_
}

當網址列的 route 改變時觸發 route_()。

51 行

let route_ = function() {
  this.showNavbarModal = false
  this.showSearchModal = false
}

當網址列的 route 改變時，SearchModal 與 NavbarModal component 都隱藏不顯示。

Conclusion

  • Navbar component 包含了 NavbarDesktop 與 NavbarMobile 兩個重要 component，且控制其是否顯示的 state
