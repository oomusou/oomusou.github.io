Using absolute for Position

2021-12-29

When using top, bottom, left and right with absolute, it is placed relative to its closest positioned ancestor.

Version

TailwindCSS 3.0

Flow Layout

Red box is located in 3 layer <div>. Each <div> has its own padding.

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <script src="https://cdn.tailwindcss.com"></script>
  <title>TailwindCSS</title>
</head>
<body>
  <div class="ml-4 p-4">
    <div class="p-4">
      <div class="p-4">
        <div class="w-32 h-32 bg-red-600">box</div>
      </div>
    </div>
  </div>
</body>
</html>

There are 3 layer <div> outside the red box. Every <div> has p-4 padding.

Line 13

<div class="w-32 h-32 bg-red-600">box</div>
  • w-32 : set block width
  • h-32 : set block height
  • bg-red-600 : set block background color

Absolute with Position

The red box is floated as a new layer. The 3rd layer <div> is shrunk only for padding.

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <script src="https://cdn.tailwindcss.com"></script>
  <title>TailwindCSS</title>
</head>
<body>
  <div class="ml-4 p-4">
    <div class="p-4">
      <div class="p-4">
        <div class="absolute w-32 h-32 bg-red-600">box</div>
      </div>
    </div>
  </div>
</body>
</html>

Line 13

<div class="absolute w-32 h-32 bg-red-600">box</div>
  • absolute : As fixed, absolute makes the red box float as a new layer. Since it doesn’t occupy any space. the 3rd layer <div> is shrunk only for padding

Position at 3rd div

The left side of the red box is close to the 3rd layer <div>.

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <script src="https://cdn.tailwindcss.com"></script>
  <title>TailwindCSS</title>
</head>
<body>
  <div class="ml-4 p-4">
    <div class="p-4">
      <div class="relative p-4">
        <div class="absolute left-0 w-32 h-32 bg-red-600">box</div>
      </div>
    </div>
  </div>
</body>
</html>

Line 13

<div class="absolute left-0 w-32 h-32 bg-red-600">box</div>
  • absolute：use absolute position
  • left-0 : set left to 0. But we don’t know the position. It will be based on its closet positioned ancestor

If fixed is used, it will be based on window, not relative

Line 12

<div class="relative p-4">
  • relative：set the 3rd <div> as relative position for absolute

Position at body

The left side of the red box is close to <body>.

<!doctype html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <script src="https://cdn.tailwindcss.com"></script>

  <div class="ml- 4 p-4">
    <div class="p-4">
      <div class="p-4">
        <div class="absolute left-0 w-32 h-32 bg-red-600">box</div>
      </div>
    </div>
  </div>
</body>
</html>

Suppose no other relative on its closest positioned ancestor. absolute is positioned to <body>.

If no relative on an ancestor, absolute is just like fixed

Conclusion

  • If absolute is used without position, it is floated to a new layer. absolute is just like fixed
  • When using top, bottom, left and right with absolute, it is placed relative to its closest positioned ancestor
  • When using top, bottom, left, and right with absolute, if not relative on an ancestor, absolute is just like fixed
