如何呼叫有相依性的 API ?

2019-08-26

實務上有些 API 呼叫有順序性，如必須先呼叫 產品 API，再根據產品資料呼叫 價格 API，像這類有相依性的 API，由於 Axios 回傳的 Promise 是 Asynchronous，直覺很容易寫成巢狀，這又回到 Callback Hell 老路子，比較好的寫法是善用 Promise 特性，無論 API 相依幾層，都只有 then() 一層，或者使用 ECMAScript 2017 的 async await，以 Synchronous 風格實現 Asynchronous。

Version

macOS MoJave 10.14.6
Vue 2.5.17
Axios 0.18.0
ECMSScript 2017

Scenario

<template>
  <div>
    {{ product.title }} : {{ price }}
  </div>
</template>

<script>
import fetchProducts from './api/products';
import fetchPrice from './api/price';

let mounted = function() {
  fetchProducts()
    .then(res => {
      this.product = res.data[0];
      fetchPrice(this.product.id)
        .then(res => this.price = res.data.price);
    });
};

export default {
  name: 'app',
  data: () => ({
    product: {},
    price: 0,
  }),
  mounted
}
</script>

11 行

let mounted = function() {
  fetchProducts()
    .then(res => {
      this.product = res.data[0];
      fetchPrice(this.product.id)
        .then(res => this.price = res.data.price);
    });
};

先由 fetchProducts() 呼叫 產品 API，再將 產品 ID 傳入 fetchPrice() 呼叫 價格 API，由於asynchronous 特性，很容易寫成這類 callback hell 式的巢狀寫法，若相依的 API 更多層，則巢狀問題將更加嚴重，將導致程式碼的可讀性變差，將來會很難維護。

Promise

<template>
  <div>
    {{ product.title }} : {{ price }}
  </div>
</template>

<script>
import fetchProducts from './api/products';
import fetchPrice from './api/price';

let mounted = function() {
  fetchProducts()
    .then(res => this.product = res.data[0])
    .then(res => fetchPrice(res.id))
    .then(res => this.price = res.data.price);
    .catch(e => console.log(e))
    .finally(() => console.log('done'));
};

export default {
  name: 'app',
  data: () => ({
    product: {},
    price: 0,
  }),
  mounted
}
</script>

11 行

let mounted = function() {
  fetchProducts()
    .then(res => this.product = res.data[0])
    .then(res => fetchPrice(res.id))
    .then(res => this.price = res.data.price)
    .catch(e => console.log(e))
    .finally(() => console.log('done'));
};

既然 Axios 回傳為 Promise，就該善用 Promise 的 then()，除了將 res 資料寫入 Vue data 外，也順便回傳 Promise，因此可以繼續用 then() 呼叫下一個 API，如此不管相依 API 有幾層，都會只有 then() 一層，這才是 Promise 設計初衷。

Promise 也自帶 catch() 與 finally()，可將 error handling 寫在此。

Async Await

<template>
  <div>
    {{ product.title }} : {{ price }}
  </div>
</template>

<script>
import fetchProducts from './api/products';
import fetchPrice from './api/price';

let mounted = async function() {
  try {
    let product = await fetchProducts();
    this.product = product.data[0];
    let price = await fetchPrice(this.product.id);
    this.price = price.data.price;
  }
  catch(e) {
    console.log(e);
  }
  finally {
    console.log('done');
  }
};

export default {
  name: 'app',
  data: () => ({
    product: {},
    price: 0,
  }),
  mounted
}
</script>

13 行

let product = await fetchProducts();
this.product = product.data[0];
let price = await fetchPrice(this.product.id);
this.price = price.data.price;

ES2017 迎來了 async await，讓 asynchronous 寫法更像 synchronous。

async 與 await 必須成對出現，若 function 使用了 await，則在 function 前需加上 async 宣告成 async function。

await 在語義上是 blocking，相當於等 promise 處理完後，才會繼續執行下一行，但本質仍是 Promise 的 non-blocking，是 then() 的 syntatic sugar。

12 行

try {
  let product = await fetchProducts();
  ...
}
catch(e) {
  console.log(e);
}
finally {
  console.log('done');
}

至於 error handling，可繼續使用 imperative 的 try...catch...finally，

Conclusion

  • 既然 Axios 回傳為 Promise，就必須善用 Promise 特性，才可避免 callback hell
  • then() 為 FP 的 pipeline 風格，then() 內的 function 建議使用 pure function
  • async await 為 imperative 風格，讓 non-blocking 的非同步看起來像是 blocking 的同步一行一行執行，但別忘其本質仍是 Promise
  • Promise 自帶 catch() 與 finally()，可用來處理 error handling；若使用 async await，可繼續使用try...catch...finally
