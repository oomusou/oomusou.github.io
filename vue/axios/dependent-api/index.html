如何呼叫有相依性的 API ?

2020-03-22

實務上有些 API 呼叫有順序性，由於 Axios 回傳的 Promise 是 Asynchronous，直覺很容易寫成巢狀 then()，這又回到 Callback Hell 老路子，比較好的寫法是善用 Promise 特性，無論 API 相依幾層，都只有 then() 一層，或者使用 ECMAScript 2017 的 async await。

Version

macOS Catelina 10.15.3
WebStorm 2019.3.4
Vue CLI 4.2.3
Vue 2.6.10
ECMSScript 2017

Callback Hell

<template>
  <div>
    {{ product.title }} : {{ price }}
  </div>
</template>

<script>
import { compose, find, propEq } from 'ramda'

let fetchProducts = async _ => ({
  data: [
    { id:1, title:"FP in JavaScript" },
    { id:2, title:"RxJS in Action" },
    { id:3, title:"Speaking JavaScript" },
  ]
})

let fetchPrice = async id => {
  let data = [
    { id:1, price:100 },
    { id:2, price:200 },
    { id:3, price:300 }
  ]

  let { price } = compose(find, propEq("id"))(id)(data)

  return {
    data:{
      price
    }
  }
}

let mounted = function() {
  fetchProducts()
    .then(x => {
      this.product = x.data[0]
      fetchPrice(this.product.id)
        .then(x => this.price = x.data.price)
    })
}

export default {
  name:"App",
  data:() => ({
    product:{},
    price:0,
  }),
  mounted
}
</script>

fetchPrice() 模擬 Axios 回傳 promise。

34 行

let mounted = function() {
  fetchProducts()
    .then(x => {
      this.product = x.data[0]
      fetchPrice(this.product.id)
        .then(x => this.price = x.data.price)
    })
}

先由 fetchProducts() 呼叫 產品 API，再將 產品 ID 傳入 fetchPrice() 呼叫 價格 API，由於 asynchronous 特性，很容易寫成這類 callback hell 式的巢狀寫法，若相依的 API 更多層，則巢狀問題將更加嚴重，將導致程式碼的可讀性變差，將來會很難維護。

由於 promise 的 then() 亦使用 callback 取得 promise 內部資料，因此很多人以為 promise 只是 callback 的另外一種形式，因此只要相依的 API 很直覺就寫成巢狀 then()。

Async Await

let mounted = async function() {
  try {
    let product = await fetchProducts()
    this.product = product.data[0]
    let price = await fetchPrice(this.product.id)
    this.price = price.data.price
  }
  catch(e) {
    console.error(e)
  }
  finally {
    console.log('done')
  }
}

比要好的方式是改用 ES2017 的 async await，除了讓原本的 promise 的 asynchronous 更像 synchronous 外，無論 API 相依幾層都不會出現 callback hell，也可繼續使用 imperative 習慣的 try...catch...finally。

async await 唯一不好的是容易將 處理 data 與 side effect 通通寫在一起，這算 imperative 原罪。

Promise Chain

let mounted = function() {
  fetchProducts()
    .then(x => this.product = x.data[0])
    .then(x => fetchPrice(x.id))
    .then(x => this.price = x.data.price)
    .catch(console.log)
    .finally(() => console.log('done'))
}

其實 promise 有 monad 的 bind() 特性：在 then() 中回傳 promise，不會是 promise 的 promise，永遠只有一層 promise。

既然如此，若有相依 API 只要在下個 then() 去呼叫即可，而不必使用巢狀的 then()，如此無論相依 API 有幾層，都只有一層 then() 而已，這才是原本 promise 設計。

不過以上寫法 處理 data 與 side effect 還是通通寫在一起，並不是最好寫法。

let mounted = function() {
  fetchProducts()
    .then(x => x.data[0])
    .then(x => this.product = x)
    .then(x => fetchPrice(x.id))
    .then(x => x.data.price)
    .then(x => this.price = x)
    .catch(console.error)
    .finally(() => console.log('done'));
}

Promise 亦有 functor 的 map() 特性，因此可在 then() 中以 pure function 先處理 promise 內部資料，如此 pure function 可與 side effect 分離。

Point-free

let mounted = function() {
  fetchProducts()
    .then(pipe(prop('data'), nth(0)))
    .then(x => this.product = x)
    .then(pipe(prop('id'), fetchPrice))
    .then(path(['data', 'price']))
    .then(x => this.price = x)
    .catch(console.error)
    .finally(() => console.log('done'));
}

Pure function 部分可進一步使用 Ramda 的 higher order function 加以 point-free。

Associative Law

let genProduct = pipe(prop('data'), nth(0))

let genPrice = pipe(
  pipe(prop('id'), fetchPrice),
  andThen(path(['data', 'price']))
)

let mounted = function() {
  fetchProducts()
    .then(genProduct)
    .then(x => this.product = x)
    .then(genPrice)
    .then(x => this.price = x)
    .catch(console.error)
    .finally(() => console.log('done'));
}

Promise 亦支援 associative law，因此可將 then() 的 pure function 先 pipe() 組合起來，一次傳給 then() 即可，如此 pure function 與 side effect 的分隔將更為明顯。

IIFE

let genProduct = pipe(prop('data'), nth(0))

let genPrice = pipe(
  pipe(prop('id'), fetchPrice),
  andThen(path(['data', 'price']))
)

let mounted = function() {
  pipe(
    fetchProducts,
    andThen(genProduct),
    andThen(x => this.product = x),
    andThen(genPrice),
    andThen(x => this.price = x),
    otherwise(console.error),
    eventually(() => console.log('done'))
  )()
}

Promise 的 then() 其實有 pipeline 特性，只可惜以 method chaining 設計，也可改用 Ramda 的 andThen()，如此也可以 pipe() 處理 promise chain，最後透過 IIFE 執行。

Conclusion

  • 既然 Axios 回傳為 promise，就必須善用 promise 特性，讓相依 API 在下一個 then() 執行即可，而不必使用巢狀 then()，才可避免 callback hell
  • async await 為 imperative 風格，讓 non-blocking 的非同步看起來像是 blocking 的同步一行一行執行，但別忘其本質仍是 promise
  • 使用 then() 時應避免 處理 data 的 pure function 與 side effect 混在一起，可使用 associative law 將 pure function 先透過 pipe() 組合起來
  • 也可使用 Ramda 的 andThen() 以 pipeline 方式處理 promise chain
