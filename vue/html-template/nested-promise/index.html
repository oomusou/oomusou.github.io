如何在 Vue 顯示 Nested Promise ?

2020-04-27

若 API 遵循 RESTful 精神設計，常會第一個 API 僅回傳 id，若要取得其值就要呼叫 第二個 API，這種實務上常見的設計在 Vue 該如何顯示呢 ?

Version

macOS Catalina 10.15.4
WebStorm 2020.1
Vue 2.6.11

Double Side Effect

Promise Chain

<template>
  <div>
    <ul v-for="(x, index) in books" :key="index">
      <li>{{ x.title }} / {{ x.price }} / {{ categories[index] }}</li>
    </ul>
  </div>
</template>

<script>
let categories = [
  { id: 1, category: 'FP' },
  { id: 2, category: 'FRP' },
  { id: 3, category: 'JS' }
]

let fetchCategory = async id =>
  categories.find(x => x.id === id).category

let fetchBooks = async () => [
  { title: 'FP in JavaScript',
    price: 100,
    categoryId: 1 },
  { title: 'RxJS in Action',
    price: 200,
    categoryId: 2 },
  { title: 'Speaking JavaScript',
    price: 300,
    categoryId: 3 }
]

let mounted = function() {
  fetchBooks()
    .then(x => {
      let ps = Promise.all(x.map(y => fetchCategory(y.categoryId)))
      return Promise.all([x, ps])
    })
    .then(([x, y]) => {
      this.books = x
      this.categories = y
    })
}

export default {
  name:'App',
  data: () => ({
    books: [],
    categories: []
  }),
  mounted,
}
</script>

19 行

let fetchBooks = async () => [
  { title: 'FP in JavaScript',
    price: 100,
    categoryId: 1 },
  { title: 'RxJS in Action',
    price: 200,
    categoryId: 2 },
  { title: 'Speaking JavaScript',
    price: 300,
    categoryId: 3 }
]

模擬 Axios 從 API 回傳 Promise，故為 async function，由於遵守 RESTFul 設計，因此僅回傳 catetoryId，若要取得 category 必須再呼叫另外一個 API。

16 行

let fetchCategory = async id =>
  categories.find(x => x.id === id).category

模擬另外一個 API 由 categoryId 取得 category。

31 行

let mounted = function() {
  fetchBooks()
    .then(x => {
      let ps = Promise.all(x.map(y => fetchCategory(y.categoryId)))
      return Promise.all([x, ps])
    })

x.map(y => fetchCategory(y.categoryId)) 回傳為 Promise array，故必須使用 Promise.all() 取得 array 內部值，但會重新包成新 Promise。

再使用 Promise.all() 將 x 與 ps 一起傳到下一個 then() 處理。

使用 Promise.all() 將 asynchronous 與 synchronous 一起傳到下一個 then() 為 promise chain 慣用手法

37 行

.then(([x, y]) => {
  this.books = x
  this.categories = y
})

在最後一個 then() 專心處理 side effect，這種方式特點是使用兩份 data 各自處理 API。

第 3 行

<ul v-for="(x, index) in books" :key="index">
  <li>{{ x.title }} / {{ x.price }} / {{ categories[index] }}</li>
</ul>

目前有 books 與 categories 兩份 data，但 v-for 只能同時處理一份，故 categories 必須靠 index 存取。

這種處理方式的特點是每個 API 在 Vue 都有各自 data，v-for 只使用其中一份，另外一份由 index 存取，優點是 API 容易處理，缺點 HTML template 必須技巧性處理兩份 data

前一種方式使用兩份 data 在 HTML template 處理比較 tricky，可使用 computed 加以改善。

31 行

let allBooks = function() {
  return this.books.map((x, i) => ({
    ...x,
    category: this.categories[i]
  }))
}

在 allBooks computed 內將 books 與 categories 兩份 data 重新 map() 成邏輯上的一份。

第 3 行

<ul v-for="(x, index) in allBooks" :key="index">
  <li>{{ x.title }} / {{ x.price }} / {{ x.category }}</li>
</ul>

由於 allBooks computed 已經整理成一份，因此 v-for 可簡單使用。

這種方式結合了 2 份 data 與單一 computed 優點，無論在 API 或 HTML template 都很容易處理

Ramda

<template>
  <div>
    <ul v-for="(x, index) in allBooks" :key="index">
      <li>{{ x.title }} / {{ x.price }} / {{ x.category }}</li>
    </ul>
  </div>
</template>

<script>
import { pipe, andThen, map, prop } from 'ramda'
import { all } from 'wink-fp'

let categories = [
  { id: 1, category: 'FP' },
  { id: 2, category: 'FRP' },
  { id: 3, category: 'JS' }
]

let fetchCategory = async id =>
  categories.find(x => x.id === id).category

let allBooks = function() {
  return this.books.map((x, i) => ({
    ...x,
    category: this.categories[i]
  }))
}

let fetchBooks = async () => [
  { title: 'FP in JavaScript',
    price: 100,
    categoryId: 1 },
  { title: 'RxJS in Action',
    price: 200,
    categoryId: 2 },
  { title: 'Speaking JavaScript',
    price: 300,
    categoryId: 3 }
]

let mounted = function() {
  pipe(
    fetchBooks,
    andThen(x => this.books = x),
    andThen(map(pipe(prop('categoryId'), fetchCategory))),
    andThen(all),
    andThen(x => this.categories = x)
  )()
}

export default {
  name:'App',
  data: () => ({
    books: [],
    categories: []
  }),
  computed: {
    allBooks
  },
  mounted,
}
</script>

呼叫 fetchBooks() API，並即時寫入 books data。

在 map() 內呼叫 fetchCategory() API，由於回傳 Promise array，因此需再傳入 Promise.all() 處理，all() 為 Promise.all() 的 funciton 版本，最後寫入 categories data。

雖然沒將 side effect 統一寫在最後一個 andThen()，但最少沒在 pure function 內處理，算可接受方式

Conclusion

  • 在 Vue 這個以 side effect 為設計的 library，或許兩份 data + computed 是最好的寫法，兼具簡易性與可讀性

  • 其實就算只用一份 data，也只是將 computed 要做的直接寫在 mounted() 內，想通了也沒很難

  • 至於使用 promise chain 或 async await 屬於個人 taste，一般而言 async await 寫法可讀性較高，但容易寫成 imperative 與 side effect，而 promise chain 較易自然使用 pure function，進而重構成 function pipeline，這也是個人較喜歡 promise chain 主因
