如何以 RxJS 顯示 Nested Observable ?

2020-05-24

若 API 遵循 RESTful 精神設計，常會第一個 API 僅回傳 id，若要取得其值就要呼叫第二個 API，也就是所謂 N + 1 Query，RxJS 對這種常見需求有很漂亮寫法。

Version

macOS Catalina 10.15.4
WebStorm 2020.1.1
Vue 2.6.11
Vue-rx 6.2
RxJS 6.5.5

Browser

FP、FRP 與 JS 並不是從單一 API 取得，而是呼叫另外一個 API。

Double Side Effect

<template>
  <div>
    <ul v-for="(x, i) in books" :key="i">
      <li>{{ x.title }} / {{ x.price }} / {{ categories[i] }}</li>
    </ul>
  </div>
</template>

<script>
import { from, of } from 'rxjs'
import { concatMap, share, toArray } from 'rxjs/operators'

let bookCategories = [
  { id: 1, category: 'FP' },
  { id: 2, category: 'FRP' },
  { id: 3, category: 'JS' }
]

let fetchCategory$ = id =>
  of(bookCategories.find(x => x.id === id).category)

let fetchBooks$ = () => from([
  { title: 'FP in JavaScript', price: 100, categoryId: 1 },
  { title: 'RxJS in Action', price: 200, categoryId: 2 },
  { title: 'Speaking JavaScript', price: 300, categoryId: 3 }
])

let subscriptions = () => {
  let books$ = fetchBooks$().pipe(share())

  let books = books$.pipe(toArray())

  let categories = books$.pipe(
    concatMap(x => fetchCategory$(x.categoryId)),
    toArray()
  )

  return { books, categories }
}

export default {
  name:'App',
  subscriptions
}
</script>

22 行

let fetchBooks$ = () => from([
  { title: 'FP in JavaScript', price: 100, categoryId: 1 },
  { title: 'RxJS in Action', price: 200, categoryId: 2 },
  { title: 'Speaking JavaScript', price: 300, categoryId: 3 }
])

模擬 RxJS 的 ajax() 從 API 回傳 Observable，由於遵守 RESTFul 設計，因此僅回傳 catetoryId，若要取得 category 必須再呼叫另外一個 API。

from() 可直接將 Array 轉成 Observable

19 行

let fetchCategory$ = id =>
  of(bookCategories.find(x => x.id === id).category)

模擬另外一個 API 由 categoryId 取得 category。

of() 可直接將單一 primitive 轉成 Observable

29 行

let books$ = fetchBooks$().pipe(share())

由 fetchBooks$() 取得 Observable，因為將以 books$ 產生兩個 side effect，為避免出現兩個 API request，因此加上 share() 共用 Observable。

31 行

let books = books$.pipe(toArray())

目前 Vue 的 HTML template 僅支援 Observable 內為 primitive，並不支援 Observable 內為 Array，因此必須使用 toArray() 由 Observable 轉成 array 後，才能在 HTML template 中使用 v-for。

33 行

let categories = books$.pipe(
  concatMap(x => fetchCategory$(x.categoryId)),
  toArray()
)

由 books$() 建立 categories，因為 books$ 本身是 Observable，而 fetchCategory$() 亦回傳 Observable，若使用單純的 map() 勢必造 Observable of Observable。

Observable 並不像 Promise 的 then() 天生具有 chain() 或 flatMap() 特性，必須選擇適當 operator 才能攤平 Observable。

對於這種 nested API 呼叫有兩個選擇： concatMap() 或 flatMap() (亦稱 mergeMap())：

  • concatMap()：合併後順序與 books$ 相同
  • flatMap()：合併後順序與 fetchCategory$ 回傳相同

一般來說，對於 Nested Observable 我們會希望維持第一個 Observable 原本順序，因此使用 concatMap()。

concatMap() 結果與第一個 Observable 的 1、3、5 相同，不受 Observable 出現時間點影響。

mergeMap() 會受 Observable 出現時間點影響，若第二個 API 因網路速度回傳順序不如預期，則會改變最後回傳 Observable 結果。

34 行

return { books, categories }

最後回傳 books 與 categories 兩個 side effect。

28 行

let subscriptions = () => {
}

由於 Observable 都沒使用到 this 存取 data，可安心使用 arrow function。

Single Side Effect

<template>
  <div>
    <ul v-for="(x, i) in books" :key="i">
      <li>{{ x.title }} / {{ x.price }} / {{ x.category }}</li>
    </ul>
  </div>
</template>

<script>
import { from, of } from 'rxjs'
import { concatMap, map, toArray } from 'rxjs/operators'

let bookCategories = [
  { id: 1, category: 'FP' },
  { id: 2, category: 'FRP' },
  { id: 3, category: 'JS' }
]

let fetchCategory$ = id =>
  of(bookCategories.find(x => x.id === id).category)

let fetchBooks$ = () => from([
  { title: 'FP in JavaScript', price: 100, categoryId: 1 },
  { title: 'RxJS in Action', price: 200, categoryId: 2 },
  { title: 'Speaking JavaScript', price: 300, categoryId: 3 }
])

let subscriptions = () => {
  let books = fetchBooks$().pipe(
    concatMap(x => fetchCategory$(x.categoryId).pipe(
      map(y => ({ ...x, category: y }))
    )),
    toArray()
  )

  return { books }
}

export default {
  name:'App',
  subscriptions
}
</script>

30 行

concatMap(x => fetchCategory$(x.categoryId).pipe(
  map(y => ({ ...x, category: y }))
)),

Promise Chain 遇到傳遞 Promise 問題在 RxJS 不會出現，因為 RxJS 需使用 Nested Pipe 完成，剛好很巧妙地讀取到 outer variable。

concatMap() 在 RxJS 5 時有第二個 argument 可以傳入 function 直接 map()，但 RxJS 6 已經拿掉，官方建議改用 inner map 實現，因此出現了 Nested Pipe。

Conclusion

  • Observable 並不像 Promise 的 then() 天生具有 flatten Promise 特性，必須明確使用正確的 operator
  • 對於有相依性的 API Request，該使用 mergeMap() 或 concatMap() 常讓初學者卻步，若第一個 API 回傳只有單一值，對第二個 API 使用 mergeMap() 或 concatMap() 皆可
  • 若第一個 API 回傳為 Array，對第二個 API 就要使用 concatMap()，如此才會與第一個 API 結果相對應，若使用 mergeMap() 會因為網路速度快慢與回傳順序導致結果受影響
