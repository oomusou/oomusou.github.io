如何以 RxJS 呼叫有相依性的 API ?

2020-05-23

由於 RESTful API 特性，實務上常會遇到無法單一 API 就能拿到結果，必須連續呼叫多次 API，本文分別以 Async Await、Promise Chain、Ramda 與 RxJS 示範，可比較其背後不同 Mental Model 。

Version

macOS Catalina 10.15.4
VS Code 1.45.0
WebStorm 2020.1.1
Vue 2.6.11
Vue-rx 6.2
RxJS 6.5.5

Browser

顯示 Bret 的所有 post。

API

https://jsonplaceholder.typicode.com/posts?userId= 能顯示所有 post，但可惜只接受 user id，因此必須先呼叫 https://jsonplaceholder.typicode.com/users?username= 取得 user id。

Async Await

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios'

let mounted = async function() {
  let result = await axios.get('https://jsonplaceholder.typicode.com/users?username=Bret')
  let user = result.data[0]
  result = await axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${user.id}`)
  this.posts = result.data
}

export default {
  name:'App',
  data: () => ({
    posts: []
  }),
  mounted
}
</script>

14 行

let result = await axios.get('https://jsonplaceholder.typicode.com/users?username=Bret')
  let user = result.data[0]
  result = await axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${user.id}`)
  this.posts = result.data

透過 await 由 asynchronous API 先取得 synchronous 結果存進 result，由於 Axios 將結果放在 data property 下，且是 array 第一個，因此由 result.data[0] 取得該 user。

再次透過 await 從 asynchronous API 取得 synchronous 結果存進 result，由於 Axios 將結果放在 data property 下，因此將 result.data 寫入 titles side effect。

我們可發現 async await 寫法雖然直覺，但必須使用很多不必要的 intermediate variable 作為銜接，也常因為要找可讀性高的 variable 名稱而想破頭

Promise Chain

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios'

let mounted = function() {
  axios.get('https://jsonplaceholder.typicode.com/users?username=Bret')
    .then(x => x.data)
    .then(x => x[0])
    .then(x => axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`))
    .then(x => x.data)
    .then(x => this.posts = x)
}

export default {
  name:'App',
  data: () => ({
    posts: []
  }),
  mounted
}
</script>

14 行

axios.get('https://jsonplaceholder.typicode.com/users?username=Bret')
  .then(x => x.data)
  .then(x => x[0])

Promise 的另一種寫法是使用 Promise Chain，讓 x 處在 synchronous 的 context 下，然後回傳 Promise 繼續回到 asynchronous。

習慣會在 then() 下使用 arrow function，一次只處理一件事情。

首先在 then() 中從 Axios 的 data property 取得資料，下一個 then() 在取得 array 中第一個 user，維持一個 then() 只處理一件事情原則。

17 行

.then(x => axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`))
.then(x => x.data)
.then(x => this.posts = x)

在 then() 的 synchronous contxt 呼叫第二個 API，回傳為 Promise，由於 then() 又會回傳 Promise，理論上應該是 Promise of Promise。

但 then() 具有 chain() 或 flatMap() 特性，會將 Nested Promise 攤平成一層 Promise，因此下一個 then() 可繼續由 data property 取得資料，並在最後一個 then() 寫入 titles side effect，符合 FP 習慣。

由於 Promise 的 then() 具有 chain() 特性，因此才不會如 callback hell 般寫出 Nested then()，這也是 Promise 設計的精妙之處

Ramda

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios'
import { pipe, andThen, prop, nth } from 'ramda'

let mounted = function() {
  pipe(
    axios.get,
    andThen(prop('data')),
    andThen(nth(0)),
    andThen(x => axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
    andThen(prop('data')),
    andThen(x => this.posts = x)
  )('https://jsonplaceholder.typicode.com/users?username=Bret')
}

export default {
  name:'App',
  data: () => ({
    posts: []
  }),
  mounted
}
</script>

15 行

pipe(
  axios.get,
  andThen(prop('data')),
  andThen(nth(0)),
  andThen(x => axios.get(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
  andThen(prop('data')),
  andThen(x => this.posts = x)
)('https://jsonplaceholder.typicode.com/users?username=Bret')

Ramda 提供 pipe() 與 andThen() 執行 asynchronous，但本質仍是 Promise Chain，只是以 function pipeline 形式而非 method chaining，另外 callback 部分則使用 Ramda 的 function 盡量 point-free。

RxJS

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { map, flatMap, pluck } from 'rxjs/operators'

let mounted = function() {
  ajax('https://jsonplaceholder.typicode.com/users?username=Bret').pipe(
    pluck('response'),
    map(x => x[0]),
    flatMap(x => ajax(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
    pluck('response'),
  ).subscribe(x => this.posts = x)
}

export default {
  name:'App',
  data: () => ({
    posts: []
  }),
  mounted
}
</script>

15 行

ajax('https://jsonplaceholder.typicode.com/users?username=Bret').pipe(
  pluck('response'),
  map(x => x[0]),

由 RxJS 內建的 ajax() 取代 Axios 讀取 API，與 axios.get() 不同的是 axios.get() 回傳為 Promise，而 ajax() 回傳為 Observable，方便後續 RxJS 使用 operator。

RxJS 與 Axios 另一點不同是 Axios 將回傳資料藏在 data property，而 RxJS 將資料藏在 response proeprty，因此先使用 pluck('response') 以 point-free 風格取出資料為 array，回傳仍是 Observable。

但我們要的是 Observable 中 array 的第一個資料而已，因此使用 map() 改變 Observable 內部資料，這與 Maybe 使用 map()，Promise 使用 then() 改變內部資料習慣一樣。

雖然口語是說改變 Observable 內部資料，但因為 immutable 原則，事實上是回傳一個全新的 Observable，而非改變原本 Observable，只是巨觀看起來 Observable 內部資料變了

18 行

flatMap(x => ajax(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
pluck('response'),

接著呼叫另一個 API，由於 ajax() 又是回傳 Observable，因此預期將會是 Observable of Observable。

Observable 並不像 Promise 的 then() 會自動攤平 Promise，必須明確使用 flatMap() 使 Observable 攤平。

既然 Observable 已攤平，就可繼續使用 pluck() 取出 response property 內部 synchronous 資料。

flatMap()
將兩層 Observable 攤平成一層 Observable

flatMap() 與 mergeMap() 在表現上完全相同。

20 行

subscribe(x => this.posts = x)

最後以 subscribe() 處理 posts side effect。

可發現 Observable 與 Promise 相比，Promise 的 then() 同時具有多重角色，兼具 map()、flatMap() 與 subscribe() 功能，但 RxJS 則必須明確使用正確 operator，這也符合 FP 習慣

Point-free

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { map, flatMap, pluck } from 'rxjs/operators'
import { head } from 'ramda'

let mounted = function() {
  ajax('https://jsonplaceholder.typicode.com/users?username=Bret').pipe(
    pluck('response'),
    map(head),
    flatMap(x => ajax(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
    pluck('response'),
  ).subscribe(x => this.posts = x)
}

export default {
  name:'App',
  data: () => ({
    posts: []
  }),
  mounted
}
</script>

17 行

map(head),

可將 x => x[0] 以 Ramda 的 head() 取代，使 map() 能 point-free。

RxJS operator 的 callback 為 synchronous context，可安心使用 Ramda 或 Sanctuary 的 function 使其 point-free

Subscription

<template>
  <div>
    <h2>Posts</h2>
    <ul v-for="(x, i) in posts$" :key="i">
      <li>{{ x.title }}</li>
    </ul>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { map, flatMap, pluck, toArray } from 'rxjs/operators'
import { head } from 'ramda'

let subscriptions = () => {
  let posts$ = ajax('https://jsonplaceholder.typicode.com/users?username=Bret').pipe(
    pluck('response'),
    map(head),
    flatMap(x => ajax(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
    pluck('response'),
  )

  return { posts$ }
}

export default {
  name:'App',
  subscriptions
}
</script>

15 行

let subscriptions = () => {
  let posts$ = ajax('https://jsonplaceholder.typicode.com/users?username=Bret').pipe(
    pluck('response'),
    map(head),
    flatMap(x => ajax(`https://jsonplaceholder.typicode.com/posts?userId=${x.id}`)),
    pluck('response'),
  )

  return { posts$ }
}

也可使用 Vue-rx 的 subscriptions，如此就不用使用 data side effect，可直接使用 arrow function。

Conclusion

  • Promise Chain 的 then() 兼具 map()、flatMap() 與 subscribe() 特性，因此用起來比較簡單，但缺點是只有一個 then() 能用，而不像 RxJS 有 100 多個 operator，可在各式各樣情境靈活使用
  • Promise Chain 雖然已經有 FP 味道，但卻使用了 method chaining
  • Ramda 雖使用了 function pipeline，但因其 synchronous 本質，所以每行都必須使用 andThen() 才支援 Promise，整體略顯累贅
  • RxJS 寫起來則最為精簡，且提供豐富的 operator 可用
