使用 switchMap() 使每次 Click 都對應一個 Observable

2019-10-07

DOM Event 為 Observable，而 API 回傳資料亦為 Observable，若希望每次 DOM Event 都會回傳 Observable，則可使用 RxJS 的 switchMap()。

Version

macOS Mojave 10.14.6
WebStorm 2019.2.3
Node 12.11.0
Yarn 1.19.0
Vue CLI 3.11.0
Vue 2.6.10
RxJS 6.5.3
Vue-rx 6.2.0

Browser

按下 + 會顯示 title 為 FP in JavaScript。

Data

{
  "book": { "title": "FP in JavaScript", "price": 100 }
}

API 回傳為以上 object。

API

api/book.js

import axios from 'axios';
import { from } from 'rxjs';

const API = process.env.VUE_APP_API;
const URI = process.env.VUE_APP_BOOKS;

export let fetchBook$ = () => from(axios.get(`${API}${URI}`));

fetchBook$() 回傳為 observable。

switchMap()

<template>
  <div>
    <button v-stream:click="{ subject: click$ }">+</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { fetchBook$ } from '@/api/books';
import { pluck, switchMap } from 'rxjs/operators';

let data$ = fetchBook$().pipe(
  pluck('data', 'book'),
);

let subscriptions = function() {
  let title$ = this.click$.pipe(
    switchMap(() => data$),
    pluck('title')
  );

  return {
    title$
  }
};

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

12 行

let data$ = fetchBook$().pipe(
  pluck('data', 'book'),
);

fetchBook$() 回傳為 observable，使用 pluck() 擷取 data.book 的資料。

16 行

let subscriptions = function() {
  let title$ = this.click$.pipe(
    switchMap(() => data$),
    pluck('title')
  );

  return {
    title$
  }
};

Button 的 click event 將對應到 click$ subject，我們希望 button 每按一下，就讀取 API 一次，也就是每 click 一次就回傳一個 observable，這相當於 switchMap()。

switchMap()
一個 observable 的每一個 value 都會對應到另外一個 observable

由於 data$ 本質為 object，我們希望讀取其 title property 即可，因此將 pluck() 也 pipe() 起來。

Point-free

<template>
  <div>
    <button v-stream:click="{ subject: click$ }">+</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { fetchBook$ } from '@/api/books';
import { pluck, switchMap } from 'rxjs/operators';
import { always } from 'ramda';

let data$ = fetchBook$().pipe(
  pluck('data', 'book'),
);

let subscriptions = function() {
  let title$ = this.click$.pipe(
    switchMap(always(data$)),
    pluck('title')
  );

  return {
    title$
  }
};

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

18 行

let title$ = this.click$.pipe(
  switchMap(always(data$)),
  pluck('title')
);

switchMap() 的 project 可使用 Ramda 的 always() 使其 point-free。

Conclusion

  • 若希望每個 DOM event 都回傳 observable，則 switchMap() 是不錯的選擇
