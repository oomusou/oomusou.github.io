使用 catchError() 處理 Rejected Promise

2020-06-10

當使用 RxJS 處理 API 回傳資料時，因為是 Asynchronous，所以並不保證一定都是 Fulfilled Promise，RxJS 該如何處理 Rejected Promise 呢 ?

Version

macOS Catalina 10.15.4
WebStorm 2020.1
Vue 2.6.11
RxJS 6.5.5

Fulfilled Promise

若回傳為 fulfilled Promise，可正確顯示 title 為 FP in JavaScript。

<template>
  <div>
    <button v-stream:click="{ subject: click$, data: 1 }">Show book</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { exhaustMap, pluck } from 'rxjs/operators'

let fetchTitle$ = x => ajax(`http://localhost:3000/books/${x}`).pipe(
  pluck('response', 'title')
)

let subscriptions = function() {
  let title$ = this.click$.pipe(
    pluck('data'),
    exhaustMap(fetchTitle$)
  )

  return { title$ }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

12 行

let fetchTitle$ = x => ajax(`http://localhost:3000/books/${x}`).pipe(
  pluck('response', 'title'),
)

fetchTitle$ 使用 RxJS 的 ajax() 直接回傳 Observable，而不必使用 from() 從 Promise 轉 Observable。

17 行

let title$ = this.click$.pipe(
  pluck('data'),
  exhaustMap(fetchTitle$),
)

使用 exhaustMap() 對 fetchTitle$() 抓資料。

當都是 Fulfilled Promise 時，這樣寫完全沒問題。

Rejected Promise

若回傳為 rejected Promise，則無法顯示 title。

<template>
  <div>
    <button v-stream:click="{ subject: click$, data: 1 }">Show book</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { exhaustMap, pluck } from 'rxjs/operators'

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title')
)

let subscriptions = function() {
  let title$ = this.click$.pipe(
    pluck('data'),
    exhaustMap(fetchTitle$)
  )

  return { title$ }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

12 行

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title')
)

URL 故意從 books 打成 book 造成 HTTP 404 error，這會產生 Rejected Promise。

catchError()

若回傳為 rejected Promise，則改顯示 title 為 RxJS in Action。

<template>
  <div>
    <button v-stream:click="{ subject: click$, data: 1 }">Show book</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { of } from 'rxjs'
import { exhaustMap, pluck, catchError } from 'rxjs/operators'

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title'),
  catchError(_ => of('RxJS in Action'))
)

let subscriptions = function() {
  let title$ = this.click$.pipe(
    pluck('data'),
    exhaustMap(fetchTitle$)
  )

  return { title$ }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

13 行

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title'),
  catchError(_ => of('RxJS in Action'))
)

使用 catchError() 捕捉 Rejected Promise，與一般 Imperative 的 catch 都是以 side effect 印出 error 不同，由於還在 stream 中，RxJS 要求在 catchError() 內回傳 Observable 繼續 stream。

值得注意的是 catchError() 要寫在 fetchBook$() 的 pipe() 內，而非 click$ 的 pipe() 內，因為 catchError() 是要攔截 fetchTitle$() 所回傳的 Rejected Promise。

let title$ = this.click$.pipe(
  pluck('data'),
  exhaustMap(fetchTitle$),
  catchError(_ => of('RxJS in Action'))
)

初學者常會將 catchError() 寫在 click$ 的 pipe()，這樣寫第一次會顯示 RxJS in Action，之後 ECMAScript 就 error 不執行了。

當 fetchBook$() 產生 Rejected Promise 時，其 pipe() 內並沒有 catchError()，因此將 error 往外層傳，click$ 的 catchError() 收到 error 顯示 RxJS in Action，但重點是 fetchBook$() 本身的 Rejected Promise 並沒有處理， 造成 runtime error 而停止。

也就是在 RxJS 觀念中，唯有在該 Observable 以 catchError() 回傳新 Observable，才算有處理 Rejected Promise，否則都算 error 未處理不繼續執行。

catchError()
處理 stream 中的 error，提供新的 Observable 繼續 stream

a、b 為原本 Observable 遇到 x error，由 catchError() 提供新的 1、2、3 … Observable 繼續。

Error Message

若想顯示 error message 而非 RxJS in Action。

<template>
  <div>
    <button v-stream:click="{ subject: click$, data: 1 }">Show book</button>
    <h1>{{ title$ }}</h1>
  </div>
</template>

<script>
import { ajax } from 'rxjs/ajax'
import { of } from 'rxjs'
import { exhaustMap, pluck, catchError } from 'rxjs/operators'

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title'),
  catchError(e => of(e.message))
)

let subscriptions = function() {
  let title$ = this.click$.pipe(
    pluck('data'),
    exhaustMap(fetchTitle$)
  )

  return { title$ }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

13 行

let fetchTitle$ = x => ajax(`http://localhost:3000/book/${x}`).pipe(
  pluck('response', 'title'),
  catchError(e => of(e.message))
)

由於 catchError()要求我們以回傳新 Observable 作為處理 Rejected Promise，因此我們只能用 of() 將 e.message 包成 Observable 回傳。

Conclusion

  • FRP 在處理 error 與 imperative 觀念不同，Imperative 是遇到 error 就直接處理 side effect，但 FRP 因為還在 stream 中，只能提供新 Observable 繼續
  • 記得 catchError() 要寫在 API Observable 的 pipe() 內，而非 DOM event Observable 的 pipe() 內，因為要處理的是 Rejected Promise，而非 DOM 的 error
