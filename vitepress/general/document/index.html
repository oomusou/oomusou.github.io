使用 VitePress 快速建立文件網站

2021-06-19

透過 VitePress 可快速建立與 VitePress 官網相同的文件網站，只需建立 Markdown 與設定 config.js 即可。

Version

VitePress 0.15.3

Document

VitePress 可透過設定顯示 sidebar 與 navbar。

當 navbar 切到 Documentation，則 sidebar 隨之改變。

當 navbar 切到 Tutorial，則 sidebar 隨之改變。

VitePress Config

.vitepress/config.js

module.exports = {
  title: 'Wink-fp (Functional Library)',
  themeConfig: {
    repo: 'oomusou/wink-fp',
    nav: [
      { text: 'Documentation', link: '/doc/array/avg' },
      { text: 'Tutorial', link: '/tutorial/install' },
    ],
    sidebar: {
      '/doc/': [
        {
          text: 'Documentation',
          children: [
            {
              text: 'Array', children: [
                { text: 'avg()', link: '/doc/array/avg' },
                { text: 'entries()', link: '/doc/array/entries' }
              ]
            },
            {
              text: 'Function', children: [
                { text: 'compose()', link: '/doc/function/compose' },
                { text: 'pipe()', link: '/doc/function/pipe' }
              ]
            },
          ]
        }
      ],
      '/tutorial/': [
        {
          text: 'Tutorial',
          children: [
            {
              text: 'Install Wink-fp',
              link: '/tutorial/install'
            },
            {
              text: 'Wink-fp on Vue',
              link: '/tutorial/vue'
            }
          ]
        },
      ],
      '/': [
        {
          text: 'Wink-fp',
          children: [
            {
              text: 'home',
              link: '/index'
            },
            {
              text: 'Why use Wink-fp',
              link: '/why'
            }
          ]
        },
      ],
    }
  }
}

完全不用寫程式，只要設定 config.js 即可。

第 2 行

title: 'Wink-fp (Functional Library)',

設定網站的 title，將顯示在最上方。

第 3 行

themeConfig: {

其餘所有設定都放在 themeConfig 下。

第 4 行

repo: 'oomusou/wink-fp',
  • repo：設定 navbar 最右側的 GitHub

目前若不設定 repo，則 navbar 無法顯示，這應該是 VitePress 的 bug

第 5 行

nav: [
  { text: 'Documentation', link: '/doc/array/avg' },
  { text: 'Tutorial', link: '/tutorial/install' },
],
  • nav：設定 navbar
  • text：設定顯示文字
  • link：設定 url

第 9 行

sidebar: {
  '/doc/': [],
  '/tutorial/': [],
  '/': [],
}
  • sidebar：設定 sidebar，由於會與 navbar 互動，因此 key 會與 nav 的 link 對應

/ 必須寫在最下面，否則 /doc/ 與 /tutorial 將不會執行

10 行

'/doc/': [
  {
    text: 'Documentation',
    children: [
      {
        text: 'Array', children: [
          { text: 'avg()', link: '/doc/array/avg' },
          { text: 'entries()', link: '/doc/array/entries' }
        ]
      },
      {
         text: 'Function', children: [
          { text: 'compose()', link: '/doc/function/compose' },
          { text: 'pipe()', link: '/doc/function/pipe' }
        ]
      },
    ]
  }
],
  • children：設定第二層 navbar

Conclusion

  • VitePress 由於還在 WIP，因此很多設定還無法在官網得知，由於 VitePress 官網本身就是使用 VitePress 建立，可直接參考 https://github.com/vuejs/vitepress/tree/master/docs 所有設定
