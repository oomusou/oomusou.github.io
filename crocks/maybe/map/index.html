使用 map() 改變 Maybe 內的值

2020-04-06

當值被包進 Maybe 後，可將 Function 傳進 map() 改變 Maybe 內的值。

Version

macOS Catalina 10.15.4
VS Code 1.43.2
Quokka 1.0.285
Ramda 0.27.0
Crocks 0.12.4

Imperative

import { inc, multiply } from 'ramda'

let f = x => {
  let y = inc(x)
  return multiply(2)(y)
}

f(1) // ?

f() 要先經過 inc()，然後再將結果傳入 multiply(2)。

Imperative 會先用 variable 保存 inc() 計算完結果，再將 variable 傳入 multiply()。

Functional

import { inc, multiply, pipe } from 'ramda'

let f = pipe(
  inc,
  multiply(2)
) 

f(1) // ?
f('hello') // ?

Functional 會使用 pipe() 將 inc() 與 multiply() 加以組合。

但 ECMAScript 是動態語言，雖然我們希望是 Number -> Number，但若傳入 string，結果就是 NaN，並不是我們預期結果。

Maybe

import { inc, multiply, pipe } from 'ramda'
import { safe, isNumber, option, map } from 'crocks'

let f = pipe(
  safe(isNumber),
  map(inc),
  map(multiply(2)),
  option(0)
)

f(1) // ?
f('hello') // ?

使用 safe(isNumber) 將 n 包成 maybe 後，須使用 map() 才能改變 maybe 內的值。

由於需經過 inc() 與 multiply() 運算，因此使用了兩次 map()。

最後再使用 option() 將 maybe 轉回 number。

map()
Maybe a ~> (a -> b) -> Maybe b
傳入 function 直接修改 maybe 內的值

Maybe a：data 為 maybe

(a -> b)：傳入 a -> b 的 function

Maybe b：將 maybe 內的 a 改成 b

Associative Laws

import { pipe, inc, multiply } from 'ramda'
import { safe, isNumber, option, map } from 'crocks'

let f = pipe(
  inc,
  multiply(2)
)

let f_ = pipe(
  safe(isNumber),
  map(f),
  option(0)
)

f_(1) // ?
f_('hello') // ?

使用兩次 map() 雖然可行，但原來的 function 已經被打散了。

事實上 maybe 具有 functor 特性，也支援 associative law，因此可將原本 function 整個傳入 map() 即可，而不用分兩次。

Conclusion

  • 不用擔心使用 maybe 後，原本的 function 需要打散後再各自 map()，因為 maybe 支援 associative law，可將整個 function 傳進 map() 即可，維持原本的 function 完全不用修改
