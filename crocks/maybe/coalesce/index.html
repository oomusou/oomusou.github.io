使用 coalesce() 提早提供 Function 處理 Nothing

2020-01-14

alt() 雖然能提早處理 Nothing，但卻是 Hardcode String，實務上使用 Function 的機會更多，此時可使用 coalensce()。

Version

macOS Mojave 10.15.2
VS Code 1.41.1
Quokka 1.0.272
Ramda 0.26.1
Crocks 0.12.4

alt()

import { prop, option, isString, and, not, isEmpty, safe, alt, Maybe } from 'crocks'
import { compose, join, split, toLower, concat, map, chain } from 'ramda'

let { of } = Maybe

let data = {
  title: '',
  price: 100,
}

let isNonEmptyString = and(not(isEmpty), isString)

let createSlug = compose(join('-'), split(' '), toLower)

let appendSlug = concat('https://lambda4.fun/crocks/')

let createUrl = compose(appendSlug, createSlug)

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  alt(of('page-not-found')),
  chain(safe(isNonEmptyString)),
  prop('title')
)

fn(data) // ?

19 行

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  alt(of('page-not-found')),
  chain(safe(isNonEmptyString)),
  prop('title')
)

使用了 chain() 做判斷，當 Maybe 為 Nothing 時，則 Maybe 內改由 page-of-found 取代。

若 fn() 可傳入 true 或 false，當 true 時為 page-not-found，false 時為 url-not-found，該如何改寫呢 ?

coalesce()

import { isString, and, not, isEmpty, prop, safe, option, map, coalesce, chain } from 'crocks'
import { compose, join, split, toLower, concat, identity, always, ifElse } from 'ramda'

let isNonEmptyString = and(not(isEmpty), isString)

let createSlug = compose(join('-'), split(' '), toLower)

let appendSlug = concat('https://lambda4.fun/crocks/')

let defaultUrl = flag => flag ? 
  always('page-not-found') : 
  always('url-not-found')

let createUrl = compose(appendSlug, createSlug);

let data = {
  title: '',
  price: 100,
};

let fn = flag => compose(
  option('https://lambda4.fun'),
  map(createUrl),
  coalesce(defaultUrl(flag), identity),
  chain(safe(isNonEmptyString)),
  prop('title')
)  

fn(true)(data) // ?

21 行

let fn = flag => compose(
  option('https://lambda4.fun'),
  map(createUrl),
  coalesce(defaultUrl(flag), identity),
  chain(safe(isNonEmptyString)),
  prop('title')
)

將 alt() 改成 coalesce()。

coalesce()
Maybe a ~> ((() -> b), (a -> b))) -> Maybe b
傳入兩個 function，當 Nothing 時執行第一個 function，Just 時執行第二個 function

Maybe a：data 為 Maybe

((() -> b), (a -> b)))：傳入兩個 function，當 Nothing 時執行 () -> b；當 Just 時執行 a -> b。

Maybe b：回傳新的 Maybe

與 alt() 比較：

alt()
Maybe a ~> Maybe a -> Maybe a
當 Maybe 為 Nothing 時，以 alt() 傳入的 Maybe 取代之

alt() 要求你傳入 Maybe，因此需使用 of() 轉成 Maybe。

但 coalesce() 則不必，只要提供 () -> b 與 a -> b 兩個 funtion 即可，會自動轉成 Maybe b。

coalesce(defaultUrl(flag), identity),

當 Nothing 時執行 defaultUrl(flag) 回傳新的 Maybe。

當 Just 時不變，因此傳入 Ramda 的 identity()。

10 行

let defaultUrl = flag => flag ? 
  always('page-not-found') : 
  always('url-not-found')

coalesce() 要求傳入 function，因此 page-not-found 與 url-not-found 要透過 Ramda 的 always() 包成 function 回傳。

Conclusion

  • alt() 與 coalesce() 功能相同，都是在 function 內提早處理 Nothing
  • alt() 直接傳入 Maybe；而 coalesce() 則要求你傳入兩個 function，當 Nothing 時執行第一個 function，Just 時執行第二個 function
