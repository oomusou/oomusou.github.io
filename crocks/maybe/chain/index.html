使用 chain() 避免兩層 Maybe

2020-04-21

若在改變 Maybe 的 map() 當中又回傳 Maybe，這樣會寫出兩層 Maybe，此時應該使用 chain() 取代 map()。

Version

macOS Catalina 10.15.4
VS Code 1.43.2
Quokka 1.0.285
Crocks 0.12.4

map()

import { pipe } from 'ramda'
import { prop, propPath, map } from 'crocks'

let data = {
  status: 200,
  body: {
    id: 1,
    userName: 'Sam',
    email: 'oomusou@gmail.com',
    address: {
      street: '111 E. West Rd.',
      city: 'Taipei',
      postalCode: '11101'
    }
  }
}

let f = pipe(
  prop('body'),
  map(propPath(['address', 'postalCode']))  
)

f(data) // ?

19 行

prop('body'),

由於使用了 Crocks 的 prop() 取得 body property，因此回傳為 Maybe。

20 行

map(propPath(['address', 'postalCode'])),

由於 prop('body') 回傳為 Maybe，f() 為了改變 Maybe 內部的值，只想取得 postalCode property 即可，因此繼續使用 map()，傳入 propPath() 取得 address.postalCode。

因為 propPath() 也回傳 Maybe，因此 f() 變成回傳兩層 Maybe。

chain()

import { pipe } from 'ramda'
import { prop, propPath, chain } from 'crocks'

let data = {
  status: 200,
  body: {
    id: 1,
    userName: 'Sam',
    email: 'oomusou@gmail.com',
    address: {
      street: '111 E. West Rd.',
      city: 'Taipei',
      postalCode: '11101'
    }
  }
}

let f = pipe(
  prop('body'),
  chain(propPath(['address', 'postalCode']))
)

f(data) // ?

18 行

let f = pipe(
  prop('body'),
  chain(propPath(['address', 'postalCode']))
)

chain()
(a -> Maybe b) -> Maybe b
傳入 function 直接修改 Maybe 內的值，並將兩層 Maybe 打層一層

與 map() 比較一下：

map()
(a -> b) -> Maybe b
傳入 function 直接修改 Maybe 內的值

可以發現 chain() 與 map() 的差異在於傳入的 function：

  • map() 為 a -> b，故直接修改 Maybe 內的值
  • chain() 為 a -> Maybe b，故可能產生兩層 Maybe

而最後 chain() 會將兩層 Maybe 攤平成一層 Maybe。

Maybe a：data 為 Maybe

(a -> b)：傳入 a -> Maybe b 的 function

Maybe b：將 Maybe Maybe b 轉成 Maybe b

option()

import { pipe } from 'ramda'
import { prop, propPath, chain, option } from 'crocks'

let data = {
  status: 200,
  body: {
    id: 1,
    userName: 'Sam',
    email: 'oomusou@gmail.com',
    address: {
      street: '111 E. West Rd.',
      city: 'Taipei',
      postalCode: '11101'
    }
  }
}

let f = pipe(
  prop('body'),
  chain(propPath(['address', 'postalCode'])),
  option('N/A')
)

f(data) // ?

15 行

let f = pipe(
  prop('body'),
  chain(propPath(['address', 'postalCode'])),
  option('N/A')
)

為了確保 f() 不是回傳 Maybe，必須再組合 option()。

Conclusion

  • Maybe 不像 Promise 的 then() 自帶 chain() 功能，因此若在 map() 中回傳 maybe 就會寫出兩層 Maybe， 別忘了使用 chain() 取代 map()，將兩層 Maybe 攤平
