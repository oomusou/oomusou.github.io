如何將 Maybe 傳入 Function ?

2020-04-10

Maybe 能讓我們確保型別正確，也能避免使用 undefined 或 null，但原本 Function 都沒針對 Maybe 設計，該如何將 Maybe 傳入原本 Function 呢 ?

Version

macOS Catalina 10.15.4
VS Code 1.43.2
Quokka 1.0.285
Crocks 0.12.4

add()

import { isNumber, safe } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b

add(safeNum1)(safeNum2) // ?

add() 只是普通 function，而 safeNum1 與 safeNum2 都是 maybe，分別為 just 1 與 just 2。

若將這兩個 maybe 直接傳入 add()，會轉型成 string 做 concat，這顯然不是我們預期的 just 3。

由於 1 與 2 都藏在 maybe 內，顯然 + 無法抓到 maybe 內的值做運算。

map()

import { isNumber, safe, pipe, ap, map, flip } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b

pipe(
  map(add),
  map(flip(map)(safeNum2))
)(safeNum1) // ?

要能改變 maybe 的內部值只能靠 map() 傳入 function，因此我們改朝 map() 下手。

將 add() 傳入 safeNum1 內，由於 add() 是 curried function，因此 maybe 內放的是 add(1)。

import { isNumber, safe, pipe, ap, map, flip } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b

pipe(
  map(add),
  map(x => safeNum2.map(x))
)(safeNum1) // ?

flip(map)(safeNum2) 比較難懂，其實只是 x => safeNum2.map(x) 的 point-free 版本，其中 x 為 add(1)，傳入 safeNum2.map() 之後回傳 just 3。

因為 safeNum1 本身已經是 maybe，而 map() 內又回傳 just 3，因此結果為 just just 3，多了一層 just。

這種寫法將 FP 帶進新的境界，透過 currying 特性，maybe 內放的已經是 function 而不是 value

chain()

import { isNumber, safe, pipe, ap, chain, map, flip } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b

pipe(
  map(add),
  chain(flip(map)(safeNum2))
)(safeNum1) // ?

若出現兩層 just，只要改用 chain() 攤平即可。

ap()

import { isNumber, safe, pipe, ap, map } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b

pipe(
  map(add),
  ap(safeNum2)
)(safeNum1) // ?

由於將 add() 傳進 map() 後，safeNum1 maybe 內是 add(1)，內部放的是 function，可使用 ap() 直接傳入 safeNum2 maybe 與 add(1) 運算。

ap()
Maybe a -> Maybe(a -> b) -> Maybe b
當 maybe 內為 function 時，可透過 ap() 傳入其他 maybe 與 function 運算

Maybe a：要傳進的另一個 maybe

Maybe(a -> b)：data 為 maybe，但其內部為 function

Maybe b：回傳新的 maybe

of()

import { Maybe, isNumber, safe, pipe, ap } from 'crocks'

let { of } = Maybe

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b
let safeAdd = of(add) 

pipe(
  ap(safeNum1),
  ap(safeNum2)
)(safeAdd) // ?

第 9 行

let safeAdd = of(add)

由於 ap() 要求的是 maybe 內部是 function，因此可使用 of() 將 add() 包進 maybe。

11 行

pipe(
  ap(safeNum1),
  ap(safeNum2)
)(safeAdd) // ?

如此可連續使用 ap() 將 safeNum1 與 safeNum2 傳入。

liftA2()

import { isNumber, safe, pipe, ap, liftA2 } from 'crocks'

let safeNum1 = safe(isNumber)(1)
let safeNum2 = safe(isNumber)(2)

let add = a => b => a + b
let safeAdd = liftA2(add)

safeAdd(safeNum1)(safeNum2) // ?

第 6 行

let add = a => b => a + b
let safeAdd = liftA2(add)

最直覺方式是使用 liftA2() 將 add() 轉成能接受 maybe 的 function，其中 A2 表示 argument 2，專門負責 binary function。

liftA2()
Applicative m => (a -> b -> c) -> m a -> m b -> m c
將普通 function 轉成能接受 maybe 的 function

a -> b -> c：輸入為普通 binary function

m a -> m b -> m c：回傳新的 binary function，但 argument 為 maybe

Conclusion

  • 直接使用 map() 或 chain() 雖然可行，但比較麻煩
  • 將 function 以 of() 包進 maybe 後，再使用 ap() 傳進 maybe 也是不錯方式
  • 最簡單直覺該屬 liftA2()，直接將普通 function lift 成能接受 maybe 的 function

Reference

Andy Van Slaars, Apply a Function in a Maybe Context to Maybe Input
