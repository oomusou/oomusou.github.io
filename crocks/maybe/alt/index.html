使用 alt() 提早處理 Nothing

2020-01-13

當使用 Maybe 後，Nothing 都交由使用端的 option() 處理，但有時候我們想在 Function 內提早處理 Nothing，此時可使用 alt()。

Version

macOS Mojave 10.15.2
VS Code 1.41.1
Quokka 1.0.272
Ramda 0.26.1
Crocks 0.12.4

Maybe

import { prop, option } from 'crocks'
import { compose, join, split, toLower, concat, map } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100,
}

let createSlug = compose(join('-'), split(' '), toLower)

let appendSlug = concat('https://lambda4.fun/crocks/')

let createUrl = compose(appendSlug, createSlug)

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  prop('title')
)

fn(data) // ?

第 4 行

let data = {
  title: 'FP in JavaScript',
  price: 100,
}

我們希望將 data.title 轉成 https://lambda4.fun/crocks/fp-in-javascript 的 url 格式。

15 行

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  prop('title')
)

Crocks 的 prop() 回傳為 Maybe，再使用 map() 傳入實際處理 url 的createUrl()，最後以 option() 處理 Nothing。

13 行

let createUrl = compose(appendSlug, createSlug)

createUrl() 組合了 createSlug() 與 appendSlug()。

第 9 行

let createSlug = compose(join('-'), split(' '), toLower)

createSlug() 組合了 toLower()、split() 與 join()，負責將 FP in JavaScript 轉成 fp-in-javascript。

11 行

let appendSlug = concat('https://lambda4.fun/crocks/')

appendSlug() 將 fp-in-javascript 組合成 https://oomusou.io/crocks/fp-in-javascript。

chain()

import { prop, option, isString, and, not, isEmpty, safe } from 'crocks'
import { compose, join, split, toLower, concat, map, chain } from 'ramda'

let data = {
  title: '',
  price: 100,
}

let isNonEmptyString = and(not(isEmpty), isString)

let createSlug = compose(join('-'), split(' '), toLower)

let appendSlug = concat('https://lambda4.fun/crocks/')

let createUrl = compose(appendSlug, createSlug)

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  chain(safe(isNonEmptyString)),
  prop('title')
)

fn(data) // ?

需求改變，希望若 title 不存在或者 title 為 empty string 時，都顯示成 https://oomusou.io/crocks/page-not-found 。

也就是 Nothing 不能再最後處理，必須在 fn() 內提早處理成 page-not-found。

第 4 行

let data = {
  title: '',
  price: 100,
}

data.title 特別改成 empty string。

17 行

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  chain(safe(isNonEmptyString)),
  prop('title')
)

加上了 safe(isNonEmptyString)() 判斷是否為 empty string，但由於 safe() 也是回傳 Maybe，為了避免兩層 Maybe，特別使用 chain() 攤平，然後繼續原來的 map(createUrl)，最後依然使用 option()。

由於 empty string 為 Nothing，最後顯示為 https://lambda4.fun，但我們需求是顯示 https://lambda4.fun/crocks/page-not-found，很明顯與需求不合。

alt()

import { prop, option, isString, and, not, isEmpty, safe, alt, Maybe } from 'crocks'
import { compose, join, split, toLower, concat, map, chain } from 'ramda'

let { of } = Maybe

let data = {
  title: '',
  price: 100,
}

let isNonEmptyString = and(not(isEmpty), isString)

let createSlug = compose(join('-'), split(' '), toLower)

let appendSlug = concat('https://lambda4.fun/crocks/')

let createUrl = compose(appendSlug, createSlug)

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  alt(of('page-not-found')),
  chain(safe(isNonEmptyString)),
  prop('title')
)

fn(data) // ?

19 行

let fn = compose(
  option('https://lambda4.fun'),
  map(createUrl),
  alt(of('page-not-found')),
  chain(safe(isNonEmptyString)),
  prop('title')
)

因為 chain(safe(isNonEmptyString)) 可能回傳 Nothing，很明顯不能再最後由 option() 處理，而要在 fn() 內提早處理 Nothing。

使用了 alt()，當 Maybe 為 Nothing 時，使用傳入 alt() 的新 Maybe 取代，of() 會將任意值轉成 Maybe，我們即將把 page-not-found 塞進 Maybe。

最後繼續使用原來的 map(createUrl)。

alt()
Maybe a ~> Maybe a -> Maybe a
當 Maybe 為 Nothing 時，以 alt() 傳入的 Maybe 取代之

Maybe a：data 為 Maybe

Maybe a：當 Maybe 為 Nothing 時，以新的 Maybe 取代

Maybe b：回傳新的 Maybe

使用端寫法完全不變，但已經顯示 https://lambda4.fun/crocks/page-not-found，因為 alt() 在 fn() 內已提前處理 Nothing。

Conclusion

  • option() 與 alt() 都能處理 Noting，但差異是 option() 處理完為 ECMAScript 原生型別；而 alt() 處理完還是 Maybe
  • alt() 適合在演算法過程中處理 Nothing，而且還要繼續使用 Maybe 時；option() 則適合使用端最後將 Maybe 轉回 ECMAScript 原生型別，順便處理 Nothing
  • 當在 Maybe 中還要加上其他邏輯判斷時，可使用 chain() + alt() 組合，讓演算法繼續使用 Maybe
