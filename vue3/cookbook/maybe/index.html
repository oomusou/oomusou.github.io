在 Vue 3 使用 Maybe Monad

2020-09-18

由於 Vue 3 的 Composition API，不只 Function Pipeline 與 Point-free 得以實現，如 FP 慣用的 Maybe Monad 在 Vue 3 裡也得以發揮。

Version

macOS Catalina 10.15.6
Vue 3.0.0
Vue-fp 0.0.3
Sanctuary 3.1.0
Ramda 0.27.0

Array.prototype.find()

輸入書本名稱，若找得到則顯示其 price，否則不顯示。

<template>
  <input v-model="searchQuery">
  {{ searchResult }}
</template>

<script>
import { ref, watchEffect } from 'vue'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

// searchQuery :: Ref String
let searchQuery = ref('')
// searchResult :: Ref String
let searchResult = ref('')

// searchTitle :: () -> ()
let searchTitle = () => {
  let result = data.find(x => x.title === searchQuery.value)
  if (result) searchResult.value = result.price
  else searchResult.value = ''
}

// setup :: () -> Data
let setup = () => {
  watchEffect(searchTitle)

  return{ searchQuery, searchResult }
}

export default {
  name:'App',
  setup
}
</script>

第 9 行

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

資料定義在 data，若輸入 FP in JavaScript，將顯示 100。

20 行

// searchTitle :: () -> ()
let searchTitle = () => {
  let result = data.find(x => x.title === searchQuery.value)
  if (result) searchResult.value = result.price
  else searchResult.value = ''
}

搭配 Vue 3 的 watchEffect()，若輸入的 searchQuery model 有所改變，將重新執行 watchEffect() 內的 searchTitle()。

使用了 ECMAScript 的 Array.prototype.find()，若找到則回傳 Object，若找不到則回傳 undefined。

由於致命的 undefined，因此必須搭配 if() 判斷，這使得 Method Chaining 中斷，必須回到 Imperative 使用 if 判斷。

27 行

// setup :: () -> Data
let setup = () => {
  watchEffect(searchTitle)

  return{ searchQuery, searchResult }
}

由 watchEffect() 監視 searchTitle() 的 dependency 有任何改變。

Maybe Monad

<template>
  <input v-model="searchQuery">
  {{ searchResult }}
</template>

<script>
import { ref, watchEffect } from 'vue'
import { unwrap, effect } from 'vue3-fp'
import { pipe, propEq, always, thunkify } from 'ramda'
import { create, env } from 'sanctuary'

let { find, maybe, prop, flip } = create ({ checkTypes: false, env })

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

// searchQuery :: Ref String
let searchQuery = ref('')
// searchResult :: Ref String
let searchResult = ref('')

// searchTitle :: () -> ()
let searchTitle = pipe(
  unwrap(searchQuery),
  propEq('title'),
  flip(find)(data),
  maybe('')(prop('price')),
  effect(searchResult)
)

// setup :: () -> Data
let setup = pipe(
  thunkify(searchTitle),
  watchEffect,
  always({ searchQuery, searchResult })
)

export default {
  name:'App',
  setup
}
</script>

25 行

// searchTitle :: () -> ()
let searchTitle = pipe(
  unwrap(searchQuery),
  propEq('title'),
  flip(find)(data),
  maybe('')(prop('price')),
  effect(searchResult)
)
  • 使用 unwrap() 讀取 searchQuery model
  • 使用 propEq() 組合 find() 所需要的 callback
  • 使用 Sanctuary 的 find()，與 Array.prototype.find() 的差異是無論找不找得到都會傳 Maybe Monad，因此不用擔心 undefined 問題
  • 使用 flip() 對 find() 的 argument 作翻轉，原本 find() 是先 predicate，然後才是 data，為了要配合 Function Pipeline，特別 flip() 先接上 data，然後才是接 propEq() 傳來的 predicate
  • 使用 maybe() 從 Maybe Monad 取得內部值，若 Nothing 則傳回 empty string，若 Just 則讀取其 price property
  • 最後使用 effect() 寫入 searchResult model

Conclusion

  • 當使用 Sanctuary 的 find() 時，若要搭配 flip()，記得要使用 Sanctuary 的 flip()，而不能使用 Ramda 的 flip()，否則無法真正確翻轉 find() 的 argument
  • 可發現由於 Sanctuary 的 find() 回傳 Maybe，因此很平順的在 Function Pipeline 使用，完全不用搭配 Imperative 的 if 做判斷，完全發揮 Vue 3 的 Composition API 優點
