當 Either 遇見 Promise

2021-04-25

當有 Function 回傳 Either Moand，而後續的 Function 都在 Either Monad 內執行時，卻又有 Function 回傳 Promise Monad，這就造成 Either 內有 Promise 的窘境，實務上該如何處理這種兩層 Monad 呢 ?

Version

Vue 3.0.11
Sanctuary 3.1.0

Promise Either

當沒有輸入任何書名時，不會繼續呼叫 API 查詢價錢。

當輸入書名時，就會繼續呼叫 API 查詢價錢。

<template lang='pug'>
input(v-model='title')
button(@click='onClick') Submit
span {{ price }}
</template>

<script setup>
import { ref } from 'vue'
import { read, write } from 'vue3-fp'
import { pipe, map, andThen as then, path, ifElse, isEmpty, thunkify } from 'ramda'
import { create, env } from 'sanctuary'
import getPrice from '/src/api/getPrice'

let { Left, Right, either, I } = create ({ checkTypes: false, env })

let title = ref('')
let price = ref('')

let chkEmpty = ifElse(
  isEmpty,
  thunkify(Left)('title is empty'),
  Right
)

let onClick = pipe(
  read(title),
  chkEmpty,
  map(getPrice),
  map(then(path(['data', 'price']))),
  map(then(write(price))),
  either(write(price))(I)
)
</script>

25 行

let onClick = pipe(
  read(title),
  chkEmpty,
  map(getPrice),
  map(then(path(['data', 'price']))),
  map(then(write(price))),
  either(write(price))(I)
)

使用 pipe() 組合出 onClick()：

  • read(title)：讀取 title state

  • chkEmpty：檢查輸入是否為空白，並回傳 Either

  • map(getPrice)：呼叫 getPrice() 從 API 查詢 price，會回傳 Promise

  • map(then(path(['data', 'price'])))：關鍵在此行，Promise 在 Either 內，因此只能 map()先拆第一層 Either，再用 then() 拆第二層 Promise，所有的 function 都在 map(then) 內
  • map(then(write(price)))：寫入 price state，後續的處理都要在 map(then) 內，以上為 happy path
  • either(write(price))(I)：處理 Left，若是 Left 則將錯誤訊息寫入 price state

19 行

let chkEmpty = ifElse(
  isEmpty,
  thunkify(Left)('title is empty'),
  Right
)

使用 ifElse() 組合出 chkEmpty()：

  • isEmpty：檢查輸入是否為空白
  • thunkify(Left)('name is empty')：若輸入為空白，則回傳 Left，相當於 Exception
  • Right：正常值則回傳 Right

Wink-fp

結果不變，但使用 Wink-fp 簡化。

<template>
  <input v-model="title">
  <button @click="onClick">Submit</button>
  {{ price }}
</template>

<script setup>
import { ref } from 'vue'
import { read, write } from 'vue3-fp'
import { pipe, map, path, ifElse, isEmpty, thunkify } from 'ramda'
import { feed } from 'wink-fp'
import { create, env } from 'sanctuary'
import getPrice from '/src/api/getPrice'

let { Left, Right, either, I } = create ({ checkTypes: false, env })

let title = ref('')
let price = ref('')

let chkEmpty = ifElse(
  isEmpty,
  thunkify(Left)('title is empty'),
  Right
)

let onClick = pipe(
  read(title),
  chkEmpty,
  map(getPrice),
  feed(path(['data', 'price'])),
  feed(write(price)),
  either(write(price))(I)
)
</script>

30 行

feed(path(['data', 'price'])),
feed(write(price)),

若覺得在 Promise Either 之後每次都要寫 map(then) 很囉唆，可改用 Wink-fp 的 feed() 取代，如此將維持在 Either 內執行時都只要一個 function 傳統。

Conclusion

  • Promise Either 乍聽很嚇人，其實只要把握用 map() 或 chain() 處理 Either Monad，用 then() 處理 Promise Monad，然後將所有 function 處理都寫在 map(then) 內即可
  • map(then) 因為實務上太常使用，可改用 Wink-fp 的 feed() 取代
