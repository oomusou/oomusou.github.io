使用 Either Monad 處理 Exception

2021-04-21

由於 Vue 3 支援 Composition API，不只 Function Pipeline 與 Point-free 得以實現，如 FP 慣用的 Either Monad 在 Vue 3 裡也得以發揮。

Version

Vue 3.0.5
Sanctuary 3.1.0

Either Moand

輸入一般值會計算出結果。

輸入 0 時會直接顯示 Exception 訊息。

<template>
  <input v-model="num">
  <button @click="onClick">Submit</button>
  {{ result }}
</template>

<script setup>
import { ref } from 'vue'
import { read, write } from 'vue3-fp'
import { pipe } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add, either, I } = create ({ checkTypes: true, env })

let num = ref(0)
let result = ref('')

let div = x => y =>
  y === 0 ?
  Left('Can not divide by zero') :
  Right(x / y)

let onClick = pipe(
  read(num),
  div(2),
  map(add(1)),
  either(I)(I),
  write(result)
)
</script>

18 行

let div = x => y =>
  y === 0 ?
  Left('Can not divide by zero') :
  Right(x / y)

自行實作 div()，當 /0 時回傳 Left，正常結果則回傳 Right 。

23 行

let onClick = pipe(
  read(num),
  div(2),
  map(add(1)),
  either(I)(I),
  write(result)
)

組合出 onClick()：

  • read(num)：讀取 num state
  • div(2)：計算除法
  • map(add(1))：使用 add() 計算加法，因為 div() 回傳 Either，所以 add() 須在 map() 內
  • either(I)(I)：使用 either(I)(I) 取出 Either 內部值
  • write(result)：寫入 result state

可發現 div() 雖然可能發出 Exception，但不會因此中斷 Function Pipeline，且後續計算都可維持在 Either Monad 內，只要最後從 Either 內部取出值即可

Promise

結果不變，但使用 Promise 模擬 Either Monad 改寫。

<template>
  <input v-model="num">
  <button @click="onClick">Submit</button>
  {{ result }}
</template>

<script setup>
import { pipe, andThen as then, otherwise, add } from 'ramda'
import { reject, resolve } from 'wink-fp'
import { ref } from 'vue'
import { read, write } from 'vue3-fp'

let num = ref(0)
let result = ref('')

let div = x => y =>
  y === 0 ?
  reject('Can not divide by zero') :
  resolve(x / y)

let onClick = pipe(
  read(num),
  div(2),
  then(add(1)),
  then(write(result)),
  otherwise(write(result))
)
</script>

16 行

let div = x => y =>
  y === 0 ?
  reject('Can not divide by zero') :
  resolve(x / y)

將 Exception 訊息改包在 Rejected Promise 內，正常值則包在 Fulfilled Promise。

21 行

let onClick = pipe(
  read(num),
  div(2),
  then(add(1)),
  then(write(result)),
  otherwise(write(result))
)

組合出 onClick()：

  • read(num)：讀取 num state
  • div(2)：計算除法
  • then(div(1))：使用 add() 計算加法，因為 div() 回傳 Promise，所以 add() 須在 then() 內
  • then(write(result))：取出 Fulfilled Promise 並寫入 result state
  • otherwise(write(result))：取出 Rejected Promise 並寫入 result state

使用 Promise 取代 Either 後，後續 function 都必須在 then() 內，且最後使用 then() 取出 Fulfilled Promise，使用 otherwise() 取出 Rejected Promise

Conclusion

  • 若 div() 使用傳統 Exception，則無法在 Function Pipeline 使用，必須回到 Imperative 寫法
  • 若改用 Either 或 Promise 回傳 Exception，配合 Vue 3 的 Composition API 則可順利以 Function Pipeline 組合出 onClick()
