Comparison between ref, reactive and $ref

2021-12-08

Reactivity in Vue 2 is simple, it only returns an object in data(). But we have ref, reactive and $ref in Vue 3. What are the differences between them ?

Version

Vue 3.2

Reactive Reference

Use ref to create a Reactive Reference :

Pros

  • ref can use both Primitive and Object
  • ref can be used in Component and Module

Cons

  • ref must use .value to get value

Primitive

<script setup>
import { ref } from 'vue'

let count = ref (0)

let onClick = () => {
  console.log ('count', count)
  console.log ('count.value', count.value)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ count }}
</template>
  • If we use console.log for Reactive Reference of Primitive, it will show RefImpl
  • We have to use .value to get the actual value of Primitive
  • We can use Reactive Reference directly without .value in the HTML template

Object

<script setup>
import { ref } from 'vue'

let store = ref ({ count: 0 })

let onClick = () => {
  console.log ('store', store)
  console.log ('store.value.count', store.value.count)
  store.value.count++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • If we use console.log for Reactive Reference of Object, it will still show RefImpl, not Proxy
  • We have to use .value first to get Object and then use property to get value
  • We can use Reactive Reference of Object directly without .value in HTML template

Object Destructuring

<script setup>
import { ref, toRefs } from 'vue'

let store = ref ({ count: 0 })

let onClick = () => {
  console.log ('store.value', store.value)
  let { count } = toRefs (store.value)
  console.log ('count', count)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • When using .value to unpack Reactive Reference of Object, we will get Proxy. So we can’t use Object Destructuring to destructure directly. We have to use toRefs before destructuring
  • Since toRefs returns Plain Object, so we can use Object Destructuring to destructure toRefs, but the destructured result is still not Primitive. It’s a Reactive Reference, so we have to use .value to get Primitive

Reactive Object

Props

  • reactive can only use Object
  • reactive no need to use .value to get Plain Object
  • reactive can be used in Component and Module

Object

<script setup>
import { reactive } from 'vue'

let store = reactive ({ count: 0 })

let onClick = () => {
  console.log ('store', store)
  console.log ('store.count', store.count)
  store.count++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • If we use console.log for Reactive Object, it will show Proxy
  • We don’t have to use .value to get the actual value of Reactive Object since it is a Proxy
  • We can use Reactive Object directly without .value in the HTML template

toRef

<script setup>
import { reactive, toRef } from 'vue'

let store = reactive ({ count: 0 })

let onClick = () => {
  let count = toRef (store, 'count')
  console.log ('count', count)
  console.log ('count.value', count.value)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • toRef : extract specific property of Reactive Object to Reactive Reference
  • The returning value of toRef is Reactive Reference, not Primitive, so we have to use .value to get Primitive

Object Destructuring

<script setup>
import { reactive, toRefs } from 'vue'

let store = reactive ({ count: 0 })

let onClick = () => {
  let store_ = toRefs (store)
  console.log ('store_', store_)

  let { count } = store_
  console.log ('count', count)
  console.log ('count.value', count.value)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • toRefs : convert Reactive Object to Plain Object with many Reactive References
  • The returning value of toRefs is Plain Object, but properties are Reactive Reference, that’s why toRefs has s
  • Since toRefs returns Plain Object, so we can use Object Destructuring to destructure toRefs, but the destructured result is still not Primitive. It’s a Reactive Reference, so we have to use .value to get Primitive

Reactive Variable

Props

  • $ref can use both Primitive and Object
  • $ref no need to use .value to get value

Cons

  • $ref can only be used in Component, not in Module

Primitive

<script setup>
let count = $ref (0)

let onClick = () => {
  console.log ('count', count)
  count++
}
</script>

<template>
  <button @click="onClick">+</button> {{ count }}
</template>
  • If we use console.log for Reactive Variable of primitive, it will show Privimite directly
  • We can use Reactive Variable directly in the HTML template

Object

<script setup>
let store = $ref ({ count: 0 })

let onClick = () => {
  console.log ('store', store)
  console.log ('store.count', store.count)
  store.count++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>
  • If we use console.log for Reactive Variable of Object, it will show Proxy like reactive
  • We don’t have to use .value to get the actual value of Reactive Variable, which is Proxy
  • We can use Reactive Variable of Object directly without .value in HTML template

toRef

<script setup>
import { toRef } from 'vue'

let store = $ref ({ count: 0 })

let onClick = () => {
  console.log ('store', store)
  let count = toRef (store, 'count')
  console.log ('count', count)
  console.log ('count.value', count.value)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }}
</template>

  • Since Reactive Variable of Object is Proxy, we have to use toRef to convert Proxy to Reactive Reference

  • toRef : extract specific property of Reactive Object to Reactive Reference

  • The returning value of roRef is Reactive Reference, not Primitive, so we have to use .value to get Primitive

Object Destructuring

<script setup>
import { toRefs } from 'vue'

let store = $ref ({ count: 0 })

let onClick = () => {
  console.log ('store', store)
  let { count } = toRefs (store)
  console.log ('count', count)
  console.log ('count.value', count.value)
  count.value++
}
</script>

<template>
  <button @click="onClick">+</button> {{ store.count }} 
</template>
  • Since Reactive Variable of Object is Proxy, we can’t use Object Destructuring to destructure it. We have to use toRefs before destructuring
  • Since toRefs returns Plain Object, so we can use Object Destructuring to destructure toRefs, but the destructured result is still not Primitive. It’s a Reactive Reference, so we have to use .value to get Primitive

Best Practice

Component

  • We recommend using Reactive Variable first. Since the syntax and usage are just like ECMAScript. We don’t have to use .value to unpack

Module

  • Primitive : We recommend using Reactive Reference. Since Reactive Variable is not available in Module
  • Object : We recommend using Reactive Object. Since we don’t have to use .value

Conclusion

  • Reactive Variable is the most intuitive way for reactivity, but we can only use it in component on Vue 3.2, not in module
