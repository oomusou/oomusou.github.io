使用 $ref 建立 Reactive Variable

2021-10-11

ref 提供了 Reactive Reference 雖然很方便，但衍生出 .value 滿天飛問題，Vue 3.2 因此提出了 Reactive Variable 新概念。

Version

Vue 3.2

ref

由經典的 counter 來看 Reactive Reference。

<script setup>
import { ref } from 'vue'

let count = ref (0)
let onClick = _ => count.value++
</script>

<template>
  <button @click="onClick">+</button> {{ count }}
</template>

使用 ref 建立 count，讀取與寫入都需使用 .value

由 Vue 2 的 this 滿天飛變成 Vue 3 的 .value 滿天飛

$ref

結果不變，但使用 Reactive Variable 改寫。

<script setup>
let count = $ref (0)
let onClick = _ => count++
</script>

<template>
  <button @click="onClick()">+</button> {{ count }}
</template>

為了解決 Reactive Reference 到處 .value 問題，Vue 3.2 提出了 $ref compiler macro，專門回傳 Reactive Variable

  • 讀取與寫入均不需 .value
  • 不需 import ref
  • 寫法更直覺，更接近原生 JavaScript

console.log

Primitive

將 Primitive 包進 Reactive Variable 後以 console.log 印出，會直接以 value 顯示，不須 .value 解開。

<script setup>
let count = $ref (0)

let onClick = _ => {
  console.log (count)
  count++
}
</script>

<template>
  <button @click="onClick()">+</button> {{ count }}
</template>

直接使用 console.log 印出 Reactive Variable。

Object

將 Object 包進 Reactive Object 後以 console.log 印出，會以 Proxy 顯示，與 reactive 完全相同

<script setup>
let state = $ref ({ count: 0})

let onClick = _ => {
  console.log (state)
  console.log (state.count)
  state.count++
}
</script>

<template>
  <button @click="onClick()">+</button> {{ state.count }}
</template>

第 4 行

let onClick = _ => {
  console.log (state)
  console.log (state.count)
  state.count++
}

直接使用 console.log 印出 Reactive Variable。

Conclusion

  • Vue 持續提供更精簡 syntax sugar，尤其 $ref 出現後，解決了 .value 滿天飛問題
  • 可發現當傳入 Object 時，$ref 與 reactive 結果相同，都是回傳 Proxy，且不必透過 .value 解開
  • $ref 算 ref 與 reactive 綜合體，當傳入 Primitive 時可不必 .value 解開，傳入 Object 時則與 reactive 相同
