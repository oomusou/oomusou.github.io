使用 reactive() 實現 Reactivity

2021-05-04

Vue 3 除了提供 ref() 實現 Reactivity，也可以使用 reactive()，但 reactive() 有些坑需要特別注意。

Version

Vue 3.0.11

reactive()

經典 counter 使用 reactive() 改寫。

<template lang='pug'>
button(@click='onClick') +
|  {{ count.data }}
</template>

<script setup>
import { reactive } from 'vue'

let count = reactive({ data: 0 })

let onClick = () => {
  let data = count.data + 1
  count = { data }
}
</script>

第 9 行

let count = reactive({ data: 0 })

改用 reactive() 宣告 count，與 ref() 不同的是 ref() 可用於 Primitive 與 Object，而 reactive() 只適用於 Object。

11 行

let onClick = () => {
  let data = count.data + 1
  count = { data }
}

因為 count 為 Reactive Object，直覺會想準備好 Object 直接指定給 count，但會發現如此將喪失 reactivity，但沒有任何錯誤訊息。

<template lang='pug'>
button(@click='onClick') +
|  {{ count.data }}
</template>

<script setup>
import { reactive } from 'vue'

let count = reactive({ data: 0 })
let onClick = () => count.data++
</script>

10 行

let onClick = () => count.data++

Reactive Object 正確使用方式是只更新其 property，而非整個 Object 更新。

toRefs()

將 reactive() 用在 module 內。

useCounter.js

import { reactive, toRefs } from 'vue'

export default () => {
  let count = reactive({
    data: 0,
    inc: () => count.data++
  })

  return count
}

若將 reactive() 用於 module，會將 data 與 function 同時包在 Reactive Object 內。

App.vue

<template lang='pug'>
button(@click='onClick') +
|  {{ count.data }}
</template>

<script setup>
import { toRefs } from 'vue'
import useCounter from '/src/modules/useCounter'

let { data, inc } = useCounter()

let onClick = inc
</script>

10 行

let { data, inc } = useCounter()

直覺會想將 Reactive Object 直接 destructure，但會發現喪失 reactivity，且沒有任何錯誤訊息。

<template lang='pug'>
button(@click='onClick') +
|  {{ count.data }}
</template>

<script setup>
import { toRefs } from 'vue'
import useCounter from '/src/modules/useCounter'

let { data, inc } = toRefs(useCounter())

let onClick = inc
</script>

10 行

let { data, inc } = toRefs(useCounter())

由於 useCounter() 是回傳 Reactive Object，而非 Plain Object，正確方式為使用 toRefs() 將 Reactive Object 轉成 Plain Object，如此才能維持其 reactivity。

Conclusion

  • reactive() 只適用於 Object，且使用上有些限制，如只能更新其 property，且必須使用 toRefs() 轉成 Plain Object 後才能 destructure
  • 若覺得 reactive() 比較麻煩，由於 ref() 可同時接受 Primitive 與 Object，因此可完全使用 ref() 取代 reactive()
