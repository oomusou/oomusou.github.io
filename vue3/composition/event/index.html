使用 Event 從 Component 內傳出 Data

2020-09-20

由於 Component 是 Unidirection Dataflow，因此 Data 只能由外往內傳，若要由內往外傳，則要使用 Event。

Version

macOS Catalina 10.15.6
Vue 3.0.0

Composition API

當按下 + 對 counter 遞增時，會即時將 data 透過 event 由內往外傳。

App.vue

<template>
  <MyCounter :start-count="outerCount" @count-change="onCountChange"></MyCounter>
  {{ outerCount }}
</template>

<script setup>
import { ref } from 'vue'
export { default as MyCounter } from '@/components/MyCounter'

// outerCount :: Ref Number
export let outerCount = ref(3)

// onCountChange :: Number -> ()
export let onCountChange = v => outerCount.value = v
</script>

第 8 行

export { default as MyCounter } from '@/components/MyCounter'

直接將 MyCounter export 出去用於 HTML template 上。

第 2 行

<MyCounter :start-count="outerCount" @count-change="onCountChange"></MyCounter>
{{ outerCount }}

<MyCounter> 除了傳入 start-count prop 外，準備接收 account-change event 所傳出 data。

10 行

// outerCount :: Ref Number
export let outerCount = ref(3)

建立 outerCount reactive reference 的初始值為 3。

13 行

// onCountChange :: Number -> ()
export let onCountChange = v => outerCount.value = v

處理 account-change event，將接收的 v 指定給 outerCount reactive reference。

MyCounter.vue

<template>
  <button @click="onClick">+</button>
</template>

<script setup="props, { emit }">
import { ref } from 'vue'

// innerCount :: Ref Number
export let innerCount = ref(0)

// onClick :: () -> ()
export let onClick = () => {
  innerCount.value++
  emit('count-change', innerCount.value)
}

innerCount.value = props.startCount

export default {
  props: [
    'startCount'
  ]
}
</script>

第 5 行

<script setup="props, { emit }">

當使用 <script setup> 時，直接在 setup 解構出 emit()。

第 8 行

// innerCount :: Ref Number
export let innerCount = ref(0)

建立 outerCount reactive reference 的初始值為 0。

17 行

innerCount.value = props.startCount

將 startCount 值指定到 innerCount reactive reference。

11 行

// onClick :: () -> ()
export let onClick = () => {
  innerCount.value++
  emit('count-change', innerCount.value)
}

當按下 + button 時，除了 innerCount 遞增外，還將 innerCount 內部值透過 emit() 發出 count-change event 傳出去。

Function Pipeline

App.vue

<template>
  <MyCounter :start-count="outerCount" @count-change="onCountChange"></MyCounter>
  {{ outerCount }}
</template>

<script setup>
import { ref } from 'vue'
import { pipe } from 'ramda'
import { effect } from 'vue3-fp'
export { default as MyCounter } from '@/components/MyCounter'

// outerCount :: Ref Number
export let outerCount = ref(3)

// onCountChange :: Number -> ()
export let onCountChange = pipe(
  effect(outerCount)
)
</script>

15 行

// onCountChange :: Number -> ()
export let onCountChange = pipe(
  effect(outerCount)
)

將 onCountChange() 由 pipe() 組合使其 point-free，將接收 data 透過 effect() 寫入 outerCount reactive reference。

MyCounter.vue

<template>
  <button @click="onClick">+</button>
</template>

<script setup="props, { emit }">
import { ref } from 'vue'
import { effect, mark, unwrap } from 'vue3-fp'
import { curryN, inc, pipe } from 'ramda'

// eject :: String -> a -> ()
let eject = curryN(2, emit)

// innerCount :: Ref Number
export let innerCount = ref(0)

// onClick :: () -> ()
export let onClick = pipe(
  unwrap(innerCount),
  inc,
  effect(innerCount),
  eject('count-change')
)

pipe(
  mark('startCount', props),
  effect(innerCount)
)()

export default {
  props: [
    'startCount'
  ]
}
</script>

24 行

pipe(
  mark('startCount', props),
  effect(innerCount)
)()
  • 使用 mark() 讀取 startCount prop
  • 使用 effect() 寫入 innerCount reactive reference

16 行

// onClick :: () -> ()
export let onClick = pipe(
  unwrap(innerCount),
  inc,
  effect(innerCount),
  eject('count-change')
)
  • 使用 unwrap() 讀取 innerCount reactive reference
  • 使用 inc() 遞增
  • 使用 effect() 寫入 innerCount reactive reference
  • 使用 eject() 將 innerCount reactive reference 從 count-change event 傳出去

10 行

// eject :: String -> a -> ()
let eject = curryN(2, emit)

eject() 其實就是 emit()，但 emit() 為 variadic function 不方便 Function Pipeline，因此使用 curryN() 將 emit() 限定為只有兩個 argument 的 eject()，且為 curry function 方便 Function Pipeline。

Conclusion

  • emit() 由 setup() 第二個 argument context 所提供，若使用 <script setup>，則可在 setup 直接 destructure 出 emit()
  • emit() 為 variadic function 不方便 Function Pipeline，所以使用 curryN() 將 emit() 轉成 eject() curry function，至於 N 為多少，可視實際需求靈活改變之
