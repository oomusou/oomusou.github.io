使用 v-model 達成 Prop 的 Two-way Binding

2021-04-30

Vue 2 可使用 v-model 與 .sync 達成 Prop 的 Two-way Binding，Vue 3 則將這兩種寫法融合成 v-mode:prop 新寫法。

Version

Vue 3.0.11

Composition API

6 為 outerCount state，而 inner + 為 MyCounter 內的 button。

outerCount 會傳進 MyCounter，且當 inner + 遞增時也會 two-way binding 更新 outerCount。

App.vue

<template lang='pug'>
span {{ outerCount }}
MyCounter(v-model:start-count='outerCount')
</template>

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

ref: outerCount = 3
</script>

10 行

ref: outerCount = 3

使用 ref sugar 定義 outerCount state 且初始值為 3。

第 2 行

span {{ outerCount }}
MyCounter(v-model:start-count='outerCount')

outerCount 除了以 v-model 傳入 <MyCounter>，也在外層顯示，因此我們希望 outerCount 能 two-way binding。

注意其 v-model 寫法與 Vue 2 不同！

  • Vue 2 在 v-model 之後不能接 prop 名稱，使用內定的 value prop
  • Vue 3 在 :prop名稱 前面加上 v-model，可使用自訂 prop
  • Vue 3 的 v-model 本質是 Vue 2 的 .sync，只是從 .sync postfix 改成 v-model prefix

MyCounter.vue

<template lang='pug'>
button(@click='onClick') inner +
</template>

<script setup>
import { defineProps, defineEmit } from 'vue'

let props = defineProps({ startCount: 0 })
let emit = defineEmit(['update:startCount'])

let onClick = () => emit('update:startCount', props.startCount + 1)
</script>

第 8 行

let props = defineProps({ startCount: 0 })

使用 defineProps() 定義 startCount prop。

第 9 行

let emit = defineEmit(['update:startCount'])

使定 defineEmit() 產生 emit()。

11 行

let onClick = () => emit('update:startCount', props.startCount + 1)

  • 使用 emit() 發出 update:startCount event

  • 若要使用 v-model，Vue 規定一定要發出 update:prop名稱 event，此與 Vue 2 的 .sync 規定相同

Point-free

結果不變，但使用 Point-free 改寫。

App.vue

<template lang='pug'>
span {{ outerCount }}
MyCounter(v-model:start-count='outerCount')
</template>

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

ref: outerCount = 3
</script>

App.vue 寫法不變。

MyCounter.vue

<template lang='pug'>
button(@click='onClick') inner +
</template>

<script setup>
import { defineProps, defineEmit } from 'vue'
import { mark, eject } from 'vue3-fp'
import { pipe, inc } from 'ramda'

let props = defineProps({ startCount: 0 })
let emit = defineEmit(['update:startCount'])

let onClick = pipe(
  mark('startCount', props),
  inc,
  eject(emit)('update:startCount')
)
</script>

13 行

let onClick = pipe(
  mark('startCount', props),
  inc,
  eject(emit)('update:startCount')
)

使用 pipe() 組合 onClick()：

  • mark('startCount', props)：讀取 startCount prop
  • inc：對值遞增
  • eject(emit)('update:startCount')：發出 update:startCount event

Conclusion

  • Vue 2 的 v-model 與 .sync 因為太類似常讓初學者困惑，Vue 3 統一使用 v-model
  • Vue 3 的 v-model 又與 Vue 2 的 v-model 不太一樣，其內涵反而比較像 .sync，只是語法從 .sync postfix 變成 v-model prefix 而已，且都是發出 update: prop名稱 event
  • 也由於 v-model 類似 .sync，因此 Vue 3 允許多個 v-model，而 Vue 2 只能有單一 v-model
