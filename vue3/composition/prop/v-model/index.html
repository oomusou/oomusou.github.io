使用 v-model 達成 Prop 的 Two-way Binding

2020-09-25

Vue 2 可使用 v-model 與 .sync 達成 Prop 的 Two-way Binding，Vue 3 則將這兩種寫法融合成 v-mode:prop 新寫法。

Version

Vue 3.0.0

Composition API

6 為 outerCount model，而 inner + 為 MyCounter 內的 button。

outerCount 會傳進 MyCounter，且當 inner + 遞增時也會 two-way binding 更新 outerCount。

App.vue

<template>
  {{ outerCount }}
  <MyCounter v-model:start-count="outerCount"></MyCounter>
</template>

<script setup>
import { ref } from 'vue'
export { default as MyCounter } from '@/components/MyCounter'

// outerCount :: Ref Number
export let outerCount = ref(3)
</script>

10 行

// outerCount :: Ref Number
export let outerCount = ref(3)

定義 outerCount reactive reference 的初始值為 3。

第 2 行

{{ outerCount }}
<MyCounter v-model:start-count="outerCount"></MyCounter>

outerCount 除了以 v-model 傳入 <MyCounter> 外，也在外層顯示，因此我們希望 outerCount能 two-way binding。

注意其 v-model 寫法與 Vue 2 不同！

  • Vue 2 在 v-model 之後不能接 prop 名稱，使用內定的 value prop
  • Vue 3 在 :prop名稱 前面加上 v-model，可使用自訂 prop
  • Vue 3 的 v-model 本直接進 Vue 2 的 .sync，只是從 .sync postfix 改成 v-model prefix

MyCounter.vue

<template>
  <button @click="onClick">inner +</button>
</template>

<script setup="props, { emit }">
// onClick :: () -> ()
export let onClick = () => emit('update:startCount', props.startCount + 1)

export default {
  props: [
    'startCount'
  ]
}
</script>

第 5 行

<script setup="props, { emit }">

因為要發出 event，在 setup 中 destructure 出 emit()。

第 6 行

// onClick :: () -> ()
export let onClick = () => emit('update:startCount', props.startCount + 1)

使用 emit() 發出 update:startCount event。

若要使用 v-model，Vue 規定一定要發出 update:prop名稱 event，此與 Vue 2 的 .sync 規定相同。

Point-free

App.vue

<template>
  {{ outerCount }}
  <MyCounter v-model:start-count="outerCount"></MyCounter>
</template>

<script setup>
import { ref } from 'vue'
export { default as MyCounter } from '@/components/MyCounter'

// outerCount :: Ref Number
export let outerCount = ref(3)
</script>

App.vue 寫法不變。

MyCounter.vue

<template>
  <button @click="onClick">inner +</button>
</template>

<script setup="props, { emit }">
import { mark } from 'vue3-fp'
import { pipe, curryN, inc } from 'ramda'

// eject :: String -> a -> ()
let eject = curryN(2, emit)

// onClick :: () -> ()
export let onClick = pipe(
  mark('startCount', props),
  inc,
  eject('update:startCount')
)

export default {
  props: [
    'startCount'
  ]
}
</script>

第 9 行

// eject :: String -> a -> ()
let eject = curryN(2, emit)

Vue 3 的 emit() 不適合 point-free，使用 curryN() 產生 curry function 版本的 eject()。

12 行

// onClick :: () -> ()
export let onClick = pipe(
  mark('startCount', props),
  inc,
  eject('update:startCount')
)
  • 使用 mark() 讀取 startCount prop
  • 使用 inc() 遞增
  • 使用 eject() 發出 update:startCount event

Conclusion

  • Vue 2 的 v-model 與 .sync 因為太類似常讓初學者困惑，Vue 3 統一使用 v-model
  • Vue 3 的 v-model 又與 Vue 2 的 v-model 不太一樣，其內涵反而比較像 .sync，只是語法從 .sync postfix 變成 v-model prefix 而已，且都是發出 update: prop名稱 event
  • 也由於 v-model 類似 .sync，因此 Vue 3 允許多個 v-model，而 Vue 2 只能有單一 v-model
