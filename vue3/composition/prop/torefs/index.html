使用 toRefs() 對 Prop 解構

2021-04-14

由於 Prop 是以 Object 形式呈現，會直覺想用 Object Destructuring 將其解構，但這會使得 Prop 喪失 Reactivity，必須透過 toRefs() 再 Destructure。

Destructure Prop

一開始 3 以 prop 傳入 component，按下 display() 可顯示。

當外層按下 + 改變 statel 到 7 時，再按下 display 卻不能同步更新 prop 所傳入資料。

App.vue

<template lang='pug'>
button(@click='onClick') +
span {{ count }}
MyCounter(:startCount='count')
</template>

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

ref: count = 3
let onClick = () => count++
</script>

第 8 行

import MyCounter from '/src/components/MyCounter.vue'

使用 import 引用 MyCounter component。

第 2 行

button(@click='onClick') +
span {{ count }}
MyCounter(:startCount='count')
  • button 負責對 count state 改變
  • 將 count 傳入 MyCounter 的 startCount prop。

11 行

ref: count = 3

定義 count state 且初始值為 3。

12 行

let onClick = () => count++

在 onClick() 對 count state 遞增。

MyCounter.vue

<template lang='pug'>
button(@click='onClick') display
span {{ count }}
</template>

<script setup>
import { defineProps } from 'vue'

let props = defineProps({ startCount: 0 })

ref: count = 0
ref: ({ startCount } = props)

let onClick = () => count = startCount
</script>

第 9 行

let props = defineProps({ startCount: 0 })

使用 defineProps() 定義 prop 且定義其預設值。

12 行

ref: ({ startCount } = props)

因為 props.startCount 可取得 prop 值，直覺會想對 prop 加以 destructure 取得其值。

如此語法並沒有錯，但 Vue 只會第一次從 prop 解構出 startCount，日後 startCount 再變動就不會 reactive 改變

toRefs()

當外層按下 + 改變 statel 到 7 時，再按下 display 能同步更新 prop 所傳入資料。

App.vue

<template lang='pug'>
button(@click='onClick') +
span {{ count }}
MyCounter(:startCount='count')
</template>

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

ref: count = 3
let onClick = () => count++
</script>

App.vue 寫法完全不變。

MyCounter.vue

<template lang='pug'>
button(@click='onClick') display
span {{ count }}
</template>

<script setup>
import { defineProps, toRefs } from 'vue'

let props = defineProps({ startCount: 0 })

ref: count = 0
ref: ({ startCount } = toRefs(props))

let onClick = () => count = startCount
</script>

12 行

ref: ({ startCount } = toRefs(props))

defineProps() 所回傳的是 Reactive Object，因此若要對其使用 object destructure，必須先使用 toRefs() 將 Reactive Object 轉成 Plain Object 後，才能繼續使用 object destructure。

Conclusion

  • 這是 Vue 3 初學者很容易遇到的地雷，若要對 prop 使用 object destructuring 且 reactive，別忘了加上 toRefs()
