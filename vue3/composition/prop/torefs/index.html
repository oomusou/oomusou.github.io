使用 toRefs 對 Prop 解構

2021-10-11

由於 Prop 是以 Object 形式呈現，會直覺想用 Object Destructuring 將其解構，但這會使得 Prop 喪失 Reactivity，必須透過 toRefs 再 Destructure。

Version

Vue 3.2.0

Destructure Prop

一開始 3 以 prop 傳入 component，按下 display 可顯示。

當外層按下 + 改變 statel 到 7 時，再按下 display 卻不能同步更新 prop 所傳入資料。

App.vue

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

let count = $ref (3)
let onClick = _ => count++
</script>

<template>
  <button @click="onClick">+</button> {{ count }}
  <MyCounter :startCount="count"/>
</template>

第 2 行

import MyCounter from '/src/components/MyCounter.vue'

使用 import 引用 MyCounter component。

第 2 行

<button @click="onClick">+</button> {{ count }}
<MyCounter :startCount="count"/>
  • button 負責對 count state 改變
  • 將 count 傳入 MyCounter 的 startCount prop。

第 4 行

let count = $ref (3)

定義 count 且初始值為 3。

第 5 行

let onClick = _ => count++

在 onClick 對 count 遞增。

MyCounter.vue

<script setup>
import { defineProps } from 'vue'

let props = defineProps ({ startCount: Number })

let count = $ref (0)
let { startCount } = props

let onClick = _ => count = startCount
</script>

<template>
  <button @click="onClick">display</button> {{ count }}
</template>

第 4 行

let props = defineProps ({ startCount: Number })

使用 defineProps 定義 startCount prop。

第 7 行

let { startCount } = props

因為 props.startCount 可取得 prop 值，直覺會想對 prop 加以 destructure 取得其值。

因為 props 為 Reactive Object，因此無法直接使用 Object Destructure，會喪失 reactivity

toRefs

當外層按下 + 改變 statel 到 7 時，再按下 display 能同步更新 prop 所傳入資料。

App.vue

<script setup>
import MyCounter from '/src/components/MyCounter.vue'

let count = $ref (3)
let onClick = _ => count++
</script>

<template>
  <button @click="onClick">+</button> {{ count }}
  <MyCounter :startCount="count"/>
</template>

App.vue 寫法完全不變。

MyCounter.vue

<script setup>
import { defineProps, toRefs } from 'vue'

let props = defineProps ({ startCount: Number })

let count = $ref (0)
let { startCount } = toRefs (props)

let onClick = _ => count = startCount.value
</script>

<template>
  <button @click="onClick">display</button> {{ count }}
</template>

第 7 行

let { startCount } = toRefs (props)

因為 props 是 Reactive Object，必須使用 toRefs 轉成 ECMAScript 的 Plain Object 之後，才能使用 Object Destructure。

第 9 行

let onClick = _ => count = startCount.value

toRefs 轉出來的雖然是 Plain Object，但每個 property 為 Reactive Reference，因此必須使用 .value 才能解開。

Conclusion

  • 這是 Vue 3 初學者很容易遇到的地雷，若要對 prop 使用 Object Destructuring 且維持 reactivity，別忘了加上 toRefs
