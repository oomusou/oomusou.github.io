v-for 之 Reactive Array

2020-11-06

Vue 2 在 v-for 的 Array 雖然能 Reactive，但有諸多限制，如不能使用 [] Index 寫法，不能使用 length 清空 Array，這些在 Vue 3 終於獲得解決。

Version

Vue 3.0.0

Vue 2

Update Array

原本顯示 1、2、3，當按下 Update Array 後修改 index 1 為 4。

<template>
  <div>
    <button @click="onClick">Update Array</button>
    <p></p>
    <ul>
      <li v-for="(item, index) in items" :key="index">{{ item }}</li>
    </ul>
  </div>
</template>

<script>
let onClick = function() {
  // this.items[1] = 4
  // this.items.splice(1, 1, 4)
  this.$set(this.items, 1, 4)
}

export default {
  data: () => ({
    items: [1, 2, 3]
  }),
  methods: {
    onClick
  },
}
</script>

12 行

let onClick = function() {
  // this.items[1] = 4
  // this.items.splice(1, 1, 4)
  this.$set(this.items, 1, 4)
}

直覺會使用 this.items[1] = 4 直接修改 Array，但這種修改方式不會使得 HTML 能 reactive 跟著改變。

Vue 2 支援兩種 reactive array 寫法：

  • 使用 splice() 直接修改 Array
  • 使用 this.$set() 直接修改 Array

Empty Array

按下 Empty Array 後會清空 Array。

<template>
  <div>
    <button @click="onClick">Empty Array</button>
    <p></p>
    <ul>
      <li v-for="(item, index) in items" :key="index">{{ item }}</li>
    </ul>
  </div>
</template>

<script>
let onClick = function() {
  // this.items.length = 0
  // this.items = []
  this.items.splice(0)
}

export default {
  data: () => ({
    items: [1, 2, 3]
  }),
  methods: {
    onClick
  },
}
</script>

12 行

let onClick = function() {
  // this.items.length = 0
  // this.items = []
  this.items.splice(0)
}

直覺會使用 this.items.length = 0 清空 Array，但這種修改方式不會使得 HTML 能 reactive 跟著改變。

Vue 2 支援兩種 reactive array 寫法：

  • 使用 [] 直接清空 array
  • 使用 splice(0) 直接清空 array

Vue 3

Update Array

<template>
  <button @click="onClick">Update Array</button>
  <p></p>
  <ul>
    <li v-for="(item, index) in items" :key="index">{{ item }}</li>
  </ul>
</template>

<script setup>
import { ref } from 'vue'

export let items = ref([1, 2, 3])

export let onClick = () => items.value[1] = 4
</script>

16 行

export let onClick = () => items.value[1] = 4

Vue 3 可直接以 [index] 修改 array 並且 reactive 改變 HTML，不必如 Vue 2 只能使用 splice() 或 $set()。

Point-free

<template>
  <button @click="onClick">Update Array</button>
  <p></p>
  <ul>
    <li v-for="(item, index) in items" :key="index">{{ item }}</li>
  </ul>
</template>

<script setup>
import { pipe, update } from 'ramda'
import { ref } from 'vue'
import { read, write } from 'vue3-fp'

export let items = ref([1, 2, 3])

export let onClick = pipe(
  read(items),
  update(1, 4),
  write(items)
)
</script>

16 行

export let onClick = pipe(
  read(items),
  update(1, 4),
  write(items)
)

也能使用 Ramda 的 update() 以 Point-free 方式修改 Array。

Empty Array

<template>
  <button @click="onClick">Empty Array</button>
  <p></p>
  <ul>
    <li v-for="(item, index) in items" :key="index">{{ item }}</li>
  </ul>
</template>

<script setup>
import { ref } from 'vue'

export let items = ref([1, 2, 3])

export let onClick = () => items.value.length = 0
// export let onClick = () => items.value = []
</script>

14 行

export let onClick = () => items.value.length = 0
// export let onClick = () => items.value = []

Vue 3 可直接以 length = 0 清空 Array 並且 reactive 改變 HTML，不必如 Vue 2 只能使用 splice(0)。

Vue 2 能使用的 [] 在 Vue 3 依然適用。

Point-free

<template>
  <button @click="onClick">Empty Array</button>
  <p></p>
  <ul>
    <li v-for="(item, index) in items" :key="index">{{ item }}</li>
  </ul>
</template>

<script setup>
import { pipe, always } from 'ramda'
import { ref } from 'vue'
import { write } from 'vue3-fp'

export let items = ref([1, 2, 3])

export let onClick = pipe(
  always([]),
  write(items)
)
</script>

16 行

export let onClick = pipe(
  always([]),
  write(items)
)

也能以 always() 與 write() 組合使其清空 Array。

Conclusion

  • 只能對 Array.prototype.splice() 或 this.$set() 支援 reactive array 算 Vue 2 很大缺點，splice() 由於其 side effect 與可讀性不高，實務上很少使用，而 $set() 因為要搭配 this 也不適合 Function Pipeline，Vue 3 的 ref() 總算解決此問題，可直覺使用 index 寫法與 length
  • Vue 3 也可改用 Point-free 寫法，完全支援 reactive array
