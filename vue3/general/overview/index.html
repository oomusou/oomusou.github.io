Vue 3 Alpha 初體驗

2020-04-09

WebStorm 2020.1 開始支援 Vue 3 Composition API，因此好奇下載 Vue 3 Alpha 測試看看，雖然離正式版一段時間，但可以先預覽 Vue 3 帶來的變革。

Version

macOS Catalina 10.15.4
WebStorm 2020.1
Vue 3.0.0-alpha.10

Installation

$ git clone https://github.com/vuejs/vue-next-webpack-preview.git vue3-experiment

Vue 3 Alpha 可先 git 下載測試。

Counter

<template>
  <button @click="inc">Clicked {{ count }} times.</button>
</template>

<script>
import { ref } from 'vue'

let count = ref(0)

let inc = () => {
  count.value++
}

export default {
  setup: () => ({
    count,
    inc
  })
}
</script>

首先來看經典的 counter 在 Vue 3 如何實現。

第 1 行

<template>
  <button @click="inc">Clicked {{ count }} times.</button>
</template>

Template 部分依舊是大家熟悉的方式。

14 行

export default {
  setup: () => ({
    count,
    inc
  })
}

原本的 Vue instance 只剩下一個 setup()，回傳 object 包含 count 與 int()，其中 count 為 data，而 inc() 為 method。

第 6 行

import { ref } from 'vue'

let count = ref(0)

count data 改由 ref() 定義，並傳入 intial value 0。

第 10 行

let inc = () => {
  count.value++
}

inc() 不需使用 this 存取 count data，這是 Vue 3 最大的變革，這導致 method 可以使用 arrow function，也可以使用 function pipeline，稍後會看到。

且 count data 需透過 count.value 才能存取。

Function Pipeline

<template>
  {{ totalPrice }}
</template>

<script>
import { ref, onMounted } from 'vue'
import { pipe, filter, pluck, sum } from 'ramda'
import { propGte } from 'wink-fp'

let totalPrice = ref(0)

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let mount = () => pipe(
  filter(propGte('price', 200)),
  pluck('price'),
  sum,
  x => totalPrice.value = x
)(data)

export default {
  setup: () => {
    onMounted(mount)

    return {
      totalPrice,
    }
  }
}
</script>

27 行

onMounted(mount)

原本的 mount hooks 需在 setup() 內以 onMounted() 掛上。

18 行

let mount = () => pipe(
  filter(propGte('price', 200)),
  pluck('price'),
  sum,
  x => totalPrice.value = x
)(data)

mount() 總算可以使用 arrow function 且能順利存取 data，這是 Vue 2 做不到的。

且 mout() 也能順利使用 Ramda 的 pipe()，唯在最後使用 arrow function 寫入 totalPrice.value，也符合在 function pipeline 時最後執行 side effect 原則。

這樣雖然可行，但有兩個小遺憾：

  • 由於 pipe() 最後傳入 data()，因此已經不是回傳 function，因此還要多一層 () => 使其成為 function
  • Side effect 使用 arrow function 而非 point-free 比較可惜

Functional Vue

<template>
  {{ totalPrice }}
</template>

<script>
import { ref, onMounted } from 'vue'
import { filter, pluck, sum } from 'ramda'
import { propGte } from 'wink-fp'

let totalPrice = ref(0)

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let pipeV = (...v) => data => () => pipe(...v)(data)

let effect = o => x => o.value = x

let mount = pipeV(
  filter(propGte('price', 200)),
  pluck('price'),
  sum,
  effect(totalPrice)
)(data)

export default {
  setup: () => {
    onMounted(mount)

    return {
      totalPrice,
    }
  }
}
</script>

18 行

let pipeV = (...v) => data => () => pipe(...v)(data)

自行建立 pipeV() higher order function 回傳 () => pipe(...v)(data)，如此 mount() 可直接使用 pipeV() 建立。

20 行

let effect = o => x => o.value = x

自行建立 effect() higher order function，只要傳入 data 即可，會幫你寫入 value property，如此 side effect 部分也可 point-free，不必使用 arrow function。

22 行

let mount = pipeV(
  filter(propGte('price', 200)),
  pluck('price'),
  sum,
  effect(totalPrice)
)(data)

如此 mount() 就可藉由 pipeV() 組合而成，side effect 部分也可藉由 effect() 使其 point-free。

Conclusion

  • 由於 Vue 3 不使用 this 存取 data，這使的 higher order function 與 function pipeline 有了用武之地，可輕易自行建立類似 pipeV() 與 effect() 的 higher order function 讓 Vue 更 FP
