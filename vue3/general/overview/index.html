Vue 3 初體驗

2020-09-27

Vue 3 招牌是其新的 Composition API，這徹底解決了 Vue 2 的 this 老問題，使得 Extract Function、Function Pipeline 與 Point-free 變得很容易，也更容易與其他 FP Library 整合。

Version

Vue 3.0.0

Vue 3

歷史性的一刻，在 Sep.18, 2020 正式發行 3.0.0 版。

Compositon API

以經典的 Counter 來看看 Vue 3 所帶來的改變。

<template>
  <button @click="onClick">+</button>
  {{ count }}
</template>

<script setup>
import { ref } from 'vue'

// count :: Ref Number
export let count = ref(0)

// onClick :: () -> ()
export let onClick = () => count.value++
</script>

第 1 行

<template>
  <button @click="onClick">+</button>
  {{ count }}
</template>

HTML template 不再限定要在單一 root element 底下，這是很親民的改變。

19 行

<script setup>

在 <script> 內加上 setup，表示自動產生 setup() boilerplate。

第 9 行

import { ref } from 'vue'

// count :: Ref Number
export let count = ref(0)

引用 ref() 建立 reactive reference。

count reactive reference 由於要 binding 到 HTML template，因此需要 reactive，Vue 3 引進 ref() reactivity API 建立 count，並將其初始值 0 傳入 ref()。

由於 count 需讓 HTML template 使用，因此加上 export。

ref()
a -> Ref a
提供初始值建立 reactive reference

a：傳入初始值

Ref a：將值包在 Object 內回傳

interface Ref<T> {
  value: T
}

其中 Ref type 只有 value property，這也是為什麼我們要透過 value 讀寫 reactive referencel。

12 行

// onClick :: () -> ()
export let onClick = () => count.value++

onClick() method 對 count reactive reference 遞增。

由於不再使用 this 讀取 reactive reference，可放心使用 arrow function。

count 基於 Ref a type，因此要使用 .value 讀取其內部值。

由於 onClick() 需讓 HTML template 使用，因此加上 export 。

以上為 Vue 3 Composition API 標準寫法，既然 Vue 3 有其 Functional 特性，可進一步使用 Function Pipeline 與 Point-free。

Function Pipeline

<template>
  <button @click="onClick">+</button>
  {{ count }}
</template>

<script setup>
import { ref } from 'vue'
import { pipe } from 'ramda'

// count :: Ref Number
export let count = ref(0)

// onClick :: () -> ()
export let onClick = pipe(
  () => count,
  x => x.value,
  x => x + 1,
  x => count.value = x
)
</script>

13 行

// onClick :: () -> ()
export let onClick = pipe(
  () => count,
  x => x.value,
  x => x + 1,
  x => count.value = x
)

onClick() 本質是 function，由於不再使用 this 讀取 reactive reference，理論上可使用 Ramda 的 pipe() 以 pure function 組合出 onClick()。

  • pipe() 要求都是 pure function，因此將 count 改成 () => count
  • 由 .value 讀取 count 內部值
  • 對 count 加 1
  • 將結果寫入 count.value

這種風格與 Promise Chain 很像，每個 then() 都是 function，且不斷將值傳到下一個 function。

Point-free

<template>
  <button @click="onClick">+</button>
  {{ count }}
</template>

<script setup>
import { ref } from 'vue'
import { pipe, inc } from 'ramda'
import { read, write } from 'vue3-fp'

// count :: Ref Number
export let count = ref(0)

// onClick :: () -> ()
export let onClick = pipe(
  read(count),
  inc,
  write(count)
)
</script>

Function Pipeline 下的 function 可進一步 point-free 使其更精簡。

14 行

// onClick :: () -> ()
export let onClick = pipe(
  read(count),
  inc,
  write(count)
)

  • 使用 read() 讀取 count，相當於 () => count.value

  • 使用 inc() 對 count 加一，相當於 x => x + 1

  • 使用 write() 寫入 count，相當於 x => count.value = x

Conclusion

  • 由經典的 counter 可發現 Vue 3 的最大改變在於 this 不見了，且 data 的初始值也搬出來由 ref() 建立，method 不再需要靠 this 存取 reactive reference，有助於 Extract Function、Function Pipeline 與 Point-free
  • 由於 onClick method 本質是 function，在沒有 this 牽絆下，可由 Function Pipeline 以 pure function 組合出 method，這也是為什麼 Vue 3 稱其為 Composition API，因為非常適合以 Higher Order Function 組合
  • 由於 Vue 3 不再使用 this，方便其他 FP library 加以整合，如 Ramda、 Sanctuary、Crocks、Wink-fp … 等
  • 在 Vue 3 正式版加上了 <script setup> 產生 setup() boilerplate，這使得 Vue 如同 Svelte 般精簡，codebase 完全沒有贅字。
  • 由於 Vue 3 的 Composition API，Vue 一躍成為 3 大 framework 中最 FP 的 framework，這也使得 Monad 實用性大增
