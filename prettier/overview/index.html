Prettier 初體驗

2024-01-15

雖然 WebStorm 可以針對 HTML/JavaScript/CSS 設定出各式各樣的 Coding Style，但也因每個人所喜歡的風格都不同，可能造成團隊合作上的困難，最簡單的方式就是大家都使用 Prettier，讓團隊的 Coding Style 統一。

Version

Prettier 3.2.2

Add Prettier

$ npm install -D prettier
  • prettier : code formatter for HTML/JavaScript/CSS

Configuration

.prettierrc

{
  "semi": false,
  "singleQuote": true,
  "bracketSameLine": true,
  "arrowParens": "avoid"
}

在專案目錄下新增 .prettierrc：

  • semi: false：JavaScript 結尾不加分號支援 ASI
  • singleQuote: true：JavaScript 的 String 使用單引號
  • bracketSameLine: true：HTML 的 attribute 造成 tag 變成多行時，> 不換行
  • arrowParens: avoid：JavaScript 的 function 參數不加 ()

WebStorm

Settings -> Languages & Frameworks -> JavaScript -> Prettier

  • Manual Prettier configuration
    • Prettier package：指定 node_modules 下的 prettier
    • Run on Reformat Code action：On
    • Run for files：**/*.{js,vue,html}
    • Run on save：On

Conclusion

  • Prettier 的哲學是盡量不提供使用者自己設定，而只使用 Prettier 預設的 coding style，但仍然有少部分可在 .prettierrc 內設定
