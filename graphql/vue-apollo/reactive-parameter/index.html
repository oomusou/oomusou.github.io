使用 Reactive Parameter 動態執行 GraphQL

2019-10-29

實務上有時我們會將 GraphQL Query 的 Argument 綁定到 Vue 的 data，讓 data 一改變就自動重新執行 GraphQL Query，此時必須使用 Vue Apollo 所提供的 Reactive Parameter。

Version

macOS Catalina 10.15
WebStorm 2019.2.3
Node 10.16.3
Yarn 1.19.1
Vue CLI 4.0.5
Vue 2.6.10
Vue-apollo 3.0.0-beta.11
Apollo-server 2.9.6

Browser

當輸入 FP 時，自動執行 GraphQL 顯示結果。

Apollo Server

index.js

import { ApolloServer, gql } from 'apollo-server';

let data = [
  { title: 'FP in JavaScript', category: 'FP'},
  { title: 'RxJS in Action', category: 'FP'},
  { title: 'Speaking JavaScript', category: 'JS'}
];

let typeDefs = gql`
  type Query {
    books(category: String): [Book]
  }

  type Book {
    title: String
    category: String
  }
`;

let books = (_, { category }) =>
  (!category) ? [] : data.filter(x => x.category === category);

let resolvers = {
  Query: {
    books
  }
};

let server = new ApolloServer({ typeDefs, resolvers });

server.listen().then(
  ({ url }) => console.log(`GraphQL Server ready at ${ url }`)
);

10 行

type Query {
  books(category: String): [Book]
}

由於 category 會動態綁定，因此 category 並非得強制提供。

20 行

let books = (_, { category }) =>
  (!category) ? [] : data.filter(x => x.category === category);

若 category 未提供為 undefined，則回傳 empty array，避免 Vue 的 v-for 失敗。

GraphQL Playground

query ($category: String) {
  books(category: $category) {
    title
    category
  }
}

在 query 宣告 $category variable 以 FP 傳入，注意 String 後面沒有 !。

query {
  books {
    title
    category
  }
}

若 books query 沒提供任何 argument 也正常回傳 empty array。

Vue Apollo

App.vue

<template>
  <div>
    <input type="text" v-model="category">
    <ul>
      <li v-for="(item, index) in books" :key="index">
        {{ item.title }}
      </li>
    </ul>
  </div>
</template>

<script>
import gql from 'graphql-tag';

let books = function() {
  let query = gql`
    query ($category: String){
      books(category: $category) {
        title
      }
    }`;

  let variables = () => ({
    category: this.category
  });

  return {
    query,
    variables
  };
};

export default {
  name: 'app',
  data: () => ({
    category: '',
    books: [],
  }),
  apollo: {
    books
  }
}
</script>

23 行

let variables = () => ({
  category: this.category
});

若 argument 要 reactive，則 variables 要由 object 改成 function 回傳 object。

Conclusion

  • ECMAScript 常看到同一個 argument 有兩用，可以傳入 primitive / object，也可傳入 callback，如此設計可盡量減少 API 個數，對 user 比較友善
