如何在 Apollo Server 以 GET 使用 REST API ?

2019-10-19

實務上仍有不少 API 以 REST 形式提供，此時我們可在 Apollo Server 使用 REST API，再提供 Graph API 給 Apollo Client。

Version

macOS Catalina 10.15
WebStorm 2019.2.3
Node 10.16.3
Apollo-server 2.9.6

Architecture

Vue Apollo 將透過 Apollo Server 存取 REST Server。

REST Server

$ mkdir rest-server
$ cd rest-server
$ yarn init --yes
$ yarn add express

使用 Node + Express 提供 REST API。

  • 在 Vue project 下建立 rest-server 目錄
  • 使用 yarn init --yes 在 rest-server 目錄下建立 package.json
  • 使用 Yarn 安裝 express

實務上可由其他語言提供 REST API，在此只是以 Node + Express 提供 REST API 示意

rest-server/index.js

let express = require('express');

let app = express();
let port = 2000;

app.get('/hello-world', (req, res) => res.send('Hello World'));

app.listen(port, () =>
  console.log(`REST Server ready at http://localhost:${ port }/`));

在 rest-server 目錄下建立 index.js。

第 1 行

let express = require('express');

在 express require 進 express()。

第 6 行

app.get('/hello-world', (req, res) => res.send('Hello World'));

提供 GET 的 /hello-world 回傳 Hello World。

第 8 行

app.listen(port, () =>
  console.log(`REST Server ready at http://localhost:${ port }/`));

以 app.listen() 啟動 REST Server，成功後以 console.log() 顯示。

Postman

$ node rest-server/index.js

直接以 Node 執行 REST Server。

Postman 已經成功取得 http://localhost:2000/hello-world 回傳資料。

Apollo Server

$ mkdir apollo-server
$ cd apollo-server
$ yarn init --yes
$ yarn add apollo-server graphql apollo-datasource-rest

使用 Apollo server 提供 GraphQL API 供 Vue Apollo 使用。

  • 在 Vue project 下建立 apollo-server 目錄
  • 使用 yarn init --yes 在 apollo-server 目錄下建立 package.json
  • 使用 Yarn 安裝 apollo-server 、 graphql 與 apollo-datasource-rest

apollo-server/index.js

let { ApolloServer, gql } = require('apollo-server');
let { RESTDataSource } = require('apollo-datasource-rest');

let typeDefs = gql`
  type Query {
    helloWorld: String
  }
`;

class RESTAPI extends RESTDataSource {
  constructor() {
    super();
    this.baseURL = 'http://localhost:2000/';
  }

  async helloWorld() {
    return await this.get('hello-world');
  }
}

let resolvers = {
  Query: {
    helloWorld: (_, __, { dataSources }) => dataSources.restAPI.helloWorld()
  }
};

let server = new ApolloServer({
  typeDefs,
  resolvers,
  dataSources: () => ({
    restAPI: new RESTAPI()
  })
});

server.listen().then(
  ({ url }) => console.log(`GraphQL Server ready at ${ url }`)
);

在 apollo-server 目錄下建立 index.js。

第 1 行

let { ApolloServer, gql } = require('apollo-server');

從 apollo-server require 進 ApolloServer 與 gql。

第 2 行

let { RESTDataSource } = require('apollo-datasource-rest');

從 apollo-datasource-rest require 進 RESTDataSource。

第 3 行

let typeDefs = gql`
  type Query {
    helloWorld: String
  }
`;

GraphQL 在 server 端第一步是建立 schema，使用的是 GraphQL SDL (GraphQL Schema Definition Language)，類似 relational database 使用 SQL 的 DDL (Data Definition Language) 定義 schema。

以 gql 開頭，後面加上 ES6 的 template string 定義 schema。

Schema 內會定義三種型別：

  • Query：查詢資料的 function
  • Mutation：修改資料的 function
  • Custom Type：自行定義的 type

第 4 行

type Query {
  helloWorld: String
}

以 type Query 開頭定義 Query，目前我們只有一個 helloWorld query，其回傳值為 String。

GraphQL 為 strongly typed language，須明確定義 type

10 行

class RESTAPI extends RESTDataSource {
  constructor() {
    super();
    this.baseURL = 'http://localhost:2000/';
  }

  async helloWorld() {
    return await this.get('hello-world');
  }
}

若要使用 Apollo Server 存取 REST API，第二步則要繼承 RESTDataSource class，並在其 constructor 內對 baseURL 設定 REST Server 的 URL。

16 行

async helloWorld() {
  return await this.get('hello-world');
}

在 RESTAPI class 自行提供 helloWorld() method 給 resolver 呼叫，主要目的是回傳在 schema 中 query 所回傳的型別，因為目前 helloWorld query 只回傳簡單的 String，因此直接使用 this.get() 回傳即可。

因為 http://localhost/hello-world 為 GET，因此使用 this.get('hello-world')，其回傳為 promise，所以使用了 async await。

21 行

let resolvers = {
  Query: {
    helloWorld: (_, __, { dataSources }) => dataSources.restAPI.helloWorld()
  }
};

GraphQL 在 server 端第三步是建立 resolver，有別於 typeDefs 為 string，resolvers 為 object。

Schema 只定義了 query 與 mutation 的 signature，其實際內容要在 resolver 定義。

定義 Query property，再以 object 定義 helloWorld() 回傳 Hello World，其 function 的前兩個 argument 依次為 parent、args，目前用不到可先用 _ 與 __ 表示。

第三個 argument 為 context，將傳入 dataSources，可透過 restAPI 呼叫 helloWorld() fetch method 回傳。

27 行

let server = new ApolloServer({
  typeDefs,
  resolvers,
  dataSources: () => ({
    restAPI: new RESTAPI()
  })
});

建立 ApolloServer object，將 typeDefs 與 resolvers 與 dataSources 組合成 object 傳進其 constructor。

其中 dataSources 為 function，需回傳 object。

35 行

server.listen().then(
  ({ url }) => console.log(`GraphQL Server ready at ${ url }`)
);

以 server.listen() 啟動 Apollo server，其回傳為 promise，將其 url destructure 之後以 console.log() 顯示。

GraphQL Playground

$ node rest-server/index.js
$ node apollo-server/index.js

直接以 Node 執行 REST Server 與 Apollo Server。

query {
  helloWorld
}

這是最簡單的 GraphQL，以 query 開頭，{} 內為要執行的 query 名稱，右側顯示回傳的 JSON object。

Conclusion

  • 若要在 Apollo Server 存取其他 REST Server 的 API，除了必須定義 schema 與 resolver 外，還要實作 data source
  • Apollo Server 要你繼承 RESTDataSource class 後，然後在 constructor 修改其 baseURL property，此屬於 OOP 與 side effect 設計，雖然不甚喜歡，但 Apollo 官方範例就是要你這樣寫，也只能暫時接受
