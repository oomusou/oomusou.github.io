如何對 Query 使用 Variable ?

2019-10-21

GraphQL 的 Query 支援 Argument，而 Argument 亦可使用 Variable 提供。

Version

macOS Catalina 10.15
WebStorm 2019.2.3
Node 10.16.3
Yarn 1.19.0
Vue CLI 3.12.0
Vue 2.6.10
Vue-apollo 3.0.0-beta.11
Apollo-server 2.9.6

Browser

Vue 使用 GraphQL 對 Apollo Server 要資料並顯示。

Apollo Server

index.js

let { ApolloServer, gql } = require('apollo-server');

let data = [
  { title: 'FP in JavaScript', category: 'FP', status: 'AVAILABLE'},
  { title: 'RxJS in Action', category: 'FP', status: 'AVAILABLE' },
  { title: 'Speaking JavaScript', category: 'JS', status: 'OUT_OF_STOCK' }
];

let typeDefs = gql`
  type Query {
    books(category: BookCategory!, status: BookStatus!): [Book]
  }

  type Book {
    title: String
    category: BookCategory
    status: BookStatus
  }

  enum BookCategory {
    FP
    FRP
    JS
  }

  enum BookStatus {
    AVAILABLE
    OUT_OF_STOCK
  }
`;

let resolvers = {
  Query: {
    books: (_, { category, status }) => data.filter(x => x.category === category && x.status === status)
  }
};

let server = new ApolloServer({ typeDefs, resolvers });

server.listen().then(
  ({ url }) => console.log(`GraphQL Server ready at ${ url }`)
);

10 行

type Query {
  books(category: BookCategory!, status: BookStatus!): [Book]
}

books query 提供 category 與 status 兩個 argument，因為一定得提供不能為 null，所以特別加上 !，GraphQL 會幫我們檢查。

20 行

enum BookCategory {
  FP
  FRP
  JS
}

enum BookStatus {
  AVAILABLE
  OUT_OF_STOCK
}

BookCategory 與 BookStatus 都為 enum。

32 行

let resolvers = {
  Query: {
    books: (_, { category, status }) => data.filter(x => x.category === category && x.status === status)
  }
};

books query 底層以 Array.prototype.filter() 實踐。

GraphQL Playground

query {
  books (category: FP) {
    title
    category
    status
  }
}

若在 GraphQL 使用 books query 只提供 category，而沒提供 status，因為在 schema 已經宣告 status: BookStatus!，GraphQL 會自動幫你擋下來，不必另外寫 code 檢查。

query {
  books(category: FP, status: AVAILABLE) {
    title
    category
    status
  }
}

提供 books query 完整的 category 與 status argument。

query ($category: BookCategory!, $status: BookStatus!) {
  books(category: $category, status: $status) {
    title
    category
    status
  }
}

若想以 variable 提供給 argument，則要在 query 後定義 variable 與其 type，並在 QUERY VARIABLES 以 JSON 提供 variable 值。

GraphQL 的 variable 以 $ 為 prefix

Ramda

index.js

let { ApolloServer, gql } = require('apollo-server');
let { filter, allPass, eqProps, always } = require('ramda');
let { useWith, unapply, converge, flip } = require('ramda');


let data = [
  { title: 'FP in JavaScript', category: 'FP', status: 'AVAILABLE'},
  { title: 'RxJS in Action', category: 'FP', status: 'AVAILABLE' },
  { title: 'Speaking JavaScript', category: 'JS', status: 'OUT_OF_STOCK' }
];

let typeDefs = gql`
  type Query {
    books(category: BookCategory!, status: BookStatus!): [Book]
  }

  type Book {
    title: String
    category: BookCategory
    status: BookStatus
  }

  enum BookCategory {
    FP
    FRP
    JS
  }

  enum BookStatus {
    AVAILABLE
    OUT_OF_STOCK
  }
`;

let books = useWith(
  flip(filter), [
    always(data),
    converge(unapply(allPass), [eqProps('category'), eqProps('status')])
  ]
);

let resolvers = {
  Query: {
    books
  }
};

let server = new ApolloServer({ typeDefs, resolvers });

server.listen().then(
  ({ url }) => console.log(`GraphQL Server ready at ${ url }`)
);

35 行

let books = useWith(
  flip(filter), [
    always(data),
    converge(unapply(allPass), [eqProps('category'), eqProps('status')])
  ]
);

books query 亦可使用 Ramda 組合而成。

books: (_, { category, status }) => data.filter(x => x.category === category && x.status === status)

若不考慮 books() 奇怪的 signature，可簡單使用以下方式使其 point-free。

compose(
  filter,
  converge(unapply(allPass), [eqProps('category'), epProps('status')])
);

由於 books() 真正 signature 為 (_, { category })，因此將 filter() 的 signature 使用 flip()反轉，再透過 useWith() 使其成為第二個 argument。

因為 filter() 第一個 argument 反而成為 data，為了配合 useWith()，使用 always() 使 data成為 function。

最後要處理的是 { category, status } 有兩個 property 要 destructure，由於 argument 為單一 object，剛好適合使用 converge() 與 unapply(AllPass)。

Vue Apollo

App.vue

<template>
  <div>
    <ul>
      <li v-for="(item, index) in books" :key="index">
        {{ item.title }}：{{ item.category }} | {{ item.status }}
      </li>
    </ul>
  </div>
</template>

<script>
import gql from 'graphql-tag';

let books = {
  query: gql`query ($category: BookCategory!, $status: BookStatus!) {
    books(category: $category, status: $status) {
      title
      category
      status
    }
  }`,
  variables: {
    category: 'FP',
    status: 'AVAILABLE'
  }
};

export default {
  name: 'app',
  apollo: {
    books
  }
}
</script>

14 行

let books = {
  query: gql`query ($category: BookCategory!, $status: BookStatus!) {
    books(category: $category, status: $status) {
      title
      category
      status
    }
  }`,
  variables: {
    category: 'FP',
    status: 'AVAILABLE'
  }
};

若要對 argument 使用 variable，則 books data 要改用 object，不能直接是 GraphQL。

其中 query property 則使用與 GraphQL Playground 相同的 GraphQL，而 varaibles property 則定義各 variable 值。

Conclusion

  • 只要對 query 的 argument 加上 !，若 client 沒提供 argument，GraphQL 會幫你擋下來，如此 backend 可不必自行檢查，也就是能進 argument 的值都已經是正確值，可直接使用

  • Vue Apollo 提供對 variable 支援，只要在 variables property 下定義 variable 即可
