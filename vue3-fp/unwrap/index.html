使用 unwrap() 讀取 Reactive Model

2020-09-05

Vue 3 希望我們使用 .value 讀取 Reactive Model 內部值，但這種方式不方便 Function Pipeline，Vue3-fp 提供了 Curry Function 版本的 unwrap()。

Version

macOS Catalina 10.15.6
Vue 3.0.0
Vue-fp 0.0.4

Composition API

<template>
  {{ name }}
</template>

<script>
import { ref, onMounted } from 'vue'

let name = ref('Sam')

let mount = () => console.log(name.value)

let setup = () => {
  onMounted(mount)
  return { name }
}

export default {
  name:'App',
  setup
}
</script>

第 8 行

let name = ref('Sam')

使用 ref() 定義 name reactive model。

10 行

let mount = () => console.log(name.value)

在 mount() 使用 console.log() 印出，並使用 .value 讀取 name reactive model 內部值。

13 行

onMounted(mount)

使用 onMounted() 將 mount() 掛到 mounted hook。

Function Pipeline

<template>
  {{ name }}
</template>

<script>
import { ref, unref, onMounted } from 'vue'
import { pipe, always, thunkify } from 'ramda'
import { log } from 'wink-fp'

let unwrap = o => pipe(
  always(o),
  unref
)

let name = ref('Sam')

let mount = pipe(
  unwrap(name),
  log
)

let setup = pipe(
  thunkify(mount),
  onMounted,
  always({ name })
)

export default {
  name:'App',
  setup
}
</script>

10 行

let unwrap = o => pipe(
  always(o),
  unref
)

unwrap() 底層使用 unref() 讀取 reactive model 內部值。

17 行

let mount = pipe(
  unwrap(name),
  log
)

mount() 使用 unwrap() 讀取 name reactive model，並且由 log() 印出。

22 行

let setup = pipe(
  thunkify(mount),
  onMounted,
  always({ name })
)

在 setup() 內將 mount() 加以 Thunk 傳入 onMounted()。

Vue3-fp

<template>
  {{ name }}
</template>

<script>
import { ref, onMounted } from 'vue'
import { pipe, always, thunkify } from 'ramda'
import { unwrap } from 'vue3-fp'
import { log } from 'wink-fp'

let name = ref('Sam')

let mount = pipe(
  unwrap(name),
  log
)

let setup = pipe(
  thunkify(mount),
  onMounted,
  always({ name })
)

export default {
  name:'App',
  setup
}
</script>

第 8 行

import { unwrap } from 'vue3-fp'

Vue3-fp 已經內建 unwrap() 可直接使用。

unwrap()
Ref a -> a
由 reactive model 取出 primitive

Ref a：包含 primitive 的 reactive model

a：回傳 reactive model 內部 primitive

Conclusion

  • 透過 .value 讀取 reactive model 並不是適合 Function Pipeline，因此另外建立了 unwrap()，可輕易在 Function Pipeline 內使用
