使用 sleep() 延遲一段時間再繼續執行

2019-09-26

很多平台都有 sleep()，讓我們可以等待一段時間再繼續執行，這常見需求在 ECMAScript 該如何實現呢 ?

Callback

let sleep = ms => cb => setTimeout(cb, ms);

let fn = msg => ms => cb => sleep(ms)(() => cb(msg));

fn('Waited 3s')(3000)(console.log);

第 5 行

fn('Waited 3s')(3000)(console.log);

提供 顯示訊息 與 延遲時間，最後傳入 console.log() 顯示結果。

第 3 行

let fn = msg => ms => cb => sleep(ms)(() => cb(msg));

呼叫底層的 sleep()。

第 1 行

let sleep = ms => cb => setTimeout(cb, ms);

所模擬的 sleep()，底層使用 setTimeout() 實作。

Async Await

let sleep = ms => new Promise(resolve => setTimeout(resolve, ms));

let fn = msg => async ms => {
  await sleep(ms);
  return msg;
};

await fn('Waited 3s')(3000); // ?

setTimeout() 本質是 asynchronous，會在 callback queue 等待，等 synchronous 都執行完才執行，在 ES5 只能使用 callback 實現。

但 ES6 迎來了 Promise 後，可改讓 sleep() 回傳 Promise 成為 asynchronous function，在 fn() 內則使用 await() 等待 sleep() ，才將 msg 回傳。

Promise Then

let sleep = ms => new Promise(resolve => setTimeout(resolve, ms));

let fn = msg => ms => sleep(ms).then(() => Promise.resolve(msg));

await fn('Waited 3s')(3000); // ?

async await 都可等價改用 then() 實現。

Pipeline

import { pipe, then, always } from 'ramda';
import { resolve } from 'wink-fp';

let sleep = ms => new Promise(resolve => setTimeout(resolve, ms));

let fn = msg => pipe(
  sleep,
  then(always(msg)),
  then(resolve),
);

await fn('Waited 3s')(3000); // ?

then() 優點是有 pipeline 概念，但可惜 then() 與 resolve() 是掛在 Promise 上，而不是獨立 function，因此只能算 method chaining。

Ramda 提供了 then()，為 Promise.prototype.then() 的 function 版本。

Wink-fp 則提供了 resolve()，為 Promise.resolve() 的 function 版本，如此就能完整實現 pipeline。

Function Composition

import { compose, then, always } from 'ramda';
import { resolve } from 'wink-fp';

let sleep = ms => new Promise(resolve => setTimeout(resolve, ms));

let fn = msg => compose(
  then(resolve),
  then(always(msg)),
  sleep,
);

await fn('Waited 3s')(3000); // ?

既然能使用 pipe() 達成 pipeline，反向使用 compose() 就是 function composition 了。

Wink-fp

import { compose, then, always } from 'ramda';
import { resolve, sleep } from 'wink-fp';

let fn = msg => compose(
  then(resolve),
  then(always(msg)),
  sleep,
);

await fn('Waited 3s')(3000); // ?

像 sleep() 這種常用的 function，Wink-fp 已經收錄，可直接使用。

sleep()
Number -> Promise
延遲一段時間後再執行

Number：ms 延遲時間

Promise：回傳為 Promise

Conclusion

  • ES6 迎來 Promise 後，使得 setTimeout() 這類 asynchronous function 有了新的用法，透過回傳 Promise，就不必再使用 callback 了
  • Promise 有兩種用法，一種是透過 ES2017 的 async await，風格接近 imperative；另一種使用 ES6 then()，風格接近 functional
  • 但 then() 又不是真的 functional，嚴格算只能算 method chaining，但已經有 pipeline 概念
  • 透過 Ramda 的 then() 與 Wink-fp 的 resolve()，可使用 Ramda 的 pipe() 加以 pipeline 組合，則更具 FP 風格
  • sleep() 因為太常使用，已經收錄在 Wink-fp
