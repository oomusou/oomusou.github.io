如何使 RxJS map() 的 callback 使用 Sanctuary 的 function ?

2020-05-10

RxJS 可處理 Synchronous 與 Asynchronous，因此實務上會統一由 RxJS 處理，Ramda 與 Sanctuary 則用於使 Callback 能 Point-free，但由於 Sanctuary 有 Type Checking 比 Ramda 嚴格，在 Point-free 過程中常會出現 Error。

Version

macOS Catalina 10.15.4
WebStorm 2020.1.1
RxJS 6.5.5
Ramda 0.27.0
Sanctuary 3.0.0

RxJS

一開始 button 會顯示 Show Book。

當 API 尚未完全回傳資料時，butotn 除了 disabled 外，且 button 顯示 Loading。

當 API 取得完整資料後，button 重回 enabled，且 button 從 Loading 顯示為 Show Book。

<template>
  <div>
    <button v-stream:click="{ subject: click$ }" :disabled="disabled$">
      {{ buttonText$ }}
    </button>
    <h1>{{ title$ }}</h1>
    <img :src="image$">
  </div>
</template>

<script>
import axios from 'axios'
import { from, merge } from 'rxjs'
import { map, pluck, switchMapTo, share } from 'rxjs/operators'
import { delay, mapTo, startWith } from 'rxjs/operators'

let fetchBook$ = id => from(axios.get(`http://localhost:3000/books/${id}`)).pipe(
  delay(3000)
)

let subscriptions = function() {
  let book$ = this.click$.pipe(
    switchMapTo(fetchBook$(1)),
    pluck('data'),
    share()
  )

  let title$ = book$.pipe(pluck('title'))

  let image$ = book$.pipe(
    pluck('image'),
    map(x => `/images/${x}`)
  )

  let disabled$ = merge(
    this.click$.pipe(mapTo(true)),
    book$.pipe(mapTo(false))
  ).pipe(startWith(false))

  let buttonText$ = disabled$.pipe(
    map(x => x ? 'Loading' : 'Show Book')
  )

  return {
    title$,
    image$,
    disabled$,
    buttonText$
  }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

17 行

let fetchBook$ = id => from(axios.get(`http://localhost:3000/books/${id}`)).pipe(
  delay(3000)
)

為了模擬慢速網路讓 Disabled Button 效果更明顯，所以 pipe() 了 delay() 3 秒鐘。

第 3 行

<button v-stream:click="{ subject: click$ }" :disabled="disabled$">
  {{ buttonText$ }}
</button>

將 disable$ Observable 綁定到 disabled attribute，由 disabled Observable 控制 button 的 disabled。

由於 button text 也會隨時間而變，因此適合使用 Observable 與 reactive programming。

35 行

let disabled$ = merge(
  this.click$.pipe(mapTo(true)),
  book$.pipe(mapTo(false))
).pipe(startWith(false))

disabled$ Observable 可由 click$ 與 book$ 兩個 Observable merge 而來，一開始由 click$mapTo true，此時 button 為 disabled，當 book$ 產生後則 mapTo false，此時 button 為 enabled。

若 disable$ 只用於綁定 disabled attribute，則沒有初始值也沒關係，即將使用 disable$ Observable 產生 buttonText$ Observable，disable$ 沒有初始值將使得其他 Observable 也沒有初始值的尷尬，因此特別加上 startWith() 指定初始值。

當使用 merge() 後的 Observable 產生其他 Observable 時，可能要考慮使用 startWith() 產生初始值

42 行

let buttonText$ = disabled$.pipe(
  map(x => x ? 'Loading' : 'Show Book')
)

buttonText$ 可由 disabled$ 判斷而來，若為 true 則顯示 Loading，否則顯示 Show Book。

buttonText$ 另外一種寫法是各自由 click$ 產生 Loading 與 book$ 產生 Show Book，最後 merge() 起來，這是比較好的寫法，本例為了展示 map() 才這樣寫

Ramda

<template>
  <div>
    <button v-stream:click="{ subject: click$ }" :disabled="disabled$">
      {{ buttonText$ }}
    </button>
    <h1>{{ title$ }}</h1>
    <img :src="image$">
  </div>
</template>

<script>
import axios from 'axios'
import { from, merge } from 'rxjs'
import { map, pluck, switchMapTo, share } from 'rxjs/operators'
import { delay, mapTo, startWith } from 'rxjs/operators'
import { concat, ifElse, identity, always } from 'ramda'

let fetchBook$ = id => from(axios.get(`http://localhost:3000/books/${id}`)).pipe(
  delay(3000)
)

let subscriptions = function() {
  let book$ = this.click$.pipe(
    switchMapTo(fetchBook$(1)),
    pluck('data'),
    share()
  )

  let title$ = book$.pipe(pluck('title'))

  let image$ = book$.pipe(
    pluck('image'),
    map(concat('/images/'))
  )

  let disabled$ = merge(
    this.click$.pipe(mapTo(true)),
    book$.pipe(mapTo(false))
  ).pipe(startWith(false))

  let buttonText$ = disabled$.pipe(
    map(ifElse(
      identity,
      always('Loading'),
      always('Show Book')
    ))
  )

  return {
    title$,
    image$,
    disabled$,
    buttonText$
  }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

31 行

let image$ = book$.pipe(
  pluck('image'),
  map(concat('/images/'))
)

map() 的 callback 可改用 Ramda 的 concat() 使其 point-free。

43 行

let buttonText$ = disabled$.pipe(
  map(ifElse(
    identity,
    always('Loading'),
    always('Show Book')
  ))
)

map() callback 中的 ternary operator 可改用 Ramda 的 ifElse() 使其 point-free。

Sanctuary

  <div>
    <button v-stream:click="{ subject: click$ }" :disabled="disabled$">

  ).pipe(startWith(false))

  let buttonText$ = disabled$.pipe(
    map(boolean('Show Book')('Loading'))
  )

  return {
    title$,
    image$,
    disabled$,
    buttonText$
  }

33 行

let image$ = book$.pipe(
  pluck('image'),
  map(concat('/images/'))
)

map() 的 callback 也可改用 Sanctuary 的 concat() 使其 point-free。

45 行

let buttonText$ = disabled$.pipe(
  map(boolean('Show Book')('Loading'))
)

map() callback 中的 ternary operator 可改用 Sanctuary 的 boolean() 使其 point-free。

boolean()
a -> a -> Boolean -> a
?: ternary operator 的 function 版本

a：當 false 的回傳值

a：當 true 的回傳值

Boolean：data 為 Boolean

a：最後回傳值

但缺出現 Expected one argument but received two arguments error。

原因是 RxJS 的 map() signature 如下：

map<T, R>(project: (value: T, index: number) => R, thisArg?: any): OperatorFunction<T, R>

map() 的 callback 會回傳 value 與 index 兩個 argument，因此被 Sanctuary 的 type checking 攔了下來。

18 行

將 checkTypes 設定為 false，則整個 component 都不採用 type checking，雖然可行但並非最佳解法。

45 行

let buttonText$ = disabled$.pipe(
  map(unchecked.boolean('Show Book')('Loading'))
)

改採用 unchecked() 下的 boolean()，表示只有這個 boolean() 不採用 type checking，其他 function 依舊使用，這算是比較好的解法。

Wrapper Function

<template>
  <div>
    <button v-stream:click="{ subject: click$ }" :disabled="disabled$">
      {{ buttonText$ }}
    </button>
    <h1>{{ title$ }}</h1>
    <img :src="image$">
  </div>
</template>

<script>
import axios from 'axios'
import { from, merge } from 'rxjs'
import { map, pluck, share } from 'rxjs/operators'
import { delay, mapTo, startWith, switchMapTo } from 'rxjs/operators'
import { create, env } from 'sanctuary'

let { boolean, concat } = create ({ checkTypes: true, env })

let fetchBook$ = id => from(axios.get(`http://localhost:3000/books/${id}`)).pipe(
  delay(3000)
)

let subscriptions = function() {
  let book$ = this.click$.pipe(
    switchMapTo(fetchBook$(1)),
    pluck('data'),
    share()
  )

  let title$ = book$.pipe(pluck('title'))

  let image$ = book$.pipe(
    pluck('image'),
    map(concat('/images/'))
  )

  let disabled$ = merge(
    this.click$.pipe(mapTo(true)),
    book$.pipe(mapTo(false))
  ).pipe(startWith(false))

  let map_ = f => map(x => f(x))

  let buttonText$ = disabled$.pipe(
    map_(boolean('Show Book')('Loading'))
  )

  return {
    title$,
    image$,
    disabled$,
    buttonText$
  }
}

export default {
  name: 'app',
  domStreams: ['click$'],
  subscriptions,
}
</script>

45 行

let map_ = f => map(x => f(x))

let buttonText$ = disabled$.pipe(
  map_(boolean('Show Book')('Loading'))
)

使用 map_() higher order function 再包一層 wrapper function，如此確保 f() 只有一個 argument，因此能通過 Sanctuary 的 type checking。

Conclusion

  • 這是當 Sanctury 搭配 RxJS 一定會遇到問題，實務上建議直接使用 unchecked()，但若該 function 常常會用到，考慮包成 wrapper function 成為 helper 也是不錯選擇
