Sanctuary 初體驗

2020-04-17

Sactuary 為著名的 Functional Library，其比 Ramda 嚴格，也能在 Runtime 檢查型別，且完整支援 Fantasy Land 的 Algebraic Data Type。

Version

macOS Catalina 10.15.4
VS Code 1.44.0
Quokka 1.0.287
Sanctuary 3.0.0

Maybe

import { create, env } from 'sanctuary'
import { pipe } from 'ramda'

let { find, maybe } = create({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let f = s => pipe(
  find(x => x.title === s),
  maybe(0)(x => x.price)
)

f('Speaking JavaScript')(data) // ?
f('RxJS')(data) // ?

第 1 行

import { create, env } from 'sanctuary'

使用 Sanctuary 第一步是引進 create() 與 env()。

第 4 行

let { find, maybe } = create({ checkTypes: true, env })

使用 create() 建立 Sanctuary instance，並從中解構出 find() 與 maybe()。

其中 checkTypes 指定 true 表示會在 runtime 檢查型別，適合 development 使用，production 則建議使用 false。

12 行

let f = s => pipe(
  find(x => x.title === s),
  maybe(0)(x => x.price)
)

Sanctuary 的 find() 回傳 Maybe，這點與 Crocks 相同。

find()
find :: Foldable f => (a -> Boolean) -> f a -> Maybe a
從 Foldable 回傳第一個符合 predicate 的 element

a -> Boolean：判斷條件的 predicate

f a：data 為 Foldable

Maybe a：將 element 包進 Maybe 內回傳

若要取出 Maybe 內的值，要使用 maybe() 取出。

maybe()
maybe :: b -> (a -> b) -> Maybe a -> b
從 Maybe 取出內部值

b：為 Nothing 提供 default value

a -> b：從 Maybe 取出時，會先執行 function

Maybe a：data 為 Maybe

b：回傳 Maybe 內經過轉換的資料

Point-free

import { create, env } from 'sanctuary'
import { pipe, propEq } from 'ramda'

let { find, maybe, prop } = create({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let f = s => pipe(
  find(propEq('title')(s)),
  maybe(0)(prop('price'))
)

f('Speaking JavaScript')(data) // ?
f('RxJS')(data) // ?

12 行

let f = s => pipe(
  find(propEq('title')(s)),
  maybe(0)(prop('price'))
)

若要使 find() 能 point-free，需使用 Ramda 的 propEq()，Sanctuary 沒有提供相對應 function。

Sanctuary 也提供 prop()，與 Ramda 相同直接回傳 value，與 Crocks 的 prop() 回傳 Maybe 不同。

Conclusion

  • Sanctuary 也提供 pipe()，但類似 Ramda 的 pipeWith() 需提供 array，因此改用 Ramda 的 pipe()
  • 與 Crocks 相比，若 Maybe 為 object，Crocks 必須先使用 chain(prop('price')) 取出 price property，再使用 option() 提供 default value 並從 Maybe 取出，總共必須兩個步驟，而 Sanctuary 只需一個 maybe() 即可，且語義也比 Crocks 優
