使用 traverse() 套用回傳 Maybe 的 Function

2021-06-14

當引入 Maybe 後，若將回傳 Maybe 的 Function 傳入 map()，則會產生 Array Maybe，若改將 Function 傳入 traverse() ，則會產生 Maybe Array，最後透過 fromMaybe() 即可得到 Array。

Version

Sanctuary 3.1.0

map()

import { map, parseInt } from 'sanctuary'

let data = ['A', 'B', 'C']

let f = map (parseInt (16))

f (data) // ?

data 為 Array，想將 Char 轉成 Integer，最後結果從 Array Char 轉成 Array Integer。

直覺會將 function 傳入 map()。

若是一般 function 傳入 map 是對的，偏偏 parseInt() 回傳 Maybe，導致結果為 Array Maybe，這並不是我們所要的。

traverse()

import { Maybe, pipe, map, parseInt, traverse, I, fromMaybe } from 'sanctuary'

let data = ['A', 'B', 'C']

let f = pipe([
  map (parseInt (16)),
  traverse (Maybe) (I),
  fromMaybe ([])
])

f (data) // ?

使用 pipe 組合 f：

  • map (parseInt (16))：每個 element 都套用 parseInt (16)
  • traverse (Maybe) (I)：每個 element 都套用 I，但最後結果為 Maybe
  • fromMaybe([])：從 Maybe 內部取出值，若為 Nothing 則回傳 Empty Array

traverse()
(Applicative f, Traversable t) => TypeRef f -> (a -> f b) -> t a -> f (t b)
將回傳 Applicative 的 function 套用到 Traversable 內，並回傳 Applicative Traversable

  • TypeRef f：傳入最後結果型別
  • (a -> f b)：傳入回傳 Applicative 的 function，每個 element 都會套用
  • t a：data 為 Traversable
  • f (t b)：回傳 Applicative Traversable 

import { Maybe, pipe, parseInt, traverse, I, fromMaybe } from 'sanctuary'

let data = ['A', 'B', 'C']

let f = pipe([
  traverse (Maybe) (parseInt (16)),
  fromMaybe ([])
])

f (data) // ?

既然 traverse() 可直接傳入 function，可將 parseInt (16) 直接傳入 traverse() 即可。

Nothing

import { Maybe, pipe, parseInt, traverse, I, fromMaybe } from 'sanctuary'

let data = ['A', 'B', 'X']

let f = pipe([
  traverse (Maybe) (parseInt (16)),
  fromMaybe ([])
])

f (data) // ?

若傳入 traverse() 的 function 會造成其中任一 element 為 Nothing，則會停止後續運算直接回傳 Nothing。

parseInt (16) ('X') 會回傳 Nothing，因此最後結果為 Nothing，經過 fromMaybe() 處理後為 Empty String。

Conclusion

  • 若 function 回傳 Maybe，傳入 map() 後會會傳 Array Maybe，也就是 Maybe 在 Array 內，這樣並不好用，若改傳入 traverse() 則回傳 Maybe Array，只要使用 fromMaybe() 就可能 Maybe 內取出資料
