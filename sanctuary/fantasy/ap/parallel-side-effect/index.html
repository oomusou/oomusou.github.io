使用 ap 平行寫入 Side Effect

2021-07-05

ap 除了用來傳入包在 Apply 的 Function 外，也常用來平行執行 Side Effect。

Version

Sanctuary 3.1.0

Array

import { ap } from 'sanctuary'
import { log, error } from 'wink-fp'

let data = [1]

let f = ap ([log, error])

f (data)

若是單一 value 想平行執行 side effect，需將 value 包進 Array 內，且 function 也要包在 Array 內一起傳進 ap。

ap
Apply f => f (a -> b) -> f a -> f b
將 function 包進 Apply 再傳入 ap 改變 Apply 內部值

f (a -> b)：傳入包進 Apply 的 function

f a：data 為 Apply

f b：回傳新 Apply

之所以要將 value 包進 Array，是為了配合 function 要包進 Array，且 ap 要求 value 與 function 都需包在 Apply 內，也就是相同 typeclass

Object

import { ap } from 'sanctuary'
import { log, error } from 'wink-fp'

let data = { x: 1, y: 2 }

let f = ap ({ x: log, y: error })

f (data)

若有多個 value，且欲各自平行執行不同 side effect，需將 value 包進 Object，且 function 也要包在 Object 內一起傳進 ap。

Conclusion

  • 一般為了平行執行 side effect 會使用 tap，但使用 ap 更為精簡

Reference

Sanctuary, ap
