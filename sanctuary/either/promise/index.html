在 Promise 內使用 Either Monad

2020-12-15

Promise 是 Monad，而 Either 也是 Monad，當這兩個 Monad 一起使用會撞出什麼火花呢 ?

Version

Sanctuary 3.1.0

Either Monad

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right } = create ({ checkTypes: true, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let f = pipe(
  fetchAPI,
  then(div(2))
)

f(0) // ?
f(1) // ?

第 8 行

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

自行實作 div()，當 /0 時以 Left 回傳 Exception，正常結果以 Right 回傳。

13 行

let f = pipe(
  fetchAPI,
  then(div(2))
)

因為 fetchAPI() 回傳 Promise，因此 div(2) 要寫在 then() 內。

可發現 Either 包在 Promise 內。

map()

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add } = create ({ checkTypes: true, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(map(add(1)))
)

f(0) // ?
f(1) // ?

13 行

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(map(add(1)))
)

由於 div() 回傳 Either，若後續還要接一般 function，是執行在 Either 內，因此要將 function 傳入 map() 內。

chain()

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add, chain } = create ({ checkTypes: true, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let mul = x => y => 
  y === 2 ?
  Left('Can not multiply by 2') :
  Right(x * y)

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(map(add(1))),
  then(chain(mul(3)))
)

f(0) // ?
f(1) // ?
f(2) // ?

13 行

let mul = x => y => 
  y === 2 ?
  Left('Can not multiply by 2') :
  Right(x * y)

mul() 為另外一個回傳 Either 的 function。

18 行

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(map(add(1))),
  then(chain(mul(3)))
)

若使用 map() 接回傳 Either 的 function，會產生 Nested Either，必須改用 chain() 將 Nested Either 攤平。

Composition Law

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add, chain } = create ({ checkTypes: true, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let mul = x => y => 
  y === 2 ?
  Left('Can not multiply by 2') :
  Right(x * y)

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3))
)  

let f = pipe(
  fetchAPI,
  then(transform),
)

f(0) // ?
f(1) // ?
f(2) // ?

18 行

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3))
)  

let f = pipe(
  fetchAPI,
  then(transform),
)

可將所有處理 Either 部分抽出 function 一次傳給 then() 即可。

either()

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add, chain, either, show, I } = create ({ checkTypes: true, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let mul = x => y => 
  y === 2 ?
  Left('Can not multiply by 2') :
  Right(x * y)

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3)),
  either(show)(I)
)  

let f = pipe(
  fetchAPI,
  then(transform),
)

f(0) // ?
f(1) // ?
f(2) // ?

Either Monad 畢竟無法用於顯示，因此必須將結果從 Either Monad 內取出。

18 行

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3)),
  either(show)(I)
)

使用 either() 從 Either Monad 中取出內部值。

fromEither()

import { pipe, andThen as then } from 'ramda'
import { create, env } from 'sanctuary'

let { Left, Right, map, add, chain, fromEither} = create ({ checkTypes: false, env })

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  Left('Can not divide by zero') : 
  Right(x / y)

let mul = x => y => 
  y === 2 ?
  Left('Can not multiply by 2') :
  Right(x * y)

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3)),
  fromEither('')
)  

let f = pipe(
  fetchAPI,
  then(transform),
)

f(0) // ?
f(1) // ?
f(2) // ?

18 行

let transform = pipe(
  div(2),
  map(add(1)),
  chain(mul(3)),
  fromEither('')
)

若需求是只要有 Exception 不必顯示錯誤訊息，顯示 Empty String 即可，可簡單使用 fromEither()。

Promise as Either

import { pipe, andThen as then, add } from 'ramda'
import { reject, resolve } from 'wink-fp'

let fetchAPI = async n => n

let div = x => y => 
  y === 0 ? 
  reject('Can not divide by zero') : 
  resolve(x / y)

let mul = x => y => 
  y === 2 ?
  reject('Can not multiply by 2') :
  resolve(x * y)

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(add(1)),
  then(mul(3))
)

f(0) // ?
f(1) // ?
f(2) // ?

Promise 搭配 Either 雖然可行，若只是想達成 fast-failing，其實可直接使用 Promise 模擬 Either。

第 6 行

let div = x => y => 
  y === 0 ? 
  reject('Can not divide by zero') : 
  resolve(x / y)

div() 不回傳 Either，只回傳 Promise，並將 Exception 以 Rejected Promise 回傳，正常結果則回傳 Fulfilled Promise。

16 行

let f = pipe(
  fetchAPI,
  then(div(2)),
  then(add(1)),
  then(mul(3))
)

由於都是 Promise，所以無論後續 function 是一般 function，或者是回傳 Promise 的 function，都可簡單使用 then()，因為 then() 兼具 map() 與 chain() 特性，不會出現 Nested Promise。

Conclusion

  • Promise 內也能順利使用 Either Monad，就如同在 Promise 內使用 Maybe Monad 一樣，只要一層一層處理 wrapper 即可
  • 由於 Promise 也具有類似 Either Monad 特性，實務上可直接使用 Promise 取代 Either，並藉由 Promise 獨特的 then() 兼具 map() 與 chain() 特性，使得 Promise 之後無論要接上一般 function，或接上回傳以 Promise 模擬 Either 的 function，都可簡單地使用 then() 即可，而不必另外使用 map() 與 chain()，這是 Promise 獨特的優越性
