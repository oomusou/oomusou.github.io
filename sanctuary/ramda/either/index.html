Sanctuary 與 Ramda 比較：Either

2020-05-03

Sanctuary 與 Ramda 另一個差異是支援 Either，這使的 ECMAScript 也能以 Functional 方式處理 Exception。

Version

macOS Catalina 10.15.4
VS Code 1.44.2
Quokka 1.0.287
Sanctuary 3.0.0

try … catch

let divide = x => y => {
  if (y === 0)
    throw new Error('Can not divide by zero')
  else
    return x / y
}

let f = x => y => divide (x) (y)

f (2) (1) // ? 
f (2) (0) // ?

ECMAScript 支援 throw 產生 Error，但並不強制使用 try...catch 處理，因此常常在特定 case 才發生 runtime exception。

let divide = x => y => {
  if (y === 0)
    throw new Error('Can not divide by zero')
  else
    return x / y

}

let f = x => y => {
  try {
    return divide (x) (y)
  } 
  catch (e) {
    console.log(e.message)
  }
}

f (2) (1) // ? 
f (2) (0) // ?

throw 必須搭配 try...catch 才完整。

但 try...catch 也面臨另外一個問題，由於遇到 throw 馬上跳到 catch，因此回傳了 undefined，又必須花另外的功處理 undefined 。

Either

import { create, env } from 'sanctuary'

let { Left, Right, either, pipe, I } = create ({ checkTypes: true, env })

let divide = x => y => 
  y === 0 ? 
  Left ('Can not divide by zero') : 
  Right (x / y)

divide (2) (1) // ?
divide (2) (0) // ?

第 5 行

let divide = x => y => 
  y === 0 ? 
  Left ('Can not divide by zero') : 
  Right (x / y)

Sanctuary 提供了 Either Monad，它包含了 Left 與 Right，其中 Left 代表 error，而 Right 代表正常值。

Either a b = Left a | Right b

與 Maybe Monad 一樣，若直接讀取 Either 會回傳 Left 或 Right，這提醒我們並需另外處理 Either，而不像 Exception 不使用 try...catch 亦可。

import { create, env } from 'sanctuary'

let { Left, Right, either, I, show, pipe } = create ({ checkTypes: true, env })

let divide = x => y => 
  y === 0 ? 
  Left ('Can not divide by zero') : 
  Right (x / y)

let f = x => pipe([
  divide (x),
  either (show) (I)
])

f (2) (1) // ?
f (2) (0) // ?

10 行

let f = x => pipe([
  divide (x),
  either (show) (I)
])

可接著使用 either()，第一個 argument 傳入處理 Left 的 function，第二個 argument 傳入處理 Right 的 function。

either()
either :: (a -> c) -> (b -> c) -> Either a b -> c
將 Either 內部值取出

a -> c：處理 Left 的 function

b -> c：處理 Right 的 function

Either a b：data 為 Either

c：回傳 Either 內部值

Conclusion

  • throw 為 side effect，且 try...catch 為 imperative 產物，很難用於 function pipeline，且很容易忘記寫 try...catch
  • Either 以 expression 回傳，方便用於 function pipeline，且因為值包在 Either 內，會自然而然使用 either() 處理
