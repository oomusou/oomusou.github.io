使用 get() 回傳 Object 的 Property

2020-12-16

Sanctuary 取得 Object 的 Property 共有兩個 Function：prop() 與 get()，其中 prop() 與 Ramda 的 prop() 相同，唯 Property 若不存在則直接 runtime error 而非回傳 undefined，而 get() 則會回傳 Maybe Monad。

Version

Sanctuary 3.0.0

prop()

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'

let { map, prop, boolean } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: false }
]

pipe(
  map(prop('inStock')),
  map(boolean('Sold Out')('Avaliable'))
)(data) // ?

先取得 inStock property 的值，再使用 boolean() 轉成 String。

get()

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, fromMaybe } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: false }
]

pipe(
  map(get(is(Boolean))('inStock')),
  map(fromMaybe(false)),
  map(boolean('Sold Out')('Avaliable'))
)(data) // ?

get() 回傳為 Maybe，因此須由 maybe() 取出內部值，若為 Nothing 則為 false，此為處理 Maybe 最標準寫法。

fromMaybe()

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, fromMaybe } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: false }
]

pipe(
  map(get(is(Boolean))('inStock')),
  map(fromMaybe(false)),
  map(boolean('Sold Out')('Avaliable'))
)(data) // ?

Maybe 多數使用情況都是 Just 就維持原值，只有 Nothing 才需要考慮，因此 maybe(false)(I) 可使用 fromMaybe(false) 取代。

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, equals, Just } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: false }
]

pipe(
  map(get(is(Boolean))('inStock')),
  map(equals(Just(true))),
  map(boolean('Sold Out')('Avaliable')),
)(data) // ?

以本例而言，關鍵在於 get() 回傳 Maybe，這使得 true 變成 Just(true)，而 false 變成 Just(false)，而 Nothing 也算廣義 false。

因此也可 equals(Just(true))，也就是只有 Just(true) 回傳 true，其餘都是 false。

在此我們看到 equals() 不只能用在 primitive，也能直接用在 Maybe

Composition Law

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, equals, Just } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: false }
]

let transform = pipe(
  get(is(Boolean))('inStock'),
  equals(Just(true)),
  boolean('Sold Out')('Avaliable')
)

pipe(
  map(transform)
)(data) // ?

根據 composition law，我們可將 map() 提出來，只需一個 map() 即可。

map() Again

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, fromMaybe } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: 'false' }
]

pipe(
  map(get(is(Boolean))('inStock')),
  map(map(boolean('Sold Out')('Avaliable'))),
  map(fromMaybe('Sold Out'))
)(data) // ?

以上寫法都是在 get() 之後馬上處理 Maybe，雖然可行，但卻中斷了原本happy path 去處理 Maybe，類似在 imperative 中插入 if 去判斷 undefined 一樣。

比較好的寫法是原本邏輯直接在 Maybe 內處理，最後再使用 fromMaybe() 取得內部值。

兩層 map() 乍看之下很怪，外層的 map() 是處理 Array，內層 map() 是處理 Maybe

Composition Law

import { pipe } from 'ramda'
import { create, env } from 'sanctuary'
import { Boolean } from 'sanctuary-def'

let { map, get, boolean, is, fromMaybe } = create ({ checkTypes: true, env })

let data = [
  { title: 'FP in JavaScript', inStock: true },
  { title: 'RxJS in Action', inStock: false },
  { title: 'Speaking JavaScript', inStock: 'false' }
]

let transform = pipe(
  get(is(Boolean))('inStock'),
  map(boolean('Sold Out')('Avaliable')),
  fromMaybe('Sold Out')
)

pipe(
  map(transform)
)(data) // ?

根據 composition law 可將處理 Array 的 map() 提出來，pipe() 內只留下處理 Maybe 的 map()，這樣就好看多了。

Conclusion

  • get() 多包了一層 Maybe 雖然比較安全，但相對處理上也必須付出代價，可在 get() 之後立即處理 Maybe，也可在 Maybe 內繼續運算，實務上建議以 happy path 在 Maybe 內，直到最後再從 Maybe 內取出
