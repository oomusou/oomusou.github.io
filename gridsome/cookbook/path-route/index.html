如何以 Markdown 的 Path 與 File Name 設定 Route ?

2020-11-01

Gridsome 預設會以 title 為其 Route，這對英文理所當然，但實務上 title 會使用中文，這對 Gridsome 則面臨挑戰，我們更希望直接以 Markdown 的 Folder 與 File Name 為 Route。

Version

Gridsome 0.7.21

Route

Route 與 markdown 的 folder 與 file name 相同。

Gridsome Config

gridsome.config.js

let { pipe, split, drop, join, concat, __ } = require('ramda')

let resolve = dir => require('path').join(__dirname, dir);

module.exports = {
  siteName: 'Gridsome',
  siteDescription: 'This is My Blog',
  plugins: [
    {
      use: '@gridsome/source-filesystem',
      options: {
        typeName: 'Blog',
        path: './content/blog/**/*.md',
      }
    }
  ],
  templates: {
    Blog: node => {
      let { fileInfo: { directory, name }} = node

      return pipe(
        split('/'),
        drop(2),
        join('/'),
        concat('/'),
        concat(__, `/${name}`)
      )(directory)
    }
  },
  resolve: {
    alias: {
      '@': resolve('src')
    }
  }
}

第 1 行

let { pipe, split, drop, join, concat, __ } = require('ramda')

引用 Ramda 的 function。

17 行

templates: {
  Blog: node => {
    let { fileInfo: { directory, name }} = node

    return pipe(
      split('/'),
      drop(2),
      join('/'),
      concat('/'),
      concat(__, `/${name}`)
    )(directory)
  }
},

在 templates 的 collection 除了能回傳 String 外，事實上也能使用 function。

Function 的 argument 傳入 static query 的 edges.node，由於我們想使用 markdown 的 path 與 file name 為 route，因此從 fileInfo 中 destructure 出 directory 與 name。

以本例而言，markdown 是被放在 content/blog/ folder 下，directory 會傳回 ./content/blog/，其中 . 與 content 為多餘必須去除：

  • split('/')：將 String 轉為 Array
  • drop(2)：將 . 與 content 去除
  • join('/')：將 Array 重新合併為 String
  • concat('/')：在 path 前面加上 /
  • concat(__, '/${name}')：在 path 後面補上 markdown 的 file name

Conclusion

  • Gridsome 雖然內建無法以 markdown 的 path 與 file name 為 URI，但所幸 template 可使用 function，因此可使用 Ramda 自行組合 function 完成
