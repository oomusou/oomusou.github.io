Gridsome 之 Layout

2019-12-31

若想定義 Page 或 Template 的架構，如 Header、Footer、Sidebar…，可使用 Layout 定義之。

Version

macOS Catalina 10.15.2
WebStorm 2019.3.1
Gridsome 0.7.0

Simple Page

src/pages/Product.vue

<template>
  <h1>Product</h1>
</template>

<script>
export default {
  name:"Product"
}
</script>

在 src/pages 目錄下新增 Product.vue，會自動成為 product page。

Page with Layout

src/pages/Product.vue

<template>
  <Layout>
    <h1>Product</h1>
  </Layout>
</template>

<script>
export default {
  name:"Product"
}
</script>

在 <h1/> 外加上 <Layout/>，會自動套用 Layout layout。

product page 馬上有 header 了。

但為什麼 Product.vue 不用 import layout component 就直接能使用呢 ?

Layout

Gridsome 在 src/layouts 目錄下放了 Default.vue，其本質是使用 slot 的 component，將由 page 或 template 提供 slot 內容。

Layout 通常會在 src/main.js 使用 Vue.component 註冊成 global component，因此每個 page 能直接使用。

由於 layout 本質就是 component，若你不想在 main.js 註冊成 global component，也可在每個 page 中單獨 import layout 使用。

Conclusion

  • Layout 特別適合建立 page 或 template 的架構，其本質就是使用 slot 的 global component，只是 Gridsome 另外提出 layout 概念，語意更佳
