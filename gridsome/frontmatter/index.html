Gridsome 之 Frontmatter

2020-01-09

Post 的 Meta Information 可寫在 Markdown 的 Frontmatter，Gridsome 可透過 GraphQL 加以讀取。

Version

macOS Catalina 10.15.2
WebStorm 2019.3.1
Gridsome 0.7.12

Frontmatter

_source/_posts/first-post.md

---
title: First Post
excerpt: First post description
date: 2020-01-08 04:04:04
tags: ['JavaScript', 'Ramda']
---

This is `first post` content

在 Markdown 最上方以 --- 所框住的部分為 frontmatter，包含幾個常用的 property：

  • title：post 的標題
  • excerpt：post 的 overview
  • date：post 的日期
  • tags：post 的 tag，可多個 tag 故為 array

Markdown 只要一修改，dev server 會自動 hot reload

GraphQL Playground

由 GraphQL schema 可發現在 frontmatter 定義的 title、excerpt、date 與 tags 都能找到相對應 field，還有根據字數自動計算的 timeToRead。

query {
  allPost {
    edges {
      node {
        title
        excerpt
        date (format: "MMM.DD, YYYY")
        tags
        timeToRead
        content
      }
    }
  }
}

日期格式非常多種，因此 Gridsome 特別為 date() 提供了 format argument。

Gridsome 底層使用了 Moment， format 的 arguemt 可參考其官網

可發現 frontmatter 資料可被 GraphQL 查詢。

Page

src/pages/Blog.vue

<template>
  <Layout>
    <h1>Blog</h1>
    <article v-for="(edge, index) in $static.allPost.edges" :key="index">
      <h2>{{ edge.node.title }}</h2>
      <p>{{ edge.node.excerpt }}</p>
      <p>Posted {{ edge.node.date }} {{ edge.node.timeToRead }} min read</p>
      <div>
        <span v-for="tag in edge.node.tags" :key="tag">#{{ tag }} </span>
      </div>
    </article>
  </Layout>
</template>

<static-query>
query {
  allPost {
    edges {
      node {
        title
        excerpt
        date (format: "MMM.DD, YYYY")
        tags
        timeToRead
        content
      }
    }
  }
}
</static-query>

<script>
export default {
  name:"Blog"
}
</script>

  <h2>{{ edge.node.title }}</h2>

使用 edge.node 存取 frontmatter。

由於 edge.node.tags 是 array，可再使用 v-for 展開。

Conclusion

  • Frotmatter 亦為各 JAMstack 必備格式，Gridsome 可透過 GraphQL 加以讀取，非常方便
