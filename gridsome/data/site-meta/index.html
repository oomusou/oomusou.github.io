Gridsome 之 Site Meta 與 GraphQL

2020-09-08

除了 Page Meta，Gridsome 也提供了 Site Meta，專門存放整個 Site 的 Global Data，並使用 GraphQL Query 查詢。

Version

macOS Catalina 10.15.2
Gridsome 0.7.12

Site Meta

在首頁左上角我們看到了 Gridsome hyperlink。

src/layouts/Default.vue

其定義在 src/layouts/Default.vue。

第 5 行

<g-link to="/">{{ $static.metadata.siteName }}</g-link>

我們看到了 $static.metadata.siteName，這正是 Gridsome String 所 binding 之處，但這又是什麼呢 ?

16 行

<static-query>
query {
  metadata {
    siteName
  }
}
</static-query>

我們看到了 Gridsome 有別於傳統 Vue 只有 <template/>、<script/> 與 <style/> 三個區塊，目前還多了 <static-query/>，而其中以 GraphQL 定義 metadata query，並回傳 siteName field。

也因此 <template/> 可在 layout 一執行時透過 $static.medata.siteName 取得 Gridsome string。

但 Gridsome string 定義在哪呢 ?

gridsome.config.js

在根目錄的 gridsome.config.js 的 module.exports 內定義了 siteName property，其值為 Gridsome。

也就是在 gridsome.config.js 所定義的資料為 site meta，可在此定義整個 site 的 global data，則所有 page 都可透過 <static-query/> 、GraphQL 與 $static 取得。

GraphQL Playground

除此之外，Gridsome 亦整合了 GraphQL Playgound，可直接在此使用 GraphQL query 查詢 site meta。

Conclusion

  • 若修改 gridsome.config.js 內的 site meta，必須手動重新啟動 yarn serve，因為該檔不在 src 目錄下，因此 Dev Server 不會 hot reload

Reference

Reed Barger, Site Meta Data, Intro to Static Queries
Gridsome, Project Configuration
