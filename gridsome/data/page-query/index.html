Gridsome 之 Page Query

2020-11-01

Blog 每篇 Post 都是獨立 Markdown，透過 Template 與 Page Query 可將 Post 以 HTML 顯示。

Version

Gridsome 0.7.21

Page Query

按下 More 後可單獨顯示 blog。

一整頁單獨顯示整篇 blog。

Gridsome Config

gridsome.config.js

module.exports = {
  siteName: 'Gridsome',
  siteDescription: 'This is My Blog',
  plugins: [
    {
      use: '@gridsome/source-filesystem',
      options: {
        typeName: 'Blog',
        path: './content/blog/**/*.md',
      }
    }
  ],
  templates: {
    Blog: '/:title'
  }
}

第 6 行

plugins: [
  {
        use: '@gridsome/source-filesystem',
    options: {
      typeName: 'Blog',
      path: './content/blog/**/*.md',
    }
  }
],

設定 @gridsome/source-filesystem plugin：

  • typeName：設定 content/blog folder 下的 *.md 的 type 為 Blog
  • path：設定 content/blog folder 下，包含所有 sub-folder 的 *.md 皆適用

13 行

templates: {
  Blog: '/:title'
},

設定 Blog collection 的 route 為 /:title，其中 title 來自於 frontmatter 的 title。

AllBlog Query

query {
  allBlog {
    edges {
      node {
        title
        path
      }
    }
  }
}

allBlog query 包含 path field，可查詢每篇 post 的 URI。

Blog Query

query($path: String!) {
  blog(path: $path) {
    content
  }
}

{
  "path": "/first-post"
}

Gridsome 另外提供 blog query，以 path 為 argument，可回傳 blog 完整內容。

Gridsome Page

pages/BlogList.vue

<template>
  <layout>
    <h1>Blog</h1>
    <article v-for="({ node: { title, date, tags, path }}, i) in $static.allBlog.edges" :key="i">
      <h2>{{ title }}</h2>
      <p>Posted: {{ date }}</p>
      <span v-for="tag in tags" :key="tag">#{{ tag }}</span>
      <p><g-link :to="path">More</g-link></p>
    </article>
  </layout>
</template>

<script>
export default {
  name:"BlogList"
}
</script>

@formatter:off
<static-query>
query {
  allBlog {
    edges {
      node {
        title
        path
        excerpt
        date (format: "MMM.DD, YYYY")
        tags
        category
        timeToRead
        content
        keynote
        feature
        cover
        fileInfo {
          name
          directory
        }
      }
    }
  }
}
</static-query>
@formatter:on

<p><g-link :to="path">More</g-link></p>

使用 <g-link>，將 path 綁定 to prop。

Gridsome Template

templates/Blog.vue

<template>
  <layout>
    <div v-html="$page.blog.content"/>
  </layout>
</template>

<script>
export default {
  name:"Blog"
}
</script>

@formatter:off
<page-query>
query($path: String!) {
  blog(path: $path) {
    content
  }
}
</page-query>
@formatter:on

13 行

@formatter:off
<page-query>
query($path: String!) {
  blog(path: $path) {
    content
  }
}
</page-query>
@formatter:on

由於 post query 帶 variable，因此必須改用 <page-query/>。

Template 的 GraphQL 都要帶 argument，所以也可說 template 要使用 page query

第 3 行

<div v-html="$page.blog.content"/>

由於從 <static-query/> 改用 <page-query/>，因此 HTML template 也要從 $static 改成 $page。

Conclusion

  • Blog 可由 template 從 Markdown 以 HTML 顯示
  • 若 GraphQL 使用 variable，Gridsome 就要改用 page query
