Gridsome 之 Frontmatter

2020-10-31

Post 的 Meta Information 可寫在 Markdown 的 Frontmatter，Gridsome 可透過 GraphQL 加以讀取。

Version

Gridsome 0.7.21

Frontmatter

Post 除了 content 外，還多了不少其他資訊，這些都來自 markdown。

content/blog/first-post.md

---
title: First Post
excerpt: First post description
date: 2020-01-08 04:04:04
tags: ['JavaScript', 'Ramda']
category: Ramda
keynote: ./images/keynote.png
feature: ./images/feature.png
cover: ./image/cover.png
---

# First Post

This is `first post` content

在 Markdown 最上方以 --- 所框住的部分為 frontmatter，包含幾個常用的 property：

  • title：blog 的標題
  • excerpt：blog 的 overview
  • date：blog 的日期
  • tags：blog 的 tag，可多個 tag 故為 array
  • category：blog 所屬 category
  • keynote：單篇 blog 時上方所使用的 image
  • feature：設定為 featured blog 時 所使用的 background image
  • cover：首頁所使有的 top image

可視實際需求在 frontmatter 加入自訂 property，GraphQL 會自動抓到

GraphQL Playground

可發現 frontmatter 所定義的 property 在 GraphQL Schema 都可找到。

query {
  allBlog {
    edges {
      node {
        title
        excerpt
        date (format: "MMM.DD, YYYY")
        tags
        category
        timeToRead
        content
        keynote
        feature
        cover
        fileInfo {
          name
          directory
        }
      }
    }
  }
}

日期格式非常多種，因此 Gridsome 特別為 date() 提供了 format argument。

亦可由 fileInfo 取得 markdown 原本 folder 與 file 名稱。

Gridsome 底層使用了 Moment， format 的 arguemt 可參考其官網

可發現 frontmatter 可被 GraphQL 查詢。

Gridsome Page

src/pages/BlogList.vue

<template>
  <layout>
    <h1>Blog List</h1>
    <article v-for="({ node: { title, date, tags }}, i) in $static.allBlog.edges" :key="i">
      <h2>{{ title }}</h2>
      <p>Posted: {{ date }}</p>
      <span v-for="tag in tags" :key="tag">#{{ tag }} </span>
    </article>
  </layout>
</template>

<script>
export default {
  name:"BlogList"
}
</script>

直接在 v-for 從 $static.allBlog.edges destructure 出 title、date 與 tags。

由於 tags 是 array，可再使用 v-for 展開。

Conclusion

  • Frotmatter 亦為各 JAMStack 必備格式，Gridsome 可透過 GraphQL 加以讀取，非常方便
