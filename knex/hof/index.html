使用 Higher Order Function 組合 Knex Query

2019-12-07

在使用 Knex 產生 SQL 時，實務上會遇到 where() 條件重複出現，此時可透過 Higher Order Function 產生 Query，並利用 Function Composition 重複使用。

Version

macOS Catalina 10.15.1
WebStorm 2019.3
Node 13.2.0
Knex 0.20.3
Ramda 0.26.1

Simple Query

src/index.js

import { ApolloServer, gql } from 'apollo-server'
import knex from 'knex'

let mySQL = knex({
  client: 'mysql',
  connection: {
    host: 'localhost',
    port: 3306,
    user: 'root',
    password: 'demo',
    database: 'Bookshelf'
  }
})

let typeDefs = gql`
  type Query {
    books(price: Int): [Book]
  }

  type Book {
    id: Int
    title: String
    category: Category
    price: Int
  }

  enum Category {
    FP
    FRP
    JS
  }
`

let books = (_, { price }) => mySQL('books')
  .where('price','>=', price)

let resolvers = {
  Query: {
    books
  }
}

let apolloServer = new ApolloServer({ typeDefs, resolvers })

apolloServer.listen()
  .then(({ url }) => `GraphQL Server ready at ${url}`)
  .then(console.log)

34 行

let books = (_, { price }) => mySQL('books')
  .where('price','>=', price)

傳統我們使用 Knex 的 method chaining 執行 where() 部分。

Higher Order Function

import { ApolloServer, gql } from 'apollo-server'
import knex from 'knex'

let mySQL = knex({
  client: 'mysql',
  connection: {
    host: 'localhost',
    port: 3306,
    user: 'root',
    password: 'demo',
    database: 'Bookshelf'
  }
})

let typeDefs = gql`
  type Query {
    books(price: Int): [Book]
  }

  type Book {
    id: Int
    title: String
    category: Category
    price: Int
  }

  enum Category {
    FP
    FRP
    JS
  }
`

let allBooks = mySQL('books')

let filterByPrice = val => x => x.where('price', '>=', val)

let books = (_, { price }) => filterByPrice(price)(allBooks)

let resolvers = {
  Query: {
    books
  }
}

let apolloServer = new ApolloServer({ typeDefs, resolvers })

apolloServer.listen()
  .then(({ url }) => `GraphQL Server ready at ${url}`)
  .then(console.log)

若我們想讓 Knex query 能重複使用，則可改用 higher order function 產生 query。

34 行

let allBooks = mySQL('books')

allBooks() 可視為 base query，將來所有 query 都會組合 allBooks()。

36 行

let filterByPrice = val => x => x.where('price', '>=', val)

實作 filterByPrice() higher order function，傳入 price 的比較值，回傳新的 query function，其中 x 就是 query。

38 行

let books = (_, { price }) => filterByPrice(price)(allBooks)

實作 books() resolver，先傳入解構的 price 回傳新的 query，最後再傳入 allBooks() base query 組合出完整 query。

Multiple Query

34 行

let books = (_, { price, title }) => mySQL('books')
  .where('price','>=', price)
  .where('title', title)

實務上不可能只有單一 where()，而是會由多個 where() 所組成。

Function Composition

import { ApolloServer, gql } from 'apollo-server'
import knex from 'knex'
import { compose } from 'ramda'

let mySQL = knex({
  client: 'mysql',
  connection: {
    host: 'localhost',
    port: 3306,
    user: 'root',
    password: 'demo',
    database: 'Bookshelf'
  }
})

let typeDefs = gql`
  type Query {
    books(price: Int, title: String): [Book]
  }

  type Book {
    id: Int
    title: String
    category: Category
    price: Int
  }

  enum Category {
    FP
    FRP
    JS
  }
`

let allBooks = mySQL('books')

let filterByPrice = val => x => x.where('price', '>=', val)

let filterByTitle = val => x => x.where('title', val)

let books = (_, { price, title }) => compose(
  filterByTitle(title),
  filterByPrice(price),
)(allBooks)

let resolvers = {
  Query: {
    books
  }
}

let apolloServer = new ApolloServer({ typeDefs, resolvers })

apolloServer.listen()
  .then(({ url }) => `GraphQL Server ready at ${url}`)
  .then(console.log)

37 行

let filterByPrice = val => x => x.where('price', '>=', val)

let filterByTitle = val => x => x.where('title', val)

對於多個 query，可將每個 where() 都寫成 higher order function，都回傳新的 query function。

41 行

let books = (_, { price, title }) => compose(
  filterByTitle(title),
  filterByPrice(price),
)(allBooks)

使用 compose() 組合所有 query function 成為新的 query 後，最後再傳入 allBooks base query 執行完整 query 。

Conclusion

  • 由於每個 where() 都可獨立以 higher order function 呈現，因此可重複使用，只要視需求組合即可，非常靈活

Reference

Andy Peterson, Using Higher-Order Functions to Build Queries in Knex.js
