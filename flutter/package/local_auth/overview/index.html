local_auth 偵測使用生物辨識的可用性

2024-10-15

生物辨識登入 已經成為手機登入的標配，local_auth 可實現 Android、iOS、macOS 與 Windows 的 跨平台 生物辨識登入。

Version

Flutter 3.24
local_auth 3.2

local_auth

  • 安裝 local_auth package

Android

Activity

android/app/src/main/java/com/example/MainActivity.java

  • import io.flutter.embedding.android.FlutterFragmentActivity
  • MainActivity 改繼承 FlutterFragmentActivity

android/app/src/main/kotlin/com/example/flutter_lab/MainActivity.kotlin

  • import io.flutter.embedding.android.FlutterFragmentActivity
  • MainActivity 改繼承 FlutterFragmentActivity

Permission

android/app/src/main/AndroidManifest.xml

  • 在 <manifest> 下新增 <uses-permission> 啟用 USE_BIOMETRIC

iOS

ios/Runner/Info.plist

  • 在 <dict> 下新增 <key> 與 <string>

Flutter

local_auth

Line 16

  • _auth：建立 local_auth object

Line 17

  • _isAuthAvailable：顯示是否有 可用 的生物辨識

Line 19

  • 偵測是否有可用的生物辨識

Line 24

  • auth.canCheckBiometrics：硬體 是否支援生物辨識

Line 25

  • auth.isDeviceSupported：用戶 是否有設定生物辨識

Line 33

  • 當 canCheckBiometrics 與 isDeviceSupported 都為 true 時，才表示能啟動生物辨識登入

Line 55

  • 顯示偵測結果

Line 41

  • 將偵測結果寫入 _isAuthAvailable state

Conclusion

  • canCheckBiometrics 與 isDeviceSupported 單從字面很難清楚知道其背後意義，簡單說 canCheckBiometrics 是判斷硬體是否 有支援，而 isDeviceSupported 試判斷 用戶是否 已設定
  • 當 canCheckBiometrics 與 isDeviceSupported 都為 true 時，才能成功使用生物辨識登入
