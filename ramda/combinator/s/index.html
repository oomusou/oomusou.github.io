使用 ap() 實現 S Combinator

2020-11-20

Binary Function 若第二個 Argument 須先經過其他 Function 運算再傳入，此為 S Combinator，可使用 Ramda 的 ap() 實現。

Version

Ramda 0.27.1

Imperative

let data = 10

let f = x => x + 3 * x

f(data) // ?

若使用 operator，會直接使用 + 與 x operator。

Functional

import { add, multiply } from 'ramda'

let data = 10

let f = x => add(x, multiply(3, x))

f(data) // ?

若將 operator 改成 function，會發現 add() 第二個 argument 須先經過 multiply() 運算過才傳入。

ap()

import { add, multiply, ap } from 'ramda'

let data = 10

let f = ap(add, multiply(3))

f(data) // ?

這種第二個 argument 須經過運算後再傳入的 pattern，就是所謂的 S Combinator，其中 S 為 S equence。

S(f)(g)(x) = f(x)(g(x))

而 ap 正是 S Combinator 的實現。

ap()
(x → a → b) → (x → a) → x → a
實現 S Combinator

x -> a -> b：第一個 argument 為 binary function

x -> a：第二個 argument 為 unary function

x：為 data

a：回傳結果

chain()

chain(f, g)(x) => f(g(x))(x)

眼尖的人應該會發現 ap() 與 chain() 非常接近：

  • ap(f)(g) = f(x, g(x))
  • chain(f)(g) = f(g(x), x)

ap() 在第二個 argument 先經過其他 function 運算，而 chain() 則在第一個 argument 先經過其他 function 運算。

Conclusion

  • Binary function 若只有一個 argument 經過 function 運算，可使用 ap() 或 chain() 組合
  • Binary function 若兩個 argument 都需經過 function 運算，可使用 converge() 組合
