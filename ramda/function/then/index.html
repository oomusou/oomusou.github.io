使用 then() 處理 Fulfilled Promise

2019-09-17

ECMAScript 2015 的最大亮點之一就是提出 Promise 這種 未來值 概念避免 Callback Hell，但仍看到很多人以 Callback 使用 Promise，事實上可用更 Functional 方式處理。

Version

macOS Mojave 10.14.6
VS Code 1.38.1
Quokka 1.0.243
ECMAScript 2015

Promise.resolve()

let fetchData = () => Promise.resolve({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData(); // ?

若 data 是 synchronous，可直接回傳沒問題，但若 data 是 asynchronous，則必須透過 Promise.resolve() 將 data 包成 promise 後回傳。

若直接讀取 fetchData()，會發現顯示多顯示了 then，表示其資料為 promise。

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData(); // ?

也可使用 ES2017 的 async 修飾 arrow function，表示回傳 promise，相當於 Promise.resolve() 的 syntatic sugar。

then()

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData().then(x => console.log(x));

該如何取得不包含 then 的資料呢 ? 要使用 promise 自帶的 then()，由其 callback 的 x 取得內部 synchronous 資料。

如此資料就不再包含 then 了。

Object Property

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData().then(x => console.log(x.data));

若要取得 data property 下的資料呢 ? 直覺會使用 x.data 取得。

但其實 x => console.log(x.data) 包含了兩件事情：

  1. 從 x.data 取得資料
  2. 使用 console.log() 印出

基於單一職責原則 (SRP)，我們會希望 callback 只做一件事情，因次較好方式是使用兩次 then()。

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData()
  .then(x => x.data)
  .then(x => console.log(x));

第 10 行

.then(x => x.data)

回傳仍是 promise，只是其內部資料改成 x.data，相當於完成從 x.data 取得資料。

.then(x => console.log(x));

由於之前回傳是 promise，因此可繼續使用 then() 抓到 x.data，最後使用 console.log() 印出。

map()

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData()
  .then(x => x.data)
  .then(x => {
    let z = x.map(y => `${y.title} : ${y.price}`).join('\n');
    console.log(z);
  });

實務上常會將 promise 內的資料透過 map() 整理成我們要的格式，直覺會將 map() 寫在 then() 的 callback 內。

這樣寫雖然可行，但有幾個問題：

  1. 在 then() 的 callback 寫 map()，勢必引出 map() 的 callback 內新變數 y，雖然名稱取好一點能稍微解決問題，但已經發現變數滿天飛問題
  2. console.log() 屬 side effect，而 then() 的 callback 應維持 pure function，因此不適合在 then() 使用 console.log()
  3. 我們希望依照 price 排序，但目前並沒有

先解決 price 排序問題。

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

fetchData()
  .then(x => x.data)
  .then(x => {
    let z = x.sort((a, b) => a.price - b.price)
    .map(y => `${y.title} : ${y.price}`).join('\n');
    console.log(z);
  });

使用 Array.prototype.sort() 對 price() 排序之後再使用 map()。

price 排序問題解決了，但會發現 sort() 、 map() 與 join() 的 method chaining 也沒使 code 可讀性變高，稍後再來解決此問題。

接下來解決 console.log() 的 side effect。

No Side Effect

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

let fn = () => fetchData()
  .then(x => x.data)
  .then(x => x.sort((a, b) => a.price - b.price)
              .map(y => `${y.title} : ${y.price}`).join('\n'));

fn().then(x => console.log(x));

理想上在處理 promise 的 then() callback 內，應該只是處理 data 的 pure function，不能有 side effect，因此不適合寫 console.log()。

因此另外建立 fn() 只用 then() 處理 data，至於 side effect 則最後由 user 處理。

不要怕建立 function，imperative 都可以任意建立 variable，為什麼 functional 不能任意建立 function 呢 ?

 let fetchData = async() => ({
   data: [
     { title: 'FP in JavaScript', price: 200 },
     { title: 'RxJS in Action', price: 100 },
     { title: 'Speaking JavaScript', price: 300 }   
   ]
 });

 let fn = () => fetchData()
   .then(x => x.data)
   .then(x => x.sort((a, b) => a.price - b.price))
   .then(x => x.map(y => `${y.title} : ${y.price}`))
   .then(x => x.join('\n'));

 fn().then(x => console.log(x));

再回到 sort()、map() 與 join() 的 method chaining，都寫在同一個 callback 內有幾個問題：

  1. 違反單一職責原則 (SRP)，同一個 callback 同時處理 sort()、map() 與 join()
  2. Method chaing 過長時，可讀性也沒很高，且還寫在 callback 內增加縮排

比較好的方式是將 sort()、map() 與 join() 分擔在每個 then() 的 callback 下，讓每個 callback 一次只做一件事情。

Point-free

 import { prop, sortBy, map, join, converge } from 'ramda';

 let fetchData = async() => ({
   data: [
     { title: 'FP in JavaScript', price: 200 },
     { title: 'RxJS in Action', price: 100 },
     { title: 'Speaking JavaScript', price: 300 }   
   ]
 });

 let toSingle = converge(
   (str1, str2) => `${str1} : ${str2}`, [
     prop('title'),
     prop('price')
   ]
 );

 let fn = () => fetchData()
   .then(prop('data'))
   .then(sortBy(prop('price')))
   .then(map(toSingle))
   .then(join('\n'));

 fn().then(console.log);

使用 Ramda 將所有的 callback 都 point-free。

Pipeline

import { prop, sortBy, map, join, converge, pipe, then } from 'ramda';

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

let toSingle = converge(
  (str1, str2) => `${str1} : ${str2}`, [
    prop('title'),
    prop('price')
  ]
);

let fn = pipe(
  fetchData,
  then(prop('data')),
  then(sortBy(prop('price'))),
  then(map(toSingle)),
  then(join('\n'))
);

fn().then(console.log);

在使用 promise chain 的 then() 時，是不是已經感覺有 pipeline 味道呢 ?

Ramda 的 then() 就是 Promise.prototype.then() 封裝，且回傳 promise，因此可用 pipe() 將所有回傳 promise 的 function 加以 pipeline 整合成 point-free function。

then()
(a → b) → (Promise e a) → (Promise e b)
Promise.prototype.then() 的 function 版本

(a -> b)：synchronous function

Promise e a：data 為 promise

Promise e b：回傳亦為 promise

Function Composition

import { prop, sortBy, map, join, converge, compose, then } from 'ramda';

let fetchData = async() => ({
  data: [
    { title: 'FP in JavaScript', price: 200 },
    { title: 'RxJS in Action', price: 100 },
    { title: 'Speaking JavaScript', price: 300 }   
  ]
});

let toSingle = converge(
  (str1, str2) => `${str1} : ${str2}`, [
    prop('title'),
    prop('price')
  ]
);

let fn = compose(
  then(join('\n')),
  then(map(toSingle)),
  then(sortBy(prop('price'))),
  then(prop('data')),
  fetchData
);

fn().then(console.log);

既然能使用 pipe() 達成 pipeline，反向使用 compose() 就是 function composition 了。

Conclusion

  • 使用 promise 的 then() 時，應謹記每個 callback 只做一件事情，且為 pure function 不應該有 side effect
  • Side effect 實務上無法避免，但不應在 then() 的 callback 處理，而是交給 function 的呼叫者處理
  • 由於 promise chain 具有 pipeline 本質，因此很適合以 function composition 呈現
