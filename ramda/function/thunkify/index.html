使用 thunkify() 使 Function 多墊一層 Unit

2019-09-29

thunkify() 在 Ramda 算罕用的 Function，但在使 Promise Chain 成為 Pipeline 且 Point-free 時，就會需要 thunkify()。

Version

macOS Mojave 10.14.6
VS Code 1.38.1
Quokka 1.0.253
Ramda 0.26.1
Wink-fp 0.1.1

thunkify()

import { thunkify } from 'ramda';

// add :: (Number, Number) -> Number
let add = (a, b) => a + b;

// fn :: (Number, Number) -> Unit -> Number
let fn = thunkify(add);

fn(1, 2)(); // ?

原本 add() 的 signature 為 (Number, Number) -> Number，當使用 Ramda 的 thunkify() 後，會變成 (Number, Number) -> Unit -> Number，很類似 Number always() 成為 Number -> (Unit -> Number)。

thunkify()
(a, b, ..., j) -> k) -> (a, b, ..., j) -> () -> k)

將原本 function 多墊一層 Unit

(a, b, ..., j) -> k)：原 function

(a, b, ..., j) -> () -> k)：signature 都不變，但多墊了一層 Unit

可以簡單想成 thunkify() 為 alway() 的 function 版本

Imperative

let resolveAfter = ms => val => new Promise(
  resolve => setTimeout(() => resolve(val), ms)
);

await resolveAfter(300)(2); // ?

resolveAfter() 欲延遲一段時間才產生 promise，標準做法是使用 setTimeout() 延遲一段時間，然後在其 callback 中呼叫 promise constructor 的 resolve()。

Pipeline

import { pipe, then, always } from 'ramda';
import { sleep, resolve } from 'wink-fp';

let resolveAfter = ms => val => pipe(
  sleep,
  then(always(val)),
  then(resolve)
)(ms);

await resolveAfter(300)(2); // ?

但 setTimeout() 畢竟是 callback-based function，在 ES6 後我們有了 Wink-fp 的 sleep()，其為 promised-based function。

既然 sleep() 回傳是 promise，就能使用 Ramda 的 then() 加以 pipeline。

Point-free

import { useWith, flip, then, thunkify } from 'ramda';
import { sleep, resolve } from 'wink-fp';

let resolveAfter = useWith(
  flip(then), 
  [sleep, thunkify(resolve)]
);

await resolveAfter(300)(2); // ?

能進一步將 resolveAfter() 的 ms 與 val point-free 嗎 ?

其實仔細觀察，會發現最後執行的 function 為 then()，sleep() 與 resolve() 會分階段吸收 argument，頗有 useWith() 味道。

但 then() 第一個 argument 為 function，第二個 argument 為 promise，但 resolveAfter() 第一個 argument 為 sleep() 所需的 ms，故必須先 flip(then) 顛倒其 signature。

resolve() 所接的是 always，並不是簡單的 data，故需使用 thunkify(resolve) 將 resolve() 多墊一層 Unit。

Conclusion

  • 由於 then() 後面都要接 function，pipeline 時會需要接 always()，此時 point-free 時就會用到 thunkify()
