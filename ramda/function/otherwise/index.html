使用 otherwise() 處理 Rejected Promise

2019-09-18

ECMAScript 2015 提供了 catch() 處理 Rejected Promise，而 2017 更提供了 try catch 直接在 catch Block 處理；事實上 Ramda 也提供了 otherwise()，讓我們以更 Functional 方式處理 Rejected Promise。

Version

macOS Mojave 10.14.6
VS Code 1.38.1
Quokka 1.0.244
Ramda 0.26.1
Wink-fp 0.0.10

Promise.resolve()

then()

let fetchData = () => Promise.resolve('Hello World');

fetchData().then(x => console.log(x));

若使用 Promise.resolve() 回傳 fulfilled promise，我們可用 then() 取得內部資料。

Await

let fetchData = () => Promise.resolve('Hello World');

let x = await fetchData();
console.log(x);

亦可使用 ES2017 的 await 處理 fulfilled promise。

Promise.reject()

then()

let fetchData = () => Promise.reject(new Error('Something wrong'));

fetchData().then(x => console.log(x));

若使用 Promise.reject() 回傳 rejected promise，則 then() 的 callback 將不會被執行。

紅色的 Something wrong 並非 console.log() 顯示，只是 Quokka 貼心地顯示 rejected promise 未處理，若在 browser 或 Node 則是 runtime 錯誤。

let fetchData = () => Promise.reject(new Error('Something wrong'));

fetchData().then(
  x => console.log(x), 
  e => console.log(e.message)
);

若要使用 then()，正確寫法要提供第二個 callback，專門負責處理 rejected promise。

藍色的 Something wrong 才是 console.log() 所顯示。

catch()

let fetchData = () => Promise.reject(new Error('Something wrong'));

fetchData()
  .then(x => console.log(x))
  .catch(e => console.log(e.message));

ES6 另外提供了 catch() 可專門處理 rejected promise。

如此 then() 只要專心處理 fulfilled promise 即可，更符合 SRP，可讀性更高。

.then(undefined, callback) 相當於 .catch(callback)

Try Catch

let fetchData = () => Promise.reject(new Error('Something wrong'));

try {
  let x = await fetchData();
  console.log(x);
}
catch (e) {
  console.log(e);
}

ES2017 的 await 可搭配傳統的 try catch ，如此處理 fulfilled promise 寫在 try block，而處理 rejected promise 寫在 catch block，這種寫法與傳統 imperative 與 synchronous 相近。

try catch 只是 .catch() 的 syntatic sugar 而已

Ramda

import { then, otherwise, pipe } from 'ramda';
import { reject, log } from 'wink-fp';

let fetchData = () => reject('Something wrong');

let fn = pipe(
  fetchData,
  then(log),
  otherwise(log)
);

fn();

Ramda 已經提供 then() 讓我們處理 fulfilled promise，同理也提供了 otherwise() 處理 rejected promise。

otherwise()
(a → b) → (Promise e a) → (Promise e b)
Promise.prototype.catch() 的 function 版本

(a -> b)：synchronous function

Promise e a：data 為 promise

Promise e b：回傳亦為 promise

import { then, otherwise, compose } from 'ramda';
import { reject, log } from 'wink-fp';

let fetchData = () => reject('Something wrong');

let fn = compose(
  otherwise(log),
  then(log),
  fetchData,
);

fn();

既然能使用 pipe() 達成 pipeline，反向使用 compose() 就是 function composition 了。

Rejected in Fulfilled

then()

let fetchData = () => Promise.resolve('Hello World');

fetchData().then(
  x => {
    console.log(x);
    return Promise.reject('Something wrong');
  }).then(x => console.log(x));

比較複雜的是若在 then() 的 callback 中回傳 rejected promise 時，若下一個 promise chain 的 then() 只提供一個 callback，一樣無法處理剛產生的 rejected promise。

Somethig wrong 無法被 console.log() 顯示，紅色的 Something worng 是 Quokka 所顯示。

let fetchData = () => Promise.resolve('Hello World');

fetchData().then(
  x => {
    console.log(x);
    return Promise.reject('Something wrong');
  },
  e => console.log(e),
).then(
  x => console.log(x),
  e => console.log(e)
);

若要正確顯示 Something worng，則第二個 then() 也要提供第二個 callback。

同理若要擔心第一個 rejected promise，第一個 then() 也必須提供第二個 callback。

可發現若要完整處理 rejected promise，則每個 then() 都要提供 rejected handler，如此就違反 DRY

catch()

let fetchData = () => Promise.resolve('Hello World');

fetchData().then(
  x => {
    console.log(x);
    return Promise.reject('Something wrong');
  }
).then(x => console.log(x)
).catch(e => console.log(e));

若改用 catch()，則 rejected handler 只要寫一次即可，就可處理所有 rejected promise，也符合 functional 風格。

Try Catch

let fetchData = () => Promise.resolve('Hello World');

try {
  let x = await fetchData();
  console.log(x);
  throw('Something Wrong')
}
catch(e) {
  console.log(e);
}

若使用 ES2017 的 await，則 rejected handler 一樣只要寫一次即可，也符合原本 imperative 與 synchronous 習慣。

Ramda

import { pipe, then, otherwise } from 'ramda';

let fetchData = () => Promise.resolve('Hello World');

let fn = pipe(
  fetchData,
  then(x => {
    console.log(x);
    return Promise.reject('Something wrong');
  }),
  otherwise(x => console.log(x))
);


fn();

若在 then() 回傳 rejected promise，則 otherwise() 也能攔截得到 Wrong again，與 catch() 及 try catch 一樣只需一個 rejected handler 即可。

import { pipe, then, otherwise, always } from 'ramda';
import { log, reject, resolve } from 'wink-fp';

let fetchData = () => resolve('Hello World');

let onFulfilled = pipe(
  log,
  always('Something wrong'),
  reject
);

let fn = pipe(
  fetchData,
  then(onFulfilled),
  otherwise(log)
);

fn();

既然使用了 otherwise()，能讓整個寫法更 FP 嗎 ?

第 6 行

let onFulfilled = pipe(
  log,
  always('Something wrong'),
  reject
);

使用了 Wink-fp 的 reject()，此為 Promise.reject() 的 function 封裝；而 log() 為 console.log() 的 function 封裝。

如此 fn() 與 onFulfilled() 都以 pipeline 產生。

import { compose, then, otherwise, always, pipe } from 'ramda';
import { log, reject, resolve } from 'wink-fp';

let fetchData = () => resolve('Hello World');

let onFulfilled = compose(
  reject,
  always('Something wrong'),
  log
);

let fn = compose(
  otherwise(log),
  then(onFulfilled),
  fetchData,
);

fn();

既然能使用 pipe() 達成 pipeline，反向使用 compose() 就是 function composition 了。

Conclusion

  • then() 必須每個 promise 都提供 rejected handler 才能完整處理 rejected promise，違反 DRY
  • 若使用 catch() 只需寫一個 rejected handler 即可，也符合 funtional 的 pipeline 習慣
  • try catch 也只需寫一個 rejected handler 即可，且可繼續使用 imperative 思維
  • otherwise() 也只需寫一個 rejected handler 即可，由於其 functional 特質，可使用 function composition
