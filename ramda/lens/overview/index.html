使用 Lens 實現 Immutable

2020-02-18

Lens 為 FP 實現 Immutable 的一種方式，其本質是一組 Getter 與 Setter，當透過 Lens 修改 Data 時，將不會修改原本資料，而是回傳一組全新資料。

Version

macOS Catalina 10.15.2
VS Code 1.41.1
Quokka 1.0.276
Ramda 0.27.0

Object

lens()

import { lens, view } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let getTitle = x => x.title
let setTitle = (y, x) => ({...x, title: y })

let titleLens = lens(getTitle)(setTitle)

Lens 可視為特定 property 的 getter 與 setter 的 pair，之後我們可拿這組 pair 對該 property 實現 immutable。

第 8 行

let getTitle = x => x.title

對 title property 提供 getter。

第 9 行

let setTitle = (y, x) => ({...x, title: y })

對 title property 提供 setter，使用了 ES6 的 spread operator 實現 immutable。

11 行

let titleLens = lens(getTitle)(setTitle)

使用 lens() 建立 lens，傳入剛建立的 getter 與 setter。

lens()
(s → a) → ((a, s) → s) → Lens s a
Lens s a = Functor f => (a → f a) → s → f s
自行提供 getter 與 setter 建立 lens

(s -> a)：getter

((a, s) → s)：setter

Lens s a：回傳 lens

lens() 也適用於 functor，稍後會說明

prop()、assoc()

import { lens, prop, assoc } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lens(prop('title'))(assoc('title'))

實務上亦常看到直接使用 prop() 與 assoc() 建立 getter 與 setter。

若是 nested object 則使用 path() 與 assocPath()

lensProp()

import { lensProp } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lensProp('title')

Ramda 提供了更精簡方式，只要透過 lensProp() 就可一步建立 lens，它會自動幫我們建立相對應 getter 與 setter，實務上都直接使用 lensProp()，甚少使用 lens()。

lensProp()
String → Lens s a
Lens s a = Functor f => (a → f a) → s → f s
提供 property 名稱對 object 建立 lens

Nested object 則使用 lensPath() 。

view()

import { lensProp, view } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lensProp('title')

view(titleLens)(data) // ?

由於 lens 就是 getter / setter，可使用 view() 藉由 lens 讀取指定 property。

view()
Lens s a → s → a
Lens s a = Functor f => (a → f a) → s → f s
使用 lens 讀取 object 指定 property

Pipeline

import { lensProp, view, toUpper, pipe } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lensProp('title')

let f = pipe(
  view(titleLens),
  toUpper
)

f(data) // ?

Q：view() 透過 lens 讀取資料有什麼好處呢 ?

由於 view() 是 curried function，因此很方便與其他 function 組合。

set()

import { lensProp, set } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lensProp('title')

set(titleLens)('Speaking JavaScript')(data) // ?

可使用 set() 藉由 lens 修改指定 property。

set() 最大優點是 immutable，它不會修改原本 object，而是回傳新的修改過 object。

set()
Lens s a → a → s → s
Lens s a = Functor f => (a → f a) → s → f s
使用 lens 寫入 object 指定 property

over()

import { lensProp, over, toUpper } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: 100
}

let titleLens = lensProp('title')

over(titleLens)(toUpper)(data) // ?

若要更靈活以 function 修改 lens 所指定的 property，則要使用 over()。

over()
Lens s a → (a → a) → s → s
Lens s a = Functor f => (a → f a) → s → f s
透過 function 使用 lens 寫入 object 指定 property

Array

lensIndex()

import { lensIndex, over, inc } from 'ramda'

let data = [1, 2, 3]

let item1Lens = lensIndex(1)

over(item1Lens)(inc)(data) // ?

由上可知 lens 的 fuction 都支援 functor，因此可透過 lensIndex() 對 array 指定 index 建立 lens。

lensIndex()
Number → Lens s a
Lens s a = Functor f => (a → f a) → s → f s
提供 index 對 array 建立 lens

lensPath()

import { lensPath, over, toUpper } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let titleLens = lensPath([0, 'title'])

over(titleLens)(toUpper)(data) // ?

Array 內放 object 是 ECMAScript 常見的應用，透過 lensPath() 可同時應付 array 與 object。

Lens Composition

Object

import { lensProp, over, inc, compose } from 'ramda'

let data = {
  title: 'FP in JavaScript',
  price: {
    Amazon: 100,
    eBay: 200
  }
}

let priceLens = lensProp('price')
let amazonLens = lensProp('Amazon')
let amazonPriceLens = compose(
  priceLens,
  amazonLens
)

over(amazonPriceLens)(inc)(data) // ?

對於 nested object，除了使用 lensPath() 外，也可為 property 各自建立 lens，然後用 compose() 組合起來。

若由 object 由外而內 角度，在 compose() lens 是 由左至右，與一般使用 compose() function 的 由右而左 不同，這點需注意

Array

import { lensProp, lensIndex, over, toUpper, compose } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let item1Lens = lensIndex(0)
let titleLens = lensProp('title')

let item1TitleLens = compose(
  item1Lens,
  titleLens
)

over(item1TitleLens)(toUpper)(data) // ?

compose() 也能用於 array 與 object 的組合。

Conclusion

  • Lens 除了支援 immutable 外，若 nested object 本身結構很複雜，可只將 lens 暴露於外界，呼叫端只要透過 lens 就能讀寫 object，而不需了解其內部結構
  • view()、set() 與 over() 都是 curried function，方便使用 function composition
