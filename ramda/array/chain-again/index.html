重新認識 chain()

2020-03-10

Ramda 的 chain() 是很神奇的 Function，很像 map() 但又不是map()，可以用在 Array 也可用在 Function，實務上常常都可重構成 chain()，但卻不容易發現。

Version

macOS Catalina 10.15.3
VS Code 1.43.0
Quokka 1.0.282
Ramda 0.27.0

chain()

chain()
Chain m => (a → m b) → m a → m b

chain() 所支援的 container 並不是 functor，而是很獨特的 chain()。

map()
Functor f => (a → b) → f a → f b

對比 map() 是不是非常類似呢 ?

差異僅在第一個 argument，chain() 是 a -> m b，而 map() 是 a -> b。

bind()

在 FP 裡，其實 signature 為 (a → m b) → m a → m b 稱為 bind()，但因為 ECMAScript 已經有 bind()，且 Ramda 也提供 function 版本的 bind()，因此原本 FP 的 bind() 在 Ramda 只好使用 chain()。

Monad

根據定義，monad 必須支援 return() 與 bind()。

  • return()：a -> m b
  • bind()：(a -> m b) -> m a -> m b

可發現 bind() 與 map() 差異是 bind() 直接傳入 return()，而 map() 只傳入 a -> b projection function。

但無論如何，傳入的 f() 都將直接改變 m a 內部資料。

若以 function 角度，傳入 a -> b 使的 m a 變成 m b ，這很容易理解。

但若傳入 a -> m b，理論上應該成為 m m b，但為什麼結果還是 m b 呢 ?

可推論 m a 內部除了 map() 外，還組合了 flatten()，才可能使 m m b 變成 m b。

這也是為什麼當 chain() 用在 array 時相當於 ECMAScript 的 flatMap()，或者相當於 compose(flatten, map)。

Conclusion

  • chain() 用於 array 時俗稱 flatMap()，本文以 map() 與 bind() 的角度，也可推論出 chain() 內部就是 flatMap()
