使用 assoc() 對 Object 新增 Property

2020-10-02

若想對 Object 新增 Property 且回傳新 Object，Ramda 提供了 assoc()。

Version

Ramda 0.27.1

Object

Imperative

let data = {
  title: 'FP in JavaScript'
}

// assoc :: String -> a -> Object -> Object
let assoc = k => v => o => {
  let result = {}

  for (let x in o)
    result[x] = o[x]

  result[k] = v

  return result
}

assoc('price')(100)(data) // ?

Imperative 會先建立 result Object，使用 for loop clone Object，然後對 result Object 新增 property。

Object.assign()

let data = {
  title: 'FP in JavaScript'
}

// assoc :: String -> a -> Object -> Object
let assoc = k => v => o => {
  let result = Object.assign({}, o)
  result[k] = v

  return result
}

assoc('price')(100)(data) // ?

ES 5 可使用 Object.assign() 實現 clone Object。

Spread Operator

let data = {
  title: 'FP in JavaScript'
}

// assoc :: String -> a -> Object -> Object
let assoc = k => v => o => ({...o, k: v})

assoc('price')(100)(data) // ?

ES6 可使用 spread operator 展開 Object 來 clone Object。

Ramda

import { assoc } from 'ramda'

let data = {
  title: 'FP in JavaScript'
}

assoc('price')(100)(data) // ?

Ramda 已內建 assoc() 可直接使用。

assoc()
String -> a -> {k: v} -> {k: v}
對 Object 新增 property，並回傳全新 Object

String : 新 property 的 key

a：新 property 的 value

{k: v}：data 為 Object

{k: v} ：回傳新 Object

若 property 已存在將直接覆蓋之

Array

Imperative

let data = [
  { title: 'FP in JavaScript' },
  { title: 'RxJS in Action' },
  { title: 'Speaking JavaScript' }
]

// f :: a -> [Object] -> [Object]
let f = v => a => {
  let result = []

  for (let x of a)
    result.push({ ...x, price: v })

  return result
}

f(100)(data) // ?

Imperative 會使用 for loop，先建立新的 result Array，使用 ... spread operator 保留原本 Object，並加上新的 price property，最後 push 進 result 回傳。

Array.prototype.map()

let data = [
  { title: 'FP in JavaScript' },
  { title: 'RxJS in Action' },
  { title: 'Speaking JavaScript' }
]

// f :: a -> [Object] -> [Object]
let f = v => arr => arr.map(x => ({ ...x, price: v }))

f(100)(data) // ?

也可使用 ECMAScript 內建的 map()，一樣使用 ... spread operator 保留原本 Object，並加上新的 price property 回傳。

Ramda

import { map, assoc } from 'ramda'

let data = [
  { title: 'FP in JavaScript' },
  { title: 'RxJS in Action' },
  { title: 'Speaking JavaScript' }
]

// f :: a -> [Object] -> [Object]
let f = v => map(assoc('price', v))

f(100)(data) // ?

也可使用 Ramda 的 map() 與 assoc()。

Point-free

import { pipe, map, assoc } from 'ramda'

let data = [
  { title: 'FP in JavaScript' },
  { title: 'RxJS in Action' },
  { title: 'Speaking JavaScript' }
]

// f :: a -> [Object] -> [Object]
let f = pipe(
  assoc('price'), 
  map
)

f(100)(data) // ?

也可使用 pipe() 將 map() 與 assoc() 組合起來使其 point-free。

Conclusion

  • assoc() 只是簡單的新增 property， 由於 Ramda 是 pure function，所以會回傳新 Object，而不是直接對原 Object 直接新增 property
  • assoc() 主要是新增 property，但若 property 已存在，將直接覆蓋之，因此也可用來只修改特定 property
