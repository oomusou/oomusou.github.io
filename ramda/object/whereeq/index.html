使用 whereEq() 直接產生 Object 的 Predicate

2019-09-28

Ramda 的 where() 可產生多個以 && 串起來的 Predicate，但若針對 Object，且都是 === 的 Predicate，就可以使用 whereEq() 更為精簡。

Version

macOS Mojave 10.14.6
VS Code 1.38.1
Quokka 1.0.253
Ramda 0.26.1

filter()

import { filter } from 'ramda';

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
];

let item = { title: 'RxJS in Action', price: 200 };

let fn = obj => filter(
  x => x.title === obj.title && x.price === obj.price
);

fn(item)(data); // ?

想要找出 title 等於 RxJS in Action 且 year 等於 2017 的 data，且條件以 object 方式呈現。

11 行

let fn = obj => filter(
  x => x.title === obj.title && x.price === obj.price
);

使用了 Ramda 的 filter() 並搭配 predicate，由於包含了兩個條件，因此必須使用 && 串起來。

where()

import { filter, where, equals, prop } from 'ramda';

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
];

let item = { title: 'RxJS in Action', price: 200 };

let fn = obj => filter(where({
  title: equals(prop('title', obj)),
  price: equals(prop('price', obj))
}));

fn(item)(data); // ?

既然 predicate 是多條件，最標準的寫法就是使用 where()，再搭配 equals() 與 prop()。

whereEq()

import { filter,whereEq, compose } from 'ramda';

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
];

let item = { title: 'RxJS in Action', price: 200 };

let fn = obj => filter(whereEq(obj));

fn(item)(data); // ?

使用 where() 雖然可行，但還是略顯麻煩，Ramda 特別提供了 whereEq()。

whereEq()
{String: *} → {String: *} → Boolean
比對兩個 object 的 property 是否相等

{String: *}：Spec object

{String: *}：Test object

Boolean：若相等傳回 true，否則傳回 false

compose()

import { filter,whereEq, compose } from 'ramda';

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
];

let item = { title: 'RxJS in Action', price: 200 };

let fn = compose(
  filter,
  whereEq
);

fn(item)(data); // ?

11 行

let fn = compose(
  filter,
  whereEq
);

先執行 whereEq()，再將結果傳給 filter() 執行，因此也可以將 whereEq() 與 filter() 兩個 function 加以組合。

Conclusion

  • 當 predicate 有多個條件，可使用 where()
  • 當 predicate 有多個條件，且資料來自於 object，並且只要判斷 === 而已，則可使用 whereEq()
