使用 whereEq() 直接接受 Object 產生 Predicate

2020-04-13

Ramda 的 where() 可產生多個以 && 串起來的 Predicate，但若針對 Object，且都是 === 的 Predicate，就可以使用 whereEq() 更為精簡。

Version

macOS Catalina 10.15.3
VS Code 1.42.1
Quokka 1.0.280
Ramda 0.27.0

Imperative

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => a => {
  let result = []

  for (let x of a)
    if (x.title === o.title && x.price === o.price)
      result.push(x)

  return result
}

f(spec)(data) // ?

第 1 行

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

想找出 data 中包含 spec 的資料，注意 spec 為 object。

13 行

if (x.title === o.title && x.price === o.price)

由 spec 為 object，所以必須使用 x.title 與 obj.title 這種方式比較。

reduce()

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => a => a.reduce((a, x) => 
  (x.title === o.title && x.price === o.price) ?
  [...a, x] :
  a
, [])

f(spec)(data) // ?

只要能使用 for loop，就能使用 reduce() 改寫。

filter()

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => a => a.filter(
  x => x.title === o.title && x.price === o.price
)

f(spec)(data) // ?

ECMAScript 可使用 filter() 並傳入判斷 title 與 price 的 predicate。

Ramda

import { filter } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => filter(
  x => x.title === o.title && x.price === o.price
)

f(spec)(data) // ?

也可使用 Ramda 的 filter() 使 f() 能 point-free。

where()

import { filter, where, equals, prop } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => filter(where({
  title: equals(prop('title', o)),
  price: equals(prop('price', o))
}))

f(spec)(data) // ?

既然 predicate 是多條件，最標準寫法就是使用 where()，再搭配 equals() 與 prop()。

whereEq()

import { filter, whereEq } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = o => filter(whereEq(o))

f(spec)(data) // ?

使用 where() 雖然可行，但還是略顯麻煩，Ramda 特別提供了 whereEq()。

whereEq()
{String: *} → {String: *} → Boolean
比對兩個 object 的 property 是否相等

{String: *}：Spec object

{String: *}：Test object

Boolean：若相等傳回 true，否則傳回 false

compose()

import { filter, whereEq, compose } from 'ramda'

let data = [
  { title: 'FP in JavaScript', price: 100 },
  { title: 'RxJS in Action', price: 200 },
  { title: 'Speaking JavaScript', price: 300 }
]

let spec = { title: 'Speaking JavaScript', price: 300 }

let f = compose(filter, whereEq)

f(spec)(data) // ?

也可使用 compose() 使 f() 完全 point-free。

Conclusion

  • 當 predicate 有多個條件，可使用 where()
  • 當 predicate 有多個條件且資料來自於 object，並且只要判斷 === 而已，則可使用 whereEq()
