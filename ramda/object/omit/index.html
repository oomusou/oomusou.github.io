使用 omit() 忽略 Object 部分 Property

2020-03-03

若要抽取 Object 部分 Property，Ramda 提供了 pick()；但若要抽取大部分 Property，可從反面思考，僅 忽略 部分的 Property，這就是 omit()。

Outline

[TOC]

Version

macOS Mojave 10.15.3
VS Code 1.42.1
Quokka 1.0.277

pick()

import { map, pick } from 'ramda'

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
]

let f = map(pick(['title', 'price']))

f(data) // ?

由 pick() 傳入想要抽取的 property array，會回傳 map() 所需的 callback。

使用 pick() 雖然直覺，但若大部分的 property 都要抽取，則傳入 array 會非常大。

omit()

import { map, omit } from 'ramda'

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
]

let f = map(omit(['id']))

f(data) // ?

omit() 為 pick() 的 complement function，僅需提供不需要的 property array 即可，適合大部分 property 都要抽取的場合。

omit()
[String] → {String: *} → {String: *}
從一個 object 中忽略部分 property 成為新 object

[string]：要忽略的 property

{String: *}：data 為 object

{Strig: *}：回傳新的 object

Function Composition

import { map, omit, compose } from 'ramda'

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
]

let f = compose(map, omit)

f(['id'])(data) // ?

若想將 property 以 argument 提供，可使用 compose() 組合 map() 與 omit()，但因為 omit() 要求 array，因此 user 必須傳入 array。

of()

import { map, omit, compose, of } from 'ramda'

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
]

let f = compose(map, omit, of)

f('id')(data) // ?

若希望 user 只提供單一值即可，可先組合 of() 產生 array 後再傳給 omit()。

dissoc()

import { map, compose, dissoc } from 'ramda'

let data = [
  { id: 1, title: 'FP in JavaScript', price: 100 },
  { id: 2, title: 'RxJS in Action', price: 200 },
  { id: 3, title: 'Speaking JavaScript', price: 300 }
]

let f = compose(map, dissoc)

f('id')(data) // ?

事實上若只想單一 property，Ramda 的 dissoc() 更適合。

Conclusion

  • pick() 與 omit() 為互補，可視需求靈活運用
  • dissoc() 就功能上與 omit() 相同，唯 dissoc() 只能去除特定 property，適合 function composition，而 omit() 適合一次去除多個 property
