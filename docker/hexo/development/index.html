如何使用 Docker 打造 Hexo 可攜式開發環境 ?

2019-08-23

本文結合 Yarn Script + Dockerfile + Docker-compose，將 Hexo + Suka 開發環境包在 Docker Image ，可快速 Deploy 到其他電腦。

Version

macOS Mojave 10.14.6
Docker Deskop for macOS 2.1.0.1(37199)
WebStorm 2019.2
Node 12.9.0
Yarn 1.17.3
Hexo 3.9.0

Why Docker ?

Hexo 雖然可透過 Node 在本機簡單安裝，但若真的要將既有 Hexo + theme 開發環境 deploy 到其他電腦，會發現以下問題：

  • Hexo 的 theme 通常也會有自己的 dependency，當在其他電腦安裝時，可能發現 module 已經不見而無法安裝
  • 固然可將 node_modules 也放進 git 避免 module 不見問題，但各平台安裝的 node_modules 也不盡相同
  • Node 隨版本更新速度越來越快，以往為了更新 Node 版本，必須在每台電腦使用 NVM 安裝新版本

比較好的方式是將 Hexo 開發環境包成 Docker image：

  • Dependency 都包在 image 內，因此其他電腦不須再安裝 module
  • 不必將 node_modules 放進 git
  • Node 版本更新，重新 build image 即可

Hexo-suka Project

將以 Hexo + Suka theme 包成 image。

  • dist：即將包進 image 的檔案，也就是 Hexo 與 Suka 檔案，簡單說就是不含 node_modules 、 source 與 public 目錄，都是不會修改部分
  • src：原 Hexo project 的 source 部分，包含 markdown 與 asset
  • dockerfile：即將包成 image 的設定檔
  • docker-compose.yml：建立 container 的設定檔
  • package.json：並不是用來管理 dependency，而是用其 Yarn script 執行 Docker command 與 Hexo command

dockerfile

FROM node:alpine
RUN yarn global add hexo-cli
COPY ./dist ./hexo-suka
WORKDIR ./hexo-suka
RUN yarn install
RUN apk update
RUN apk add git
RUN apk add openssh-client
RUN git config --global user.name user
RUN git config --global user.email email

第 1 行

FROM node:alpine

使用最新版 Node 的 alpine 版本，file size 較小，適合作為 image。

第 2 行

RUN yarn global add hexo-cli

Node image 內已經包含 Yarn，可直接使用 Yarn 安裝 Hexo CLI。

第 3 行

COPY ./dist ./hexo-suka

將 dist 目錄下所有檔案複製進 image 內 hexo-suka 目錄下。

dist 目錄為 Hexo project 中不變的部分，適合包進 image

第 4 行

WORKDIR ./hexo-suka

將目前目錄移到 hexo-suka，觀念上類似 cd ./hexo-suka。

第 5 行

RUN yarn install

由於複製進 image 內檔案包含 package.json，使用 yarn install 安裝 node_modules 目錄下檔案，此為 Hexo 與 Suka 所需 dependency。

因為各平台的 node_modules 檔案不盡相同，因此需重新安裝

第 6 行

RUN apk update
RUN apk add git
RUN apk add openssh-client

因為 hexo deploy 會用到 git 與 ssh，但 node:alpine 並沒有包含，所以必須使用 apk add 自行安裝 git 與 ssh。

若使用 node:latest image 則不必自行安裝 git 與 ssh，但 image size 會大很多

第 9 行

RUN git config --global user.name user
RUN git config --global user.email email

安裝完 git 之後，一定要設定 user.name 與 user.email 才能執行，為了不將 user 與 email 寫死在 dockerfile，使用了 user 與 email arg，稍後 docker build 時會傳進來。

docker-compose.yml

version: "3"
services:
  hexo-suka:
    image: hexo-suka:0.0.2
    container_name: MyBlog
    restart: always
    ports:
      - 4000:4000
    tty: true
    volumes:
      - ./src:/hexo-suka/source
      - ~/.ssh:/root/.ssh

第 3 行

hexo-suka:

Service 名稱取為 hexo-suka 。

第 4 行

image: hexo-suka:0.0.1

使用 image 為 hexo-suka:0.0.1 。

第 5 行

container_name: MyBlog

Container 名稱取名為 MyBlog，方便稍後在 package.json 中使用 Yarn script 控制 container。

第 6 行

restart: always

當 container crash 時，會自動重新啟動。

第 7 行

ports:
  - 4000:4000

將 container 的 4000 port export 到 localhost 的 4000 port，將來 hexo server 時，可由外部的 browser 存取。

第 9 行

tty: true

Container 執行時，並沒有任何 process 執行，導致 container 會立即結束，因此特別加上 tty: true 在背景執行避免釋放。

第 10 行

volumes:
  - ./src:/hexo-suka/source
  - ~/.ssh:/root/.ssh

不想包進 image 部分：

  • src：也就是 Hexo project 的 source 部分，為 markdown 與 asset，可自行將此以 git 管理
  • ~/.ssh：hexo deploy 將會以 git 上傳到 GitHub，因此 container 將借用 host 的 .ssh 設定

package.json

{
  "name": "hexo-suka",
  "version": "0.0.2",
  "private": true,
  "git": {
    "name": "oomusou",
    "email": "oomusou@gmail.com"
  },
  "scripts": {
    "build": "docker build -t hexo-suka:$npm_package_version --build-arg name=$npm_package_git_name --build-arg email=$npm_package_git_email .",
    "tar": "docker save hexo-suka:$npm_package_version > hexo-suka-$npm_package_version.tar",
    "load": "docker load < ./hexo-suka-$npm_package_version.tar",
    "tgz": "cd .. && tar -zcf MyBlog.tgz MyBlog",
    "up": "docker-compose up -d",
    "down": "docker-compose down",
    "serve": "docker exec -it MyBlog hexo server",
    "clean": "docker exec -it MyBlog hexo clean",
    "gen": "docker exec -it MyBlog hexo generate",
    "new": "docker exec -it MyBlog hexo new",
    "deploy": "docker exec -it MyBlog hexo deploy",
    "all": "yarn clean && yarn gen && yarn deploy",
    "cli": "docker exec -it MyBlog sh"
  }
}

第 10 行

"build": "docker build -t hexo-suka:$npm_package_version --build-arg name=$npm_package_git_name --build-arg email=$npm_package_git_email .",

在 host 使用 yarn build 執行 docker build 建立 docker image。

-t：image 版本編號將直接由 package.json 的 version 提供，使用 $npm_package_version 即可讀取到 version 資訊。

--build-arg：由於在 dockerfile 有 RUN git config --global user.name user 與 RUN git config --global user.email email 需要 user 與 email arg，因此特別使用 --build-arg 將資料傳進去。

第 5 行

"git": {
  "name": "oomusou",
  "email": "oomusou@gmail.com"
},

至於所需的 name 與 email，則在 git property 建立，Yarn script 可以 $npm_package_git_name 與 $npm_package_git_email 讀取到。

11 行

"tar": "docker save hexo-suka:$npm_package_version > hexo-suka-$npm_package_version.tar",

在 host 使用 yarn tar 執行 docker save 將 image 匯出成 tar。

12 行

"load": "docker load < ./hexo-suka-$npm_package_version.tar",

在 host 使用 yarn load 將 tar 匯入為 image。

13 行

"tgz": "cd .. && tar -zcf hexo-suka-$npm_package_version.tgz hexo-suka",

在 host 使用 yarn tgz 將整個 project 壓成 tgz。

若要將整個 Hexo 環境快速 deploy 到其他電腦：

  • 使用 yarn tar 將 image 匯出成 tar
  • 使用 yarn tgz 將整個 project 連同 tar 壓成 tgz
  • 將 tgz 複製到其他電腦
  • 使用 tar -xvf 將 tgz 解壓縮
  • 使用 tarn load 將 tar 匯入為 image

如此整個 Hexo 開發環境就建置完成了。

14 行

"up": "docker-compose up -d",

在 host 使用 yarn up 執行 docker-compose up -d 啟動 container。

第 15 行

"down": "docker-compose down",

在 host 使用 yarn down 執行 docker-compose down 停止 container。

16 行

"serve": "docker exec -it MyBlog hexo server",

在 host 使用 yarn serve 執行 hexo server 啟動 web server。

在 host 一樣使用 http://localhost:4000 preview，與原本 Hexo 跑在本機行為完全一樣。

17 行

"clean": "docker exec -it MyBlog hexo clean",

在 host 使用 yarn clean 執行 hexo clean 將 public 目錄刪除，將 db.json 刪除。

18 行

"gen": "docker exec -it MyBlog hexo generate",

在 host 使用 yarn gen 執行 hexo generate 將 markdown 編譯成 html。

19 行

"new": "docker exec -it MyBlog hexo new",

在 host 使用 yarn new [filename] 執行 hexo new 建立新的 markdown 檔案。

20 行

"deploy": "docker exec -it MyBlog hexo deploy",

在 host 使用 yarn deploy 執行 hexo deploy publish 到 GitHub。

21 行

"all": "yarn clean && yarn gen && yarn deploy",

在 host 使用 yarn all 一次執行 hexo clearn、hexo generate 與 hexo deploy。

22 行

"cli": "docker exec -it MyBlog sh"

在 host 使用 yarn cli 進入 container 內，可自行輸入特殊的 Linux 與 Hexo command。

Conclusion

  • 可將 Hexo Docker image 視為 Hexo 跨平台執行版本，只要安裝了 image，不必重新 yarn install 也能執行 Hexo，也不用擔心 module 不見而無法安裝
  • 控制 docker-compose 部分，其實不用捨近求遠使用 Bash 或 Makefile，別忘了 JavaScript 生態圈有強大的 Yarn script + Node.js 可用
  • 原本 Hexo 常用 command 都一樣，只是由 hexo 改成 yarn 而已
  • 透過 yarn tar 與 yarn tgz 壓成 tgz 後，可將 tgz 複製到其他電腦解壓縮，執行 yarn load 就可還原整個 Hexo 開發環境
