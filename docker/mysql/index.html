使用 Docker 安裝 MySQL

2019-11-17

若要在 macOS 使用 MySQL，不需使用 Homebrew 安裝 macOS 版 MySQL 或安裝 MAMP，直接使用 Docker 安裝 Docker 版 MySQL 即可。

Version

macOS Catalina 10.15
Docker Desktop 2.1.0.4 (39773)
MySQL 8.0.18
WebStorm 2019.2.4
Database Tools and SQL 192.7142.35

Docker Compose

docker-compose.yml

version: "3"

services:
  mysql:
    image: mysql:latest
    container_name: MySQL
    volumes:
      - ${MYSQL_HOST_DIR}:/var/lib/mysql
    ports:
      - ${MYSQL_PORT}:3306
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
    restart: always

第 5 行

image: mysql:latest

使用 MySQL 最新版 image。

第 6 行

container_name: MySQL

定義 container 名稱為 MySQL。

第 7 行

volumes:
  - ${MYSQL_HOST_DIR}:/var/lib/mysql

設定 MySQL 實際儲存在 host os 路徑，避免 container 結束後 database 也跟著刪除。

第 9 行

ports:
  - ${MYSQL_PORT}:3306

設定 MySQL 對外使用 host os 的 port。

11 行

environment:
  - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}

設定 MySQL root 的 password。

13 行

restart: always

設定當 MySQL crash 時會自動重啟。

Environment Variable

.env

MYSQL_HOST_DIR=.mysql
MYSQL_PORT=3306
MYSQL_ROOT_PASSWORD=demo

建立 .env 設定 docker-compose.yml 所需環境變數。

  • MYSQ_HOST_DIR：設定 MySQL database 要存到 Host OS 的位置
  • MYSQL_PORT：設定 MySQL 對應到 Host OS 的 port
  • MYSQL_ROOT_PASSWORD：設定 root 的 password

Start MySQL

$ docker-compose up -d

使用 docker-compose up -d 啟動 MySQL。

Connect MySQL

在 WebStorm 的 Database 下新增 Data Source，選擇 MySQL。

  • Host：localhost
  • User：root
  • Password：demo

按 OK 建立 connection。

連進去後能看到 information_schema 眾多 table，表示 MySQL 成功啟動。

Stop MySQL

$ docker-compose down

使用 docker-compose down 結束 MySQL。

Conclusion

  • 對於 Linux 的 server，不必再如以前安裝 macOS 版本，畢竟移植後還是有些差異，比較好的方式是改用 Docker 版本，不僅原汁原味，且安裝移除都很乾淨
  • 使用 Docker 可使每個 project 都有自己的 MySQL，而不需如傳統必須所有 project 共用同一個 MySQL，導致 database 滿天飛
