深入淺出 Functor

2020-02-12

Functor 是學習 FP 第一個會遇到的 Algebraic Data Type，了解它有助於靈活使用 Function Composition。

Version

macOS Catalina 10.15.3
VS Code 1.42.0
Quokka 1.0.277
Ramda 0.26.1

Definition

學習 Ramda 時，第一個要學的就是 map()，但其 signature 卻給你一個大大的 Functor，為什麼不是 list 或 array 呢 ?

Functor
可使用 map() 的資料結構

白話就是 mappable，只是用了 functor 字眼聽起來很嚇人。

Array

import { map, inc } from 'ramda'

let data = [1, 2, 3]

map(inc)(data) // ?

最典型的 functor 就是 array，我們可使用 map() 改變 array 內的 element。

Object

import { map, inc } from 'ramda'

let data = {
  x: 1,
  y: 2,
  z: 3
}

map(inc)(data) // ?

Object 也是 functor，map() 會直接對所有 property 套用傳入 function。

Ramda 的文件也明確指出 map() 適用於 object，因此 object 也是 functor。

String

既然 array 是 functor，而 string 本質是 array of char，所以 string 也是 functor 嗎 ??

注意 map() 的定義，其 callback 為 a -> b，結果為 f b。

當然 b 可以是 a，因此就是：

Functor f => (a -> a) -> f a -> f a

這就是 string，但若 callback 為 a -> b 時，string 就不適用，因此 string 不是 functor。

Ramda 很多 function 都是 array 與 string 共用，此時 signature 會特別加上 string，若 string 是 functor，其 signature 大可寫成：

Functor f => f a -> f a -> f a

但 Ramda 文件並沒有這樣寫，可見 string 並不是 functor。

不是說能使用 map() 就是 functor 嗎 ?

將 123 套用 map() 也能得出 [2, 3, 4] 啊！

但別忘了 map() 定義是 f a 變成 f b，也就是其 f 是相同的，但現在卻是 string 變 array，已經是不同 f。

這是因為 ECMAScript 的 string 可視為 array of char，因此 123 被視為 ['1', '2', '3'] 操作，如此 f 仍是相同的。

這只能證明 array 是 functor，但 string 不是 functor。

Function

import { multiply, inc, map, compose } from 'ramda'

let double = multiply(2)

let data = 2

let f1 = map(inc)(double)
let f2 = compose(inc, double)

f1(data) // ?
f2(data) // ?

double() 為 function，若將 double() 拿去 map() 傳入 inc() 會如何 ?

f1 仍是 function，也可發現在此 map() 等效於 compose()。

什麼 ? 所以 function 也是 functor ?

雖然 functor 的定義是 可使用 map() 的資料結構，但綜觀之前的 array 與 object，可發現 functor 類似一個 container，內包含可使用 map() 的 有限 element。

double() 可想像成一個 container，內包含可使用的 map() 的 無限 element，任意 x 經過 double() 成為任意 y，因為 map() 定義為 f a -> f b，所以 double() 使用 map() 結果仍然是 function。

因此 function 去 map() 另外一個 function，也相當於兩個 function 去 compose()。

Ramda 的文件也明確指出 map() 適用於 function，因此 function 也是 functor。

Simple Functor

import { map, inc } from 'ramda'

class Functor {
  constructor(val) {
    this._val = val
  }

  map(f) {
    return new Functor(f(this._val))
  }
}

let data = new Functor(2)
map(inc)(data) // ?

Array、object 與 function 都是 ECMAScript 所提供的，我們能自行實作 functor 嗎 ?

凡實作 map() 的 class，Ramda 的 map() 都會呼叫其 map()。

還記得 functor 也稱為 mappable 嗎 ?

若以 OOP 角度看 functor：

凡實作 mappable interface 的 class 則稱為 functor，其中 mappable interface 包含 map()

Ramda 的 map() 文件也說如果第二個 argument 有 map() 則呼叫之。

因為 object 提供 map()，可使用 Ramda 的 map()，因此是 functor。

The Functor Laws

學習 functor 有什麼用呢 ? 因為 functor 有兩個重要特性：

Identity Law

import { map, identity } from 'ramda'

let data = [1, 2, 3]

map(identity)(data) // ?

若 map() 傳入 identity()，則回傳自己，其中 identity = x => x

Identity law
Functor.map(x => x) === Functor

這相當於數學的 a + 0 = a 或 a * 1 = a。

Composition Law

import { map, inc, multiply, pipe } from 'ramda'

let data = [1, 2, 3]

let f1 = pipe(
  map(multiply(2)),
  map(inc)
)

let f2 = pipe(
  multiply(2),
  inc
)

f1(data) // ?
map(f2)(data) // ?

第 5 行

let f1 = pipe(
  map(multiply(2)),
  map(inc)
)

f1(data) // ?

實務上常常會 map() 多次。

12 行

let f2 = pipe(
  multiply(2),
  inc
)

map(f2)(data) // ?

相當於將所有 callback 組合起來一次 map()。

Composition law
Functor.map(g).map(f) === Functor.map(f(g))

這相當於數學的 ax + ay = a(x + y)。

實務上常常對於 functor 使用 composition law，如此只要 map() 一次即可

Conclusion

  • 不只 array 可以使用 map()，object 與 function 也可使用 map()
  • 為什麼 string 不可使用 map() ? 因為 string 不是 functor
  • 多次 map() 可透過 composition law 先組合 function，如此只使用一次 map() 即可
  • Functor 並不是什麼黑魔法，我們也可自行實作 functor，只要有 map() 即可
