使用 Functor 的 Associative Law

2020-03-27

Array 是最典型的 Functor，藉由 Associative Law，我們可更靈活使用 map()。

Version

macOS Catalina 10.15.4
VS Code 1.43.2
Quokka 1.0.284
Ramda 0.27.0

map()

let data = [1, 2, 3]

let f = arr => arr
  .map(x => x * 2)
  .map(x => x + 1)

f(data) // ?

若要使 array 的每個 element 乘以 2，最後再加 1，可直接搭配兩次 map() 與 arrow function 完成。

Named Function

let data = [1, 2, 3]

let double = x => x * 2
let add = x => x + 1

let f = arr => arr
  .map(double)
  .map(add)

f(data) // ?

亦可將 arrow function 改用 named function。

若只使用 ECMAScript，能重構的步驟大概就只能如此

Ramda

import { map, inc, multiply, pipe } from 'ramda'

let data = [1, 2, 3]

let f = pipe(
  map(multiply(2)),
  map(inc)
)

f(data) // ?

若使用 Ramda，可直接使用 Ramda 內建的 inc()，double() 亦可由 multiply(2) 產生。

Associative Law

import { map, inc, multiply, pipe } from 'ramda'

let data = [1, 2, 3]

let mapper = pipe(
  multiply(2),
  inc
)

let f = map(mapper)

f(data) // ?

連續的 map() 相當於使用 pipe() 組合兩個 map()。

f(x,y) = ax + ay

可將 f() 抽象成以上數學式： pipe() 相當於 +，a 相當於 map()。

Array 是 ADT 的 functor，支援以下特性：

function.map(g).map(f) === functor.map(x => f(g(x)))

也就是數學的 composition law。

f(x, y) = ax + ay = a(x + y)

第 5 行

let mapper = pipe(
  multiply(2),
  inc
)

因此可先使用 pipe() 組合 multiply(2) 與 inc()。

第 10 行

let f = map(mapper)

最後只要使用一次 map() 即可。

Conclusion

  • 為什麼 functor 可使用 associative law 呢 ? 因為 functor 本質上只是一種 mapping，如同 pure function 一樣，這與 math function 類似，因此數學的 associative law 也能適用於 fuctor
  • Pure function 雖然限制很多，但優點是因為與 math function 一樣，因此很多數學觀念與公式都能套用在 pure function，這就是 FP 的優勢
  • 藉由 functor 的 associative law，我們可靈活組合多次 map function，最後只需一次 map() 即可
