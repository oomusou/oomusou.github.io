使用 Alpine Style 實作 Todo List

2024-01-28

Todo List 常用來練習不同前端 Framework，直接在 Hugo 平台使用 Petite-vue 以 Alpine Style 實現 Todo List。

Version

Hugo 0.121.2
Petite-vue 0.41

Todo List

<!doctype html>
<html>
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <script src="js/petite-vue.iife.js" defer init></script>
    {{ $title := "Hugo X Petite-vue Lab" }}
    <title>{{ $title }}</title>
  </head>
  <body v-scope="{
    newTodo: '',
    todos: [],
    filter: () => true,
    get active() {
      return this.todos.filter(x => !x.isCompleted)
    },
    get completed() {
      return this.todos.filter(x => x.isCompleted)
    },
    get filteredTodos() {
      return this.todos.filter(this.filter)
    }
  }">
    <div>
      <input type="text" v-model="newTodo" />
      <button @click="
        if (!newTodo) return

        todos = [...todos, {
          id: Symbol(),
          isCompleted: false,
          isEdit: false,
          todo: newTodo
        }]
        newTodo = ''
      ">Add</button>
    </div>
    <div>
      <ul>
        <li v-for="(item, i) in filteredTodos" :key="i">
          <input type="checkbox" v-model="item.isCompleted" />
          <span v-if="!item.isEdit" v-text="item.todo"></span>
          <span v-else><input type="text" v-model="item.todo" /></span>
          <span v-if="!item.isEdit"><button @click="item.isEdit = true">edit</button></span>
          <span v-else><button @click="item.isEdit = false">save</button></span>
          <button @click="todos = todos.filter(x => x.id !== item.id)">delete</button>
        </li>
      </ul>
    </div>
    <div>
      <span v-text="active.length"></span> <span>items left</span>
      <button @click="filter = () => true">All</button>
      <button @click="filter = x => !x.isCompleted">Active</button>
      <button @click="filter = x => x.isCompleted">Completed</button>
      <button v-if="completed.length" @click="todos = todos.filter(x => !x.isCompleted)">Clear Completed</button>
    </div>
  </body>
</html>

Line 10

<body v-scope="{
  newTodo: '',
  todos: [],
  filter: () => true,
}">
  • 在 v-scope 內宣告 state
  • newTodo：儲存新輸入的 todo
  • todos：儲存所有 todo
  • filter：儲存 All、Active 與 Completed 所需要的 filter function

Line 14

get active() {
  return this.todos.filter(x => !x.isCompleted)
},
  • 在 v-scope 內宣告 getter
  • active：回傳 未完成 的 todo

Line 17

get completed() {
  return this.todos.filter(x => x.isCompleted)
},
  • 在 v-scope 內宣告 getter
  • completed：回傳 已完成 的 todo

Line 20

get filteredTodos() {
  return this.todos.filter(this.filter)
}
  • 在 v-scope 內宣告 getter
  • 根據目前 filter 內的 filter function 回傳 todo

Line 25

<input type="text" v-model="states.newTodo" />
  • v-model：將欲輸入的新 todo 以 雙向綁定 到 newTodo

Line 26

<button @click="
  if (!newTodo) return

  todos = [...todos, {
    id: Symbol(),
    isCompleted: false,
    isEdit: false,
    todo: newTodo
  }]
  newTodo = ''
">Add</button>
  • 直接將 click event handler 的代碼寫在 HTML 內
  • 使用 array destructure 展開 todos 實現新增 todo
  • id 使用 Symbol() 實現 UUID

Line 40

<ul>
  <li v-for="(item, i) in filteredTodos" :key="i">
    <span v-text="item.todo"></span>
  </li>
</ul>
  • v-for ：列舉 filteredTodos

Line 41

<input type="checkbox" v-model="item.isCompleted" />
  • v-model：將 checkbox 以 雙向綁定 到 item.isCompleted

Line 42

<span v-if="!item.isEdit" v-text="item.todo"></span>
<span v-else><input type="text" v-model="item.todo" /></span>
  • v-if：若不是 編輯模式，則直接顯示
  • v-else：否則顯示 <input>

Line 44

<span v-if="!item.isEdit"><button @click="item.isEdit = true">edit</button></span>
<span v-else><button @click="item.isEdit = false">save</button></span>
  • v-if：若不是 編輯模式，則顯示 edit button
  • v-else：否則顯示 save button

Line 46

<button @click="todos = todos.filter(x => x.id !== item.id)">delete</button>
  • 使用 filter() 取代 splice() 刪除 Array 中的 element

Line 51

<span v-text="active.length"></span> <span>items left</span>
  • 顯示目前尚未完成的 todo 筆數

Line 52

<button @click="filter = () => true">All</button>
  • 顯示所有 todo，直接提供 filter function

Line 53

<button @click="filter = x => !x.isCompleted">Active</button>
  • 顯示尚未完成 todo，直接提供 filter function

Line 54

<button @click="filter = x => x.isCompleted">Completed</button>
  • 顯示已完成 todo，直接提供 filter function

Line 55

<button v-if="completed.length" @click="todos = todos.filter(x => !x.isCompleted)">Clear Completed</button>
  • v-if：若有已完成 todo，則顯示 Clear Completed button
  • 使用 filter() 取代 splice() 刪除 Array 中的 element

Conclusion

  • Petite-vue 其實並不需如 Vue 一樣將 HTML 與 JavaScript 分開，可直接如 Tailwind CSS 寫在 HTML 即可
  • Getter 也算廣義的 data，所以寫在 v-scope 內
  • 若以 FP 風格開發 JavaScript，常常只有一行而已，這種寫在 HTML 剛剛好
  • Petite-vue = FP + HTML 算 coding style 最高境界，且實作也非常精簡，包含 HTML 僅需 58 行而已，比使用 Alpine 行數還少
