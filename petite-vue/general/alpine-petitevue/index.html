Alpine 與 Petite-vue 比較

2024-01-27

Alpine 與 Petite-vue 為兩個最適合 SSG 的 Library，實際使用 Alpine 與 Petite-vue 之後，發現兩者在 DX 還是存在一些差異。

Version

Hugo 0.121.2
Alpine 3.13.5
Petite-vue 0.41

Alpine

不能換 Dilimeter

  • 僅能使用 x-text，但 DX 上仍是 delimiter 較佳

沒有 x-else

  • Alpine 有提供 x-if，卻沒提供 x-else，因此只能連續使用兩次 x-if ，DX 體驗較差

x-if 與 x-for 必須搭配 template

  • x-if 與 x-for 需搭配 <template> 才能使用，HTML 會無故多了一層，DX 體驗較差

支援 ESM 不完整

  • Alpine 雖然能夠使用 ESM 啟動，卻無法支援以 ESM 方式抽出代碼

依賴 this

  • 當 HTML 與 JavaScript 分開寫時，代碼風格類似 Vue 的 Options API，仍然強烈依賴 this

Petite-vue

能換 Dilimeter

  • 能換 dilimeter 避開 Go Template，DX 體驗較佳

支援 v-else

  • Petite-vue 支援 v-if 與 v-else，DX 體驗較佳

v-if 與 v-for 可搭配任何 tag

  • v-if 與 v-for 可直接搭配任何 HTML tag，不必多一層 <template>

完整支援 ESM

  • Petite-vue 完整支援 ESM，可如同 Vue 3 一樣以 ESM 方式抽出代碼

Composition API

  • Petite-vue 雖然沒支援完整 Composition API，僅能使用 reactive()，這使的不使用 this 為可能，代碼風格可類似 FP，都是一堆短短的 function 組合而成

Conclusion

  • 若要 HTML 與 JavaScript 寫在一起，則 Alpine 與 Petite-vue 的 DX 體驗差不多，但 Petite-vue 的 v-if、v-else、v-for 的 DX 體驗較佳，不需搭配 <template> 就可使用
  • 若要 HTML 與 JavaScript 分開寫，則 Alpine 寫起來類似 Options API，仍然強烈依賴 this
  • 若要 HTML 與 JavaScript 分開寫，因為 Petite-vue 能使用 Composition API 的 reactive()，因此不需依賴 this，可寫出 類似 Script Setup 風格的 Functional 寫法，雖然最後還是需要靠 createApp() 整合
  • Alpine 支援 ESM 有限，但 Petite-vue 則能完全支援 ESM
  • 若只是輕量級的 JavaScript 應用，想將 JavaScript 直接寫在 HTML 裡，則 Alpine 或 Petite-vue 的 DX 體驗差不多，但 Petite-vue 略勝
  • 若是較複雜的 JavaScript 應用，想將 JavaScript 與 HTML 分開寫，則 Petite-vue 的 DX 體驗較佳
  • 若喜歡 FP 以眾多短短的 function 所構成，則 Petite-vue 是首選
