如何在 Gridsome 使用 Tailwind UI ?

2020-07-15

Tailwind UI 基於 Tailwind CSS，使用之前須先設定好 Tailwind CSS，然後再設定 Tailwind UI。

Version

macOS Catalina 10.15.5
WebStorm 2020.1.3
Gridsome CLI 0.3.3
Gridsome 0.7.19
Tailwind 1.4.6

Tailwind CSS

Create Project

$ gridsome create gridsome-tailwindui

使用 Gridsome CLI 建立 project。

Install Package

$ yarn add tailwindcss

使用 Yarn 安裝 Tailwind。

在 Tailwind 1.4.0 之後，預設已經安裝 PurgeCSS。

$ yarn add autoprefixer --dev

使用 Yarn 安裝 Purgecss 與 AutoPrefixer 在 devDependencies。

  • AutoPrefixer：自動加上各 browser 的 vendor prefix，如 ::-webkit-input-、::-moz-、::-mu-input … 等

Add Tailwind to CSS

src/assets/css/tailwind.css

@tailwind base;
@tailwind components;
@tailwind utilities;

新增 src/assets/css/tailwind.css。

Tailwind 在 build-time 會將以上以標準 CSS 取代。

Tailwind Configuration

$ npx tailwindcss init

使用 tailwind init 在 project 根目錄建立 tailwind.config.js。

tailwind.config.js

module.exports = {
  purge: [
    './src/**/*.vue',
    './src/**/*.js',
    './src/**/*.jsx',
    './src/**/*.html',
    './src/**/*.pug',
    './src/**/*.md',
  ],
  theme: {
    extend: {},
  },
  variants: {},
  plugins: [],
}

Tailwind 1.4 新增 purge property，可直接設定 PuregeCSS 要管理的檔案。

Gridsome Configuration

gridsome.config.js

let tailwindcss = require("tailwindcss")
let autoprefixer = require('autoprefixer')

module.exports = {
  siteName: 'Gridsome',
  plugins: [],
  css: {
    loaderOptions: {
      postcss: {
        plugins: [
          tailwindcss,
          autoprefixer
        ],
      },
    },
  }
}

第 7 行

css: {
  loaderOptions: {
    postcss: {
      plugins: [
        tailwindcss,
        autoprefixer
      ],
    },
  },
}

在 gridsome.config.js 設定使用 Tailwind 與 AutoPrefixer。

PostCSS Configuration

postcss.config.js

module.exports = {
  plugins: [
    require('tailwindcss'),
    require('autoprefixer'),
  ]
}

在 project 根目錄建立 postcss.config.js。

Tailwind 基於 PostCSS，之前安裝的 Autoprefixer 也必須靠 PostCSS 啟動。

之前必須在 postcss.config.js 寫很多 PurgeCSS 設定，Tailwind 1.4 則不需要。

main.js

src/main.js

import DefaultLayout from '~/layouts/Default.vue'
import './assets/css/tailwind.css'

export default function (Vue, { router, head, isClient }) {
  // Set default layout as a global component
  Vue.component('Layout', DefaultLayout)
}

第 2 行

import './assets/css/tailwind.css'

在 main.js 引入 tailwind.css。

Vue Component

src/pages/index.vue

<template>
  <div>
    <h1 class="text-4xl font-bold text-red-700">Hello, world!</h1>
  </div>
</template>

<script>
export default {
  name: 'index',
}
</script>

第 3 行

<h1 class="text-4xl font-bold text-red-700">Hello, world!</h1>

直接在 <h1/> 套用 Tailwind 的 utility，可發現其可讀性很高，就算不熟 Tailwind，也能由其 class name 了解其意義。

Development Mode

$ yarn serve

啟動 development mode 的 DevServer。

可發現 Tailwind CSS 已經成功套用。

Production Mode

$ yarn build

使用 production mode 編譯 Gridsome 並啟動 PurgeCSS。

在 dist/css 目錄下，可發現 CSS 只有 4 kb，因為 PurgeCSS 只取出用到部分。

Tailwind UI

Install Package

$ yarn add @tailwindcss/ui

使用 Yarn 安裝 Tailwind UI。

Tailwind Configuration

tailwind.config.js

module.exports = {
  purge: [
    './src/**/*.vue',
    './src/**/*.js',
    './src/**/*.jsx',
    './src/**/*.html',
    './src/**/*.pug',
    './src/**/*.md',
  ],
  theme: {
    extend: {
      fontFamily: {
        sans: ['Inter var', ...defaultTheme.fontFamily.sans],
      },
    },
  },
  variants: {},
  plugins: [
    require('@tailwindcss/ui')({
      layout: 'sidebar',
    })
  ]
}

在 tailwind.config.js 加入 Tailwind UI 所需設定。

Add Inter Font

src/index.html

<!DOCTYPE html>
<html ${htmlAttrs}>
<head>
  <link rel="stylesheet" href="https://rsms.me/inter/inter.css">
  ${head}
</head>
<body ${bodyAttrs}>
  ${app}
  ${scripts}
</body>
</html>

Gridsome 預設並不如 Vue 有 public/index.html，所以要加入 Inter Font 遇到了小麻煩。

若要自行提供 index.html 給 Gridsome，只要在 src 目錄下新增 index.html，Gridsome CLI 會自動取代原本內建的 index.html。

第 4 行

<link rel="stylesheet" href="https://rsms.me/inter/inter.css">

根據 Gridsome 提供的 index.html 範本加入 Inter Font。
