VS Code 安裝與設定

2020-07-19

VS Code 並不是我的主力開發工具，只是用來搭配 Quokka 測試 ECMAScript，真正 Project 開發會使用 WebStorm。

Version

macOS Catalina 10.15.5
VS Code 1.38.1
Quokka 1.0.308

Installation

$ brew cask install visual-studio-code

使用 Homebrew 安裝 VS Code。

Extensions

Webstorm IntelliJ Darcula Theme

VS Code 模仿 WebStorm 最像的 theme。

Bracket Pair Colorizer 2

() 或 {} 會以相同顏色成對出現，特別適合 FP。

Quokka.js

即時顯示 ECMAScript 2015+ 的結果，還支援 import 外部 module，因此也適合用來學習 Ramda、Crocks、RxJS … 等 library。

Settings

Code -> Preferences -> Settings

Commonly Used

  • Editor: Tab Size：2

Text Editor

  • Parameter Hints: Enabled：false
  • Font -> Font Family：Fira Code
  • Font -> Font Ligatures：true
  • Font -> Font Size：16 (13")、13 (15")
  • Minimap -> Enabled：false

Workbench

  • Startup Editor：none
  • Appearance -> Activity Bar: Visible：false
  • Appearance -> Color Theme：Webstorm IntelliJ Darcula Theme
  • Appearance -> Sidebar: Location：right
  • Breadcrumbs -> Enabled：false
  • Editor Management > Show Icons：false
  • Editor Management -> Show Tabs：false

Window

  • Zoom Level：1

Features

  • Extensions -> Ignore Recommendations：true

Extensions

  • TypeScript -> JavaScript > Validate: Enable：false
  • TypeScirpt -> TypeScript > Validate: Enable：false
  • TypeScript -> JavaScript > Suggestion Actions: Enable：false
  • TypeScript -> TypeScript > Suggestion Actions: Enable：false

Status Bar 變黑色

~/Library/Application Support/Code/User/settings.json

"workbench.colorCustomizations": {
  "statusBar.background": "#303030",
  "statusBar.noFolderBackground": "#303030",
  "focusBorder": "#303030"
},

讓 Title 乾淨顯示 Untitled-1

Conclusion

  • VS Code 是一個需要深度客製化工具，由於我只配合 Quokka 使用，因此只做了一小部分設定而已
